# 安全政策

## 支持范围

当前维护分支为 `master`。安全问题会优先在最新 `master` 上确认、修复和发布说明。

## 漏洞报告

如果你发现安全漏洞、权限绕过、多租户数据隔离问题、敏感信息泄露、注入风险、认证缺陷或供应链风险，请不要先公开披露。请通过以下方式联系作者：

- Email: [chaoqiang.tian@gmail.com](mailto:chaoqiang.tian@gmail.com)
- WeChat: `chaoqiang68`

报告中建议包含：

- 受影响的模块、接口、页面或部署方式。
- 复现步骤和必要的请求样例。
- 影响范围评估，例如是否涉及数据泄露、越权、远程执行或权限提升。
- 你已经验证过的版本、分支或提交号。
- 建议修复方向，如有。

## 处理流程

1. 作者确认报告并判断影响范围。
2. 对高危问题优先建立私下修复方案。
3. 修复完成后补充测试、安全说明和必要的升级指引。
4. 在不暴露可利用细节的前提下公开说明影响范围和修复版本。

## 安全边界

请不要在未经授权的真实生产环境、第三方租户数据或他人系统上验证漏洞。允许在你自己的本地环境、测试环境或明确授权的环境中复现问题。

本项目限制商业使用。安全研究、漏洞报告或修复贡献不构成商业授权；商业使用仍需先取得作者书面同意。
