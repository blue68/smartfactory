## 1. 总体承接原则

这批工厂作业说明书，当前系统按四层承接：

1. `工序配置`
- 维护步骤
- 维护工时
- 维护步骤投料
- 维护规格说明和工艺参数

2. `工单`
- 冻结当前模板快照
- 固定步骤、投料、说明、参数

3. `排产`
- 继续只吃步骤、工作站、标准工时
- 不额外承接复杂说明书字段

4. `生产任务`
- 展示该步骤的作业说明
- 展示该步骤输入项、规格参数、工艺参数

换句话说：
- 说明书不是直接挂在工单或任务上编辑
- 它先落到 `工序配置`
- 再通过工单快照冻结
- 最后在任务页展示给执行人

## 2. 源表到系统的主映射

| 源说明书列/信息 | 当前界面维护位置 | 当前系统字段/结构 | 后续在哪体现 |
| --- | --- | --- | --- |
| 工序 | 工序配置 -> 步骤节点 | `process_steps.step_name` | 排产、生产任务 |
| 工序代码 | 当前不单独前台编辑，导入/脚本映射 | `catalogMatch.procedureCode`，后续可入扩展字段 | 排产口径、工艺识别 |
| 原材料 | 工序配置 -> 步骤投料与工艺参数 | `process_step_materials.input_sku_id` | 任务输入项 |
| 原材料尺寸 | 工序配置 -> 规格说明 | `process_step_materials.spec_text` | 任务页 `规格参数` |
| 单位 | 由物料 SKU 自身单位承接 | `skus.stock_unit / production_unit` | 任务输入项 |
| 净用量 | 工序配置 -> 单件净用量 | `process_step_materials.usage_per_unit` | 任务需求数量 |
| 损耗系数 | 工序配置 -> 损耗率 | `process_step_materials.loss_rate` | 任务投料口径 |

| 门幅 | 工艺参数(JSON) | `process_params_json.doorWidth` | 任务页 |
| 用料米数/面积值 | 单件净用量 + 参数 | `usage_per_unit` + `process_params_json` | 任务页 |

## 3. 你最关心的三类信息，系统现在怎么承接

### 3.1 路线类

源表典型值：
- `开料`
- `钻孔`
- `钉打`
- `裁剪`
- `扪制`
- `车缝`

当前承接：
- 页面：`工序配置 -> 步骤节点`
- 字段：`step_name`
- 运行体现：
  - 工单会冻结这条步骤
  - 排产按步骤排
  - 任务详情会显示当前步骤

这类已经具备主承接能力。

### 3.2 投料类

源表典型值：
- 原材料
- 规格尺寸
- 用量
- 损耗
- 材料属性

当前承接：
- 页面：`工序配置 -> 步骤投料与工艺参数`
- 字段：
  - `input_sku_id`
  - `usage_per_unit`
  - `loss_rate`
  - `spec_text`
  - `process_params_json`
  - `is_key_material`

运行体现：
- 工单快照冻结
- 任务详情显示输入项
- 输入项可继续显示规格参数、工艺参数

这类也已经具备主承接能力。

### 3.3 工艺尺寸/公式类

源表典型值：
- `1930mm x 195mm x 12mm`
- `门幅 1450`
- `面积 234360`
- `930 x 252`
- `F/右01护翼布-白01，M01，米02`

当前承接：
- 页面：
  - 简短文本 -> `规格说明`
  - 结构化参数 -> `工艺参数(JSON)`
- 字段：
  - `spec_text`
  - `process_params_json`

任务体现：
- 任务详情 -> 输入项卡片
- 显示：
  - `规格参数 xxx`
  - `工艺参数 xxx`

这类已经能承接，但当前展示还是“文本型展示”，不是结构化表格。

## 4. 页面级映射

### 4.1 工序配置页

路径：
- `/master-data/process-config`

页面职责：
- 这是“说明书落地”的主维护页

你现在能看到并维护的关键字段：

| 页面区块 | 字段 | 对应源说明书 |
| --- | --- | --- |
| 基本信息 | 步骤名称 | 工序 |
| 工时设置 | 标准工时 | 标准工时 |
| 工时设置 | 最大工时/极限工时 | 员工时 / 极限工时 |
| 操作说明 | guide_text / 附件 | 工艺说明、作业文件 |
| 步骤投料与工艺参数 | 单件净用量 | 用料数量 |
| 步骤投料与工艺参数 | 损耗率 | 损耗系数 |
| 步骤投料与工艺参数 | 规格说明 | 原材料尺寸 |
| 步骤投料与工艺参数 | 工艺参数(JSON) | 门幅、面积、裁片尺寸、公式 |
| 计件单价 | 单价 | 工价/计件价 |

结论：
- 如果你问“说明书主要应该在哪个界面维护”
- 答案就是：`工序配置`

### 4.2 工单

页面职责：
- 不主维护
- 只冻结快照

冻结内容：
- 步骤
- 工时
- 说明
- 附件
- 投料
- 规格说明
- 工艺参数

结论：
- 工单承接“冻结”
- 不承担日常编辑

### 4.3 排产

页面职责：
- 继续按步骤和工时排

当前体现：
- 使用步骤
- 使用工作站类型
- 使用标准工时

不建议这阶段往排产页塞过多说明书细节。

### 4.4 生产任务

页面职责：
- 把冻结下来的工艺细节展示给现场

当前可见：
- 任务输入 / 输出清单
- 输入项下的：
  - SKU
  - 需求量
  - 可用量
  - 仓库/库位
  - `规格参数`
  - `工艺参数`

结论：
- 任务页是“执行查看页”
- 不是主数据维护页

## 5. 现在已经支持到什么程度

### 5.1 已支持

- 工序步骤维护
- 工时维护
- 步骤投料维护
- 规格说明维护
- 工艺参数 JSON 维护
- 工单快照冻结
- 任务页展示规格参数/工艺参数

### 5.2 半支持

- 工序代码
  - 目前主要在导入/脚本层映射
  - 前台没有独立字段

- 工价来源
  - 已能承接单价
  - 但“工作/针工23元小时”这种原始表达，前台没有一比一展示

- 尺寸/面积/门幅
  - 已能存
  - 但任务页当前还是文本拼接展示

### 5.3 还未完成

- 作业说明书一键导入的正式前台入口
- 参数结构化表格展示
- 步骤投料的完整新增/删除/排序 UI
- 多工步模板批量重构助手

## 6. 对你当前业务最重要的判断

如果你的目标是：
- 不改工单/排产/任务主流程
- 先把工厂说明书落进系统
- 后续让工人能在任务里看到关键工艺信息

那现在这套承接关系已经是正确方向：

`说明书`
-> `工序配置维护`
-> `工单快照冻结`
-> `排产吃步骤和工时`
-> `生产任务展示规格/参数`

这条链已经能跑。

真正下一步需要补的，不是推翻主链路，而是两件事：
- 把说明书数据批量导进 `工序配置`
- 把任务页里的参数展示做得更适合现场看

## 7. 建议你本地先这样验

1. 打开 `工序配置`
- 看步骤抽屉里是否能维护：
  - 单件净用量
  - 损耗率
  - 规格说明
  - 工艺参数(JSON)

2. 创建或查看一个已有工单/任务
- 看任务详情输入项是否出现：
  - `规格参数`
  - `工艺参数`

3. 判断是否符合工厂使用习惯
- 如果觉得“JSON 对现场维护不友好”
- 下一步应该做“参数表单化”
- 而不是改工单或排产主流程
