# 智造管家商业化 UI 升级规范

更新时间：2026-04-23

## 1. 参考图拆解

### 参考图 1：生产驾驶舱风格

优点：
- 有明确的“数据看板 + 明细 + 任务卡片”层级。
- 头部与核心区采用浅蓝渐变，具备轻微科技感。
- KPI、表格、任务卡片的关系很清楚，适合 MES/ERP 这种高密度信息系统。

可借鉴点：
- 顶部品牌区采用柔和渐变氛围，而不是纯色平铺。
- 关键指标卡、明细表格、任务卡片要统一圆角和边框。
- 生产类页面要强调“状态、进度、风险”的视觉可扫描性。

不建议直接照搬：
- 页面局部信息略多，如果全部页面都保持这个密度，会显得压迫。

### 参考图 2：轻商务后台风格

优点：
- 整体留白舒适，商务感强，容易商业化。
- 左侧导航、顶栏、数据卡片、列表区域都很规整。
- 信息层级清晰，阅读成本低。

可借鉴点：
- 采用大面积白色卡片和低对比浅灰背景，显得干净、稳定。
- KPI 数字、任务区、列表区采用一致卡片容器。
- 适合销售、采购、主数据、系统管理这类操作型页面。

不建议直接照搬：
- 科技感偏弱，直接整体照搬会让智能制造系统失去行业特征。

### 参考图 3：工业设备控制面板风格

优点：
- 强调控件规整性，输入框、参数块、分组面板非常统一。
- 左右分区、工具栏、参数表单的秩序感很强。
- 更适合主数据配置、工艺配置、参数设置这类复杂表单页面。

可借鉴点：
- 表单面板要有强分组感和稳定网格。
- 参数输入区适合采用大圆角、低阴影、清晰分隔线。
- 工具栏和操作区可以更强调“设备/生产软件”的专业感。

不建议直接照搬：
- 黑白对比与设备 UI 气质较重，若全站都用会显得生硬。

## 2. 综合后的统一风格定位

统一风格定义：

`轻科技商务制造风`

关键词：
- 商业化
- 稳定
- 清晰
- 轻科技
- 高密度但不压迫
- 可长期扩展

最终策略：
- 以参考图 2 的商务整洁感作为基础骨架
- 融合参考图 1 的蓝青科技氛围和制造场景识别度
- 吸收参考图 3 的参数面板秩序感和专业控制气质

## 3. 视觉原则

### 3.1 页面气质

- 整体必须明亮，不使用大面积深色背景。
- 背景以浅灰蓝为底，叠加轻微科技感渐变与柔和光斑。
- 卡片统一采用“白色面板 + 细描边 + 低饱和阴影”。
- 所有模块必须看起来属于同一套系统，而不是多个模板拼接。

### 3.2 颜色策略

- 主色：工业蓝 `#2563EB ~ #3B82F6`
- 辅色：青蓝 `#0EA5E9 / #14B8A6`
- 强提醒：橙色 `#F59E0B / #F97316`
- 成功：绿色 `#22C55E`
- 错误：红色 `#EF4444`
- 中性底色：`#F4F7FB / #F8FAFC / #FFFFFF`
- 边框主色：`#D9E5F4`
- 主文本：`#142033`
- 次文本：`#607089`

原则：
- 蓝色用于主操作、主导航、关键态。
- 橙色只用于提醒、AI、风险、紧急类信息。
- 成功/错误严格语义化，不允许到处乱用。

### 3.3 圆角与阴影

- 页面主卡片：18px ~ 22px
- 表单控件：14px ~ 16px
- 按钮：14px，胶囊按钮可用 9999px
- 大弹框 / 抽屉：24px

阴影原则：
- 默认阴影轻，避免飘。
- Hover 提升不超过一个层级。
- 不使用浓黑阴影，改用低透明蓝灰阴影。

## 4. 布局规范

### 4.1 系统框架

- 顶栏：白色半透明商务顶栏，带轻玻璃质感。
- 侧栏：浅色导航面板，偏商务后台风格，当前激活项使用蓝色高亮胶囊。
- 内容区：统一大留白和稳定页面边距。

### 4.2 内容模块

- KPI 区：采用统一数据卡容器。
- 筛选区：单独卡片承载，不直接裸露在背景上。
- 列表区 / 表格区：统一卡片外壳。
- 详情区 / 表单区：采用“面板 + 分组标题 + 规则网格”结构。

### 4.3 页面节奏

- 首屏先看关键数据，再看操作，再看明细。
- 同一屏幕不出现两种以上完全不同的卡片语法。
- 模块间距统一，避免页面上下跳动感。

## 5. 组件规范

### 5.1 按钮

- 主按钮：蓝色渐变，具备轻投影和轻发光。
- 次按钮：白底蓝边，hover 微蓝底。
- 文字按钮：仅用于低优先级次操作。
- 危险按钮：红色，但保持和主按钮同一结构语法。

按钮必须满足：
- 高度统一
- 左右 padding 统一
- 字重统一
- hover / active / disabled 明确

### 5.2 输入框与下拉框

- 所有输入框、下拉框、文本域统一高度、圆角、边框和聚焦效果。
- 聚焦态不允许出现粗暴双层描边。
- placeholder 统一偏浅灰蓝。
- 包裹式搜索框由外层壳负责聚焦，不允许内层再套第二层边框。

### 5.3 卡片

- 卡片表面统一：白底、细边框、浅阴影。
- KPI 卡允许有左侧色条或顶部强调线，但不允许花哨。
- 风险卡、AI 卡、状态卡在同一套卡片语法里，只改变强调色。

### 5.4 表格

- 表头背景统一为浅灰蓝。
- 表头字体更轻，但更整齐。
- 行 hover 用浅蓝，不用深色反转。
- 分页、筛选、表格容器必须同一语言。

### 5.5 Modal / Drawer

- 使用更大的圆角和更清晰的头部区。
- 大弹框偏“汇报式面板”，不是传统生硬对话框。
- 内容区允许大块白色面板与清晰分组。

### 5.6 Tag / Badge / Status

- Tag 采用柔和填充，不用过强对比。
- 状态色必须统一映射。
- 所有英文状态展示优先转换为中文。

## 6. 页面级应用原则

### 6.1 销售、采购、系统管理

- 以商务后台感为主。
- 强调筛选、列表、详情、审批的清晰层次。

### 6.2 生产、排产、任务、缺料

- 在商务后台骨架之上增加轻科技感和状态扫描能力。
- 允许更强的状态标签、进度条、任务卡，但结构必须仍然整齐。

### 6.3 主数据、工艺、配置中心

- 更强调参数面板和表单秩序。
- 分组结构清楚，避免过多装饰。

## 7. 本轮实现边界

本轮只做：
- 设计 token 升级
- 全局布局升级
- 共用组件升级
- 通过共用层提升全站统一感

本轮不做：
- 业务交互逻辑重构
- 页面信息结构重排
- 功能新增或权限逻辑调整

## 8. 验收标准

- 销售、采购、生产、主数据、系统管理从第一眼上看属于同一套产品。
- 整体气质达到可对外演示、可商业交付的程度。
- 所有输入框、按钮、卡片、表格、弹框风格统一。
- 页面具备适度科技感，但不浮夸、不廉价。
- 现有功能、交互、数据请求不被破坏。
