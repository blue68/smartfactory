## 四、SDD 模式开发要求

基于评审发现的问题，对前后端工程师提出以下 SDD 开发规范：

### 4.1 后端 SDD 规范
1. 每个模块必须包含完整的 Entity → Repository → Service → Controller → Routes 五层
2. 所有数据库查询必须通过 BaseRepository 或显式携带 tenant_id
3. Controller 层必须使用 Zod Schema 校验所有输入参数
4. Service 层必须定义明确的 Interface（输入/输出类型）
5. 错误码必须在 ApiResponse.ts 中集中注册

### 4.2 前端 SDD 规范
1. 每个页面必须参照 UI 设计稿 100% 还原
2. 类型定义必须与后端 API 响应完全对齐（models.ts + enums.ts）
3. API 调用必须通过 React Query Hooks 封装（api/ 目录）
4. 组件必须使用项目通用组件库（components/common/）
5. 样式必须使用 CSS Module + 项目 Design Tokens
