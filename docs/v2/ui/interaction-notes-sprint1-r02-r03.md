## 1. R-02 供应商导出

### 1.1 入口位置

导出按钮位于供应商列表页的搜索筛选栏右侧，与「新增供应商」等操作按钮并排显示，使用 `btn--secondary` 样式。

```
[搜索框] [状态筛选] [品类筛选]  ·····空间·····  [📥 导出 Excel]
```

导出按钮遵循**权限可见性**规则：仅 admin / supervisor 角色可见；工人和销售角色不渲染该按钮（前端主动判断 role，不依赖接口）。

### 1.2 用户操作流程

```
用户点击「导出 Excel」
  ↓
按钮切换为 Loading 态（禁用，显示旋转图标 + 文字「导出中…」）
  ↓
前端调用 GET /suppliers/export?{当前筛选参数}
  ↓
  ├─ 成功（200）→ 触发浏览器下载（文件名：供应商列表_YYYY-MM-DD.xlsx）
  │               按钮恢复正常态
  │               Toast 成功：「导出成功，供应商列表_YYYY-MM-DD.xlsx 已开始下载」
  │
  ├─ 空数据（后端返回 204 / 空列表）→ 不触发下载
  │               Toast 警告：「当前筛选条件下无数据，无法导出」
  │               按钮恢复 disabled 状态
  │
  └─ 失败（4xx / 5xx）→ 按钮恢复正常态
                  Toast 错误：「导出失败，请稍后重试」
```

### 1.3 状态定义

| 状态 | 触发条件 | 视觉表现 |
|------|---------|---------|
| 正常（可点击） | 当前筛选有数据且用户有权限 | `btn--secondary`，灰色边框 |
| Loading（导出中） | 点击后等待接口返回 | 禁用 + 旋转 spinner + 文字「导出中…」 |
| 禁用（空数据） | 当前筛选无结果 | `disabled` 样式，鼠标 `cursor: not-allowed`，Tooltip 说明「当前无数据」 |
| 隐藏 | 非 admin / supervisor 角色 | `display: none`（不占位） |

### 1.4 动画效果

- 按钮进入 Loading 态：`transition: all 150ms ease`，图标淡入 + 文字替换，无跳变
- Toast 出现：`slide-in-right 300ms ease`（从右侧滑入）
- Toast 消失：3 秒后反向播放 `slide-in-right` 淡出

### 1.5 错误反馈

| 场景 | 提示方式 | 文案 |
|------|---------|------|
| 网络超时（> 15s） | Toast error | 「导出超时，请检查网络连接后重试」 |
| 服务端 500 | Toast error | 「服务异常，请稍后重试或联系系统管理员」 |
| 权限不足（403） | 不展示按钮，接口层不可达 | 无需提示 |
| 空数据导出 | Toast warning | 「当前筛选条件下无数据，无法导出」 |

## 2. R-02 供应商绩效对比

### 2.1 入口交互

供应商列表每行左侧提供 checkbox（数据列第一列），全选 checkbox 支持三态（全选 / 半选 / 全不选）。

```
选中 ≥ 1 家
  ↓
表格上方出现「选中工具栏」（animation: fade-in 150ms ease）
工具栏内容：
  [已选 N 家供应商]  [提示文字]  [取消选中] [绩效对比（conditional）]
```

绩效对比按钮的激活条件：

| 已选数量 | 按钮状态 | 工具栏背景 |
|---------|---------|-----------|
| 0 | 工具栏隐藏 | — |
| 1 | 显示工具栏，按钮 disabled | 蓝色浅底（`color-primary-50`），提示「请选择 2-5 家供应商进行对比」 |
| 2-5 | 按钮激活（`btn--primary`） | 蓝色浅底，提示「再选 X 家可达上限」 |
| 6+ | 按钮 disabled | 橙色警告底（`color-warning-50`），提示「超出上限，最多同时对比 5 家」 |

### 2.2 对比页面打开方式

点击「绩效对比」按钮 → 打开全屏弹框（`position: fixed; inset: 0`，`z-index: var(--z-modal)`）。

动画：`animation: fade-in 200ms ease`（整体淡入），弹框内容区 `animation: slide-up 250ms ease`（向上滑入）。

全屏弹框不使用路由跳转，保留原始列表页状态，关闭后用户可继续操作列表。

### 2.3 对比页面结构与交互

```
弹框顶栏
  ├─ 标题「供应商绩效对比」+ 已选数量
  ├─ 时间范围切换器（近3月 / 近6月 / 近12月 / 全部）
  └─ 关闭按钮（✕）

供应商标签行
  └─ 每个标签可点击 ✕ 移除（最少保留 2 家，移除后立即刷新图表）

空数据告警条（conditional）
  └─ 某供应商无历史数据时出现，橙色警告样式

对比图表区（两列网格）
  ├─ 左：雷达图（SVG，6 个维度多边形）
  └─ 右：月度采购金额折线图（SVG）

核心指标对比表格
  └─ 最优值绿底，最差值红底，无数据显示「—」
```

### 2.4 时间范围切换

点击时间范围按钮：
1. 按钮切换为 `time-range-btn--active`（蓝色实底），150ms 过渡
2. 图表区显示骨架屏 shimmer（加载中）
3. 接口返回后，图表以 `fade-in 200ms` 重新渲染
4. 若切换后某供应商该时间段无数据，图表对应多边形收缩至 0 值，动画过渡 `300ms ease`

### 2.5 供应商标签移除

- 点击标签上的 ✕：对应供应商从对比中移除，标签消失（`fade-in reverse 150ms`）
- 若移除后仅剩 1 家，弹出 Toast 警告：「至少需要 2 家供应商进行对比，无法继续移除」，阻止操作
- 图表和表格立即重新渲染（剩余供应商数据）

### 2.6 雷达图视觉规范

- 6 个维度从正上方顺时针排列：准时交货率、价格竞争力、质量合格率、响应速度、订单完成率、服务评分
- 5 层同心六边形网格，从中心到外圈分别代表 20 / 40 / 60 / 80 / 100 分
- 每家供应商使用独立色彩（最多 5 个）：蓝 `#3B82F6` / 橙 `#F97316` / 绿 `#22C55E` / 紫 `#8B5CF6` / 红 `#F43F5E`
- 无数据维度值显示为 0（顶点收缩至中心），图例旁标注「*数据不足」
- SVG 多边形 `fill-opacity: 0.15`，`stroke-width: 2`

### 2.7 折线图视觉规范

- X 轴：近 6 个月的月份标签
- Y 轴：采购金额（万元），自动计算最大值并向上取整
- 无数据供应商：使用虚线（`stroke-dasharray: 6,4`）+ `opacity: 0.5` 展示，视觉降权
- 悬浮 tooltip（生产实现时）：显示当月具体金额，跟随鼠标位置

### 2.8 核心指标对比表高亮规则

```
同一行中：
  最优值（最高分 / 最短周期）→ 绿底 + 绿色文字 + 加粗
  最差值                     → 红底 + 红色文字
  无数据                     → 「—」灰色
  全部无数据                  → 整行所有列显示「—」
```

注意：若同一行所有供应商分数相同，则均不高亮（无最优/最差之分）。

### 2.9 关闭对比弹框

- 点击 ✕ 按钮：弹框以 `fade-in 200ms ease reverse` 淡出关闭
- 按 Esc 键：同上（`document.addEventListener('keydown', handleEsc)`）
- 关闭后：选中工具栏仍保持已选状态（保留用户的供应商勾选，不清空）

## 3. R-03 采购价格批量导入

### 3.1 入口

位于采购价格管理页顶部操作区，使用 `btn--primary` 样式，始终可见（有导入权限的角色：admin / 采购主管）。

### 3.2 向导整体流程

```
点击「批量导入」
  ↓
打开导入向导（Modal 或全页面，推荐全页面路由 /purchase/prices/import）
  ↓
Step 1：下载模板 → Step 2：上传文件 → Step 3：预览校验 → Step 4：确认导入
```

向导顶部步骤条：
- 已完成步骤：绿色实心圆 + 对勾图标
- 当前步骤：蓝色实心圆 + `pulse-glow` 动画（2s 循环）
- 未到达步骤：灰色空心圆 + 文字 disabled 色
- 步骤间连接线：已完成为绿色，未到达为灰色，`transition: background 300ms ease`

### 3.3 Step 1 — 下载模板

| 用户操作 | 系统响应 |
|---------|---------|
| 点击「立即下载」或文件卡片 | 触发 `GET /purchase-prices/template` 下载，文件名 `采购价格导入模板_v1.xlsx` |
| 点击「下一步」 | 无需验证是否下载，直接进入 Step 2 |

### 3.4 Step 2 — 上传文件

#### 3.4.1 拖拽区交互

| 用户行为 | 视觉变化 |
|---------|---------|
| 鼠标拖入文件 | 拖拽区变色（蓝底 + 蓝色虚线边框 + `scale(1.01)`），`dragover` 事件触发 |
| 鼠标离开（未释放） | 恢复默认样式，`dragleave` 事件触发 |
| 释放文件（正确格式） | 进入文件已选状态，显示文件名 + 大小 + 行数预估 |
| 释放文件（错误格式） | Toast error：「仅支持 .xlsx / .xls 格式，请重新选择」 |
| 点击上传区 | 触发 `<input type="file" accept=".xlsx,.xls">` |

动画：拖拽区 `transition: all 200ms ease`；文件图标有持续的 `drop-bounce 2.5s ease infinite` 动画（吸引用户注意）。

#### 3.4.2 文件校验（前端即时）

| 校验项 | 不通过时的提示 |
|-------|-------------|
| 文件格式 | Toast error 「仅支持 .xlsx / .xls 格式」 |
| 文件大小 > 10MB | Toast error 「文件大小超限（最大 10 MB），请压缩后重新上传」 |
| 读取失败 | Toast error 「文件损坏或格式异常，无法读取」 |

文件超行数（> 5000 行）：前端上传后由后端解析返回错误，前端展示错误横幅（红底 inline 错误）和「返回重新选择」按钮，「下一步」按钮保持 disabled。

#### 3.4.3 大文件进度条（> 1000 行）

后端解析期间展示进度：
- 前端每 2 秒轮询 `GET /purchase-prices/import/{taskId}/status`
- 进度条 `width` 用 CSS `transition: width 300ms ease` 平滑更新
- 进度文字：「已解析 X / Y 行，请稍候」
- 允许终止解析（`POST /purchase-prices/import/{taskId}/cancel`），终止后 Toast info：「解析已终止」

### 3.5 Step 3 — 预览校验

#### 3.5.1 页面顶部统计汇总

```
[共 X 行]  [正确 X 行]  [错误 X 行]  [重复追加 X 行]  [价格偏高 X 行]
```

每个 chip 使用语义色区分：总行（灰）/ 正确（绿）/ 错误（红）/ 重复（蓝）/ 警告（橙）。

#### 3.5.2 行级高亮规则

| 行状态 | 背景色 | 来源规则 |
|-------|-------|---------|
| 正确行 | 白色（默认） | 通过所有校验 |
| 错误行 | `color-error-50`（浅红） | 任意必填字段为空 / 格式错误 / 编码不存在 |
| 重复追加行 | `#FEFCE8`（黄色-50） | （SKU + 供应商 + 报价日期）三元组已存在 |
| 价格偏高告警行 | `color-warning-50`（浅橙） | 单价超历史最高价 3 倍 |

同一行可同时满足「重复」且「价格偏高」，此时使用**错误色优先**原则（错误行 > 重复追加行 > 价格偏高行）。

#### 3.5.3 单元格级高亮

错误所在的具体单元格内容使用 `cell--error-field` 包裹（红底红字），方便用户定位具体错误字段。价格偏高单元格使用 `cell--warn-field`（橙底）。

#### 3.5.4 处理方式选择

单选组，默认选中「跳过错误行，仅导入正确行」。

- 选择「取消导入，修正后重新上传」时，「下一步」按钮文案变为「确认放弃」，点击后关闭向导回到价格管理页，Toast info：「已取消本次导入」
- 选择「跳过错误行」时，「下一步」按钮文案为「下一步：确认导入 →」

#### 3.5.5 预览表格展示规则

- 默认展示前 100 行
- 超出部分折叠，底部显示「共 X 行，已显示前 100 行」+ 「展开全部」按钮
- 点击「展开全部」：加载所有行（虚拟滚动或分页加载，避免 DOM 性能问题）
- 「下载错误明细」：下载包含所有错误行原始数据 + 错误原因的 Excel 文件

### 3.6 Step 4 — 确认导入

#### 3.6.1 确认前展示

在导入执行前，展示本次操作概况（只读汇总卡片）：

```
将导入 X 行 | 跳过错误 X 行 | 追加重复 X 行
```

「确认导入」按钮点击后不可撤销，无需二次确认弹窗（过于繁琐，Step 3 已提供放弃入口）。

#### 3.6.2 导入执行中

- 按钮变为 `btn--loading`（「导入中…」+ spinner），上一步按钮 disabled
- 进度条：同解析进度，轮询 `GET /purchase-prices/import/{taskId}/status`
- 提示文字：「已写入 X / Y 条，请勿关闭页面」
- 提供「终止导入」按钮（危险操作，使用 `btn--danger`），点击后二次确认弹窗（`Modal` 组件，`size=sm`）：「确认终止本次导入？已写入的数据不会撤回。」

#### 3.6.3 导入完成结果页

布局：成功动画 + 三项统计卡片 + 价格异常告警区 + 失败明细下载区

**成功动画**：SVG 圆形描边 + 对勾路径描边动画，`stroke-dashoffset` 从 230 → 0（圆形 300ms），再从 60 → 0（对勾 500ms，延迟 300ms 开始）。

**价格异常告警**（conditional，仅有价格偏高行时显示）：
- 橙色警告卡片
- 表格列出告警行：行号、SKU、供应商、导入单价、历史最高价、偏高倍数
- 标注为告警，不阻断导入，已成功写入

**意外失败行**（conditional）：
- 红色信息卡 + 「下载失败明细」按钮
- 失败明细 Excel 包含：原始行数据 + 失败原因

**结果页操作按钮**：
- 「再次导入」：重置向导回 Step 1
- 「查看导入数据」：跳转采购价格管理页，并高亮本次导入的数据批次

## 4. 设计规范补充（Design Tokens 对应）

### 4.1 色彩使用规则

| 场景 | Token | 说明 |
|------|-------|------|
| 主操作按钮 | `--color-primary-600` / `700` hover | 导出、下一步、确认导入 |
| 成功反馈 | `--color-success-500` ~ `700` | 导出完成 Toast / 结果成功数 |
| 错误状态 | `--color-error-50` / `500` / `700` | 错误行背景 / 错误单元格 / 文字 |
| 警告状态 | `--color-warning-50` / `500` / `700` | 价格偏高行 / 超选告警 |
| 重复追加行 | `#FEFCE8`（Tailwind yellow-50） | 区别于警告和错误 |
| 禁用状态 | `--color-gray-100` / `--text-disabled` | 禁用按钮背景 / 文字 |
| 选中状态 | `--color-primary-50` | 表格选中行 / 工具栏背景 |
| 对比颜色 1-5 | 蓝 / 橙 / 绿 / 紫 / 红 | 雷达图多边形 / 折线图 |

### 4.2 间距规范

- 向导内容区外边距：`padding: var(--space-8) var(--space-12)`
- 步骤卡片 padding：`var(--space-8)`
- 表格单元格：`padding: var(--space-3) var(--space-4)`
- 操作按钮间距：`gap: var(--space-3)`

### 4.3 字体规范

- 数字（统计、金额、进度）：`font-family: var(--font-family-number)` + `font-variant-numeric: tabular-nums`
- 编码（SKU 编码、供应商编码）：`font-family: var(--font-family-mono)`（预览表格中）
- 标题区：`font-weight: 700`，`font-size: var(--text-h4)`
- 说明文字：`font-size: var(--text-body-s)`，`color: var(--text-secondary)`

### 4.4 圆角规范

| 组件 | 圆角 Token |
|------|-----------|
| 步骤卡片 | `--radius-xl` (16px) |
| 结果统计卡 | `--radius-xl` |
| 普通 Card | `--radius-lg` (12px) |
| 按钮 | `--radius-md` (8px) |
| Badge / Chip | `--radius-full` (9999px) |
| 表格错误单元格 | `--radius-sm` (4px) |

## 5. 组件状态一览表

### 5.1 导出按钮（ExportButton）

| 状态名 | class 标记 | 描述 |
|-------|-----------|------|
| default | `btn--secondary` | 可点击，灰色边框 |
| loading | `btn--secondary btn--loading` + `disabled` | 旋转图标 + 「导出中…」 |
| disabled-empty | `btn--secondary` + `disabled` | 无数据时，鼠标 not-allowed |
| hidden | `display: none` | 无权限角色不渲染 |

### 5.2 绩效对比按钮（CompareButton）

| 状态名 | 触发条件 | class |
|-------|---------|-------|
| hidden | 未选中任何供应商 | `display: none` |
| disabled-single | 选中 1 家 | `btn--primary disabled` |
| active | 选中 2-5 家 | `btn--primary` |
| disabled-over | 选中 6+ 家 | `btn--primary disabled` |

### 5.3 向导步骤（WizardStep）

| 状态名 | class |
|-------|-------|
| pending（未到达） | `wizard-step--pending` |
| active（当前） | `wizard-step--active` + `pulse-glow` 动画 |
| done（已完成） | `wizard-step--done` + 对勾图标 |

### 5.4 上传区域（UploadZone）

| 状态名 | class |
|-------|-------|
| default | `.upload-zone` |
| dragover | `.upload-zone--dragover` |
| file-selected（正常） | 替换为 `file-selected-card` |

## 6. 无障碍要求（WCAG 2.1 AA）

### 6.1 键盘导航

- 全屏对比弹框打开后：焦点自动移入弹框容器（`tabIndex={-1}` + `focus()`）
- 按 Esc 关闭弹框，焦点回到触发按钮
- 向导步骤底部操作按钮组：Tab 顺序为「上一步 → 取消 → 下一步」
- 上传区 `role="button"` + `tabIndex="0"` + 支持 Enter/Space 触发文件选择

### 6.2 ARIA

### 6.3 颜色对比

## 8. 边界情况处理汇总

### 8.1 R-02 导出

| 边界场景 | 处理方式 |
|---------|---------|
| 导出中用户刷新页面 | 无特殊处理，用户刷新后重新操作 |
| 导出文件名含特殊字符 | 日期固定格式 YYYY-MM-DD，不含特殊字符 |
| 导出数据量极大（>10000条） | Toast info：「导出数据量较大，请耐心等待」，超时 30s 显示超时 Toast |
| 非中文浏览器 | Content-Disposition 中文文件名编码问题由后端处理（RFC 5987） |

### 8.2 R-02 绩效对比

| 边界场景 | 处理方式 |
|---------|---------|
| 所有对比供应商均无数据 | 雷达图所有顶点在中心，折线图平行于 X 轴（0 值），顶部信息告警条 |
| 移除供应商后剩 1 家 | Toast warning + 阻止移除操作 |
| 时间范围切换后无数据 | 图表清空 + inline 空态提示「该时间范围内无采购记录」 |
| 网络请求失败 | 图表区显示错误状态 + 「重新加载」按钮 |

### 8.3 R-03 批量导入

| 边界场景 | 处理方式 |
|---------|---------|
| 文件 > 10MB | 前端即时拦截，不发起上传请求 |
| 文件行数 > 5000 | 上传后后端返回错误，前端展示错误横幅 |
| 全部行均为错误行 | Step 3 中「下一步」按钮变为 disabled；提示「全部为错误行，请修正后重新上传」 |
| 用户在 Step 3 选择「取消导入」 | 直接关闭向导，不执行任何导入 |
| 导入过程中网络断开 | 进度停止更新，展示 Toast error：「网络连接中断，请检查网络后刷新页面查看导入状态」 |
| 二次打开向导（上次未完成） | 检测 localStorage 中的 taskId，提示「上次导入任务未完成，是否继续？」或放弃 |
| Excel 文件编码异常 | 后端解析失败返回 400，前端展示：「文件编码异常，请另存为 UTF-8 格式的 .xlsx 文件」 |
| 所有行均为重复追加行 | 正常展示，顶部 stat-chip 显示「重复追加 X 行」，不阻断导入 |
| 价格历史无记录时 | 不触发偏高告警（无法比对）|
