## 一、R-07 销售客户管理

### 1.1 页面结构

```
销售管理
└── 客户管理（CustomerPage）
    ├── 统计卡片行（全部 / 活跃 / VIP）
    ├── 筛选工具栏
    ├── 客户列表表格
    ├── 新增/编辑客户弹框（Modal）
    └── 客户详情侧边栏（Drawer）
```

### 1.2 用户行为与交互流程

#### 客户列表

| 行为 | 触发动作 | 系统响应 |
|---|---|---|
| 搜索框输入 | 防抖 300ms 后触发 API | 表格内容实时刷新，无结果时展示空态 |
| 等级筛选下拉切换 | 即时触发筛选 | 表格重新加载，当前页重置为第 1 页 |
| 状态筛选切换 | 同上 | 同上 |
| 点击"新增客户" | 打开新增弹框 | Modal 动画进入（scale 0.95 → 1），背景模糊锁定 |
| 点击"详情" | 打开详情侧边栏 | Drawer 从右侧滑入（translateX 100% → 0） |
| 点击"编辑" | 打开编辑弹框 | Modal 进入，字段预填当前客户数据 |
| 点击"停用" | 内联确认或 Confirm 弹窗 | 确认后行透明度变 0.7，状态标签切换为"停用" |
| 点击"启用" | 直接调用 API | 状态恢复"启用"，Toast 提示 |
| 分页切换 | 加载对应页数据 | 表格 skeleton 加载态 → 内容 |

#### 新增/编辑弹框

| 行为 | 交互规范 |
|---|---|
| 表单提交 | 客户名称、等级必填；提交时全量校验，错误字段边框变红+错误提示文案显示在字段下方 |
| 客户编码 | 只读，系统自动生成，格式 `CUS-YYYY-NNN`，新增时显示占位文案"自动生成" |
| 信用额度 | 数值输入，最小值 0；不填写视为无信用额度限制 |
| 账期天数 | 整数输入，最小值 0，空值视为即时付款 |
| 保存成功 | Modal 关闭，表格新增行闪烁高亮（背景色渐变）0.8s，Toast 提示"客户信息已保存" |
| 保存失败 | Modal 保持打开，Footer 区显示错误提示，Button 恢复可点击状态 |
| 按 Esc / 点击遮罩 | Modal 关闭，未保存的内容不保留（若表单已修改，需弹出"是否放弃修改"二次确认） |

#### 客户详情侧边栏

| 区域 | 交互规范 |
|---|---|
| 基本信息卡片 | 只读展示，不可内联编辑，修改须点击底部"编辑客户"按钮 |
| 联系人列表 | 支持新增联系人（+ 按钮）、编辑单条（铅笔图标）、删除单条（垃圾桶图标）；删除时二次确认提示 |
| 主要联系人设置 | 每个客户只能有一名主要联系人；将其他联系人设为主要联系人时，原主要联系人标记自动取消 |
| 历史订单列表 | 仅展示最近 5 条，点击"查看全部"跳转至销售订单页并自动筛选该客户 |
| 关闭 Drawer | 点击右上角 ✕、点击遮罩层、按 Esc 均可关闭 |

#### 删除客户保护逻辑

- 若客户有历史订单：弹出警告弹框，告知"该客户有 X 条历史订单，无法删除，可将状态设为停用"，仅提供"停用"和"取消"选项，不提供删除按钮。
- 若客户无历史订单：弹出确认删除弹框，用户须输入客户名称进行二次确认（防误操作）。

### 1.3 等级标签色彩规范

| 等级 | 背景色 | 文字色 | 边框色 | 语义 |
|---|---|---|---|---|
| VIP | `#FFF0F9` | `#9D174D` | `#F9A8D4` | 粉红系，高价值客户 |
| A 级 | `#EFF6FF` | `#1D4ED8` | `#BFDBFE` | 蓝色系，重要客户 |
| B 级 | `#F0FDF4` | `#166534` | `#BBF7D0` | 绿色系，普通优质客户 |
| C 级 | `#F8FAFC` | `#475569` | `#CBD5E1` | 灰色系，一般客户 |

### 1.4 状态标签规范

| 状态 | 样式类 | 语义 |
|---|---|---|
| 启用 | `status-badge--active` | 绿色点 + 绿色文字 |
| 停用 | `status-badge--inactive` | 灰色点 + 灰色文字，行透明度 0.7 |

### 1.5 Loading / 错误 / 空态

- **加载中**：表格行替换为 5 行 skeleton（灰色矩形动画），宽度随机 60-90%
- **加载失败**：表格区域展示错误提示卡片，包含错误描述和"重试"按钮
- **空态**：搜索无结果时展示中央图标 + "未找到匹配的客户" + "清除筛选条件"操作

## 二、R-08 销售订单（含紧急插单）

### 2.1 页面结构

```
销售管理
└── 销售订单（OrderPage）
    ├── Admin 待审批横幅（仅 admin 角色可见）
    ├── 统计卡片行（全部 / 待确认 / 生产中 / 已完成）
    ├── 筛选工具栏
    ├── 订单列表表格（紧急行高亮）
    ├── 新增订单弹框（全屏 Modal）
    ├── 紧急插单审批弹框（Modal）
    └── 订单详情侧边栏（Drawer）
```

### 2.2 订单状态流转

```
提交订单
  │
  ├── [普通订单] ──→ 待确认 ──→ 已确认 ──→ 生产中 ──→ 已完成
  │                                                    └──→ 关闭（异常终止）
  │
  └── [紧急插单] ──→ 待排产确认 ──→（admin 审批）
                          │
                          ├── 通过 ──→ 已确认 ──→ 生产中 ──→ 已完成
                          └── 驳回 ──→ 已关闭（并通知销售人员）
```

### 2.3 用户行为与交互流程

#### 订单列表

| 行为 | 交互规范 |
|---|---|
| 紧急订单行 | 浅红背景（`#FEF2F2`），紧急标签闪烁动画（opacity 1 → 0.3，周期 1.2s），醒目提示 |
| 点击统计卡片 | 切换当前选中状态（蓝色边框高亮），自动筛选该状态的订单列表 |
| 日期范围筛选 | 两个 date input，结束日期不可早于开始日期（前端校验） |
| 点击"新增订单" | 全屏 Modal 打开（max-width: 900px，max-height: 95vh） |
| 点击"审批" | 打开审批 Modal，仅 admin 角色可见此按钮 |

#### 新增订单弹框

| 行为 | 交互规范 |
|---|---|
| 客户字段 | 带搜索功能的下拉，支持模糊搜索客户名称或编码，至少输入 2 个字符后开始搜索 |
| 订单号 | 只读，系统自动生成 `SO-YYYY-NNN` |
| 紧急插单开关 | 默认关闭（普通优先级）；开启后开关区域背景变红（视觉强提示）；开启后实时查询产能，展示影响提示横幅 |
| 产能影响横幅 | 显示：最早可排产日期、预计完工日期、受影响的在产工单列表（最多展示 5 条，超出折叠） |
| 明细行 - 产品下拉 | 仅显示成品（FG）类 SKU；选中后自动带入默认单价（来自最新报价） |
| 明细行 - 数量/单价 | 变更后小计列自动重新计算；底部合计金额同步更新 |
| 删除明细行 | 至少保留 1 行，仅剩 1 行时"删除"按钮 disabled |
| 提交订单 | 校验：客户必选、至少 1 条明细、交付日期必填；校验通过后提交；若为紧急插单，提交后状态为"待排产确认"，普通订单状态为"待确认" |
| 保存草稿 | 保存当前填写内容，不做必填校验，状态为"草稿" |

#### 紧急插单审批弹框（仅 admin 可见）

| 行为 | 交互规范 |
|---|---|
| 弹框内容 | 订单摘要（6 个字段）+ 影响评估横幅 |
| 确认接单 | 调用 `POST /orders/:id/confirm-urgent`，成功后 Modal 关闭，订单状态变为"已确认"，触发生产工单创建，Toast 提示 |
| 点击"驳回" | 展开驳回原因输入框（textarea，必填），按钮文案变为"确认驳回"；再次点击"取消驳回"收起 |
| 确认驳回 | 驳回原因未填写时弹出 Toast 提示错误；填写后调用驳回接口，订单状态变为"已关闭"，系统推送站内通知给原提交销售人员 |
| Admin 横幅 | 列表页顶部显示"待审批数量"横幅，数量徽章实时更新（审批完一条减 1），降至 0 时横幅自动隐藏 |

#### 订单详情侧边栏

| 区域 | 交互规范 |
|---|---|
| 状态时间线 | 纵向时间线，已完成节点绿色实心圆+勾，当前节点蓝色实心圆+动态脉冲，未来节点灰色空心圆，透明度 40% |
| 操作按钮区（动态显示） | 根据当前状态和用户角色，底部按钮动态渲染：待确认→"确认订单"；已确认→"开始排产"（仅 admin/supervisor）；生产中→"标记发货"；已发货→"确认完成"；紧急待审批→"审批此订单"（仅 admin） |
| 紧急订单标记 | 详情侧边栏标题行固定显示紧急标签和红色交付日期，保持强视觉提示 |

### 2.4 权限矩阵（BD-003）

| 功能点 | 销售人员（sales） | 车间主管（supervisor） | 老板（admin） |
|---|---|---|---|
| 查看订单列表 | 仅自己提交的 | 全部 | 全部 |
| 新增订单 | 允许 | 不允许 | 允许 |
| 勾选紧急标记 | 允许 | 不允许 | 允许 |
| 查看产能影响评估 | 允许（只读） | 允许（只读） | 允许 |
| 待审批横幅 | 不可见 | 不可见 | 可见 |
| 审批按钮（表格操作列） | 不渲染 | 不渲染 | 渲染 |
| 确认接单（审批弹框） | 不允许 | 不允许 | 允许 |
| 确认驳回 | 不允许 | 不允许 | 允许 |

**前端实现要求**：权限控制必须在组件渲染层实现（`v-if` / 条件渲染），不得仅依赖路由守卫或按钮 `disabled`。后端接口须二次校验角色（`POST /orders/:id/confirm-urgent` 返回 403 若非 admin）。

### 2.5 紧急订单特殊交互规范

1. **视觉强提示层级**：紧急行背景色 > 紧急标签闪烁 > 交付日期红色 > 待审批横幅，四层视觉强度递进。

2. **产能影响提示触发时机**：
   - 勾选紧急开关后，立即发起 `GET /orders/capacity-check?delivery_date=xxx` 请求
   - 请求期间按钮显示 loading spinner，横幅区域显示 skeleton
   - 若后端无可靠数据（排产数据不足），显示"当前排产数据不足，无法精确评估影响，请联系车间主管确认"

3. **交付日期合法性校验**：
   - 交付日期不可早于下单日期 + 3 天（最短生产周期保护）
   - 若紧急订单要求交付日期早于系统评估的最早完工日期，日期字段边框变红，并展示差距天数

4. **审批通知**：
   - 插单提交后，所有 admin 角色用户收到站内通知（消息中心红点 +1）
   - 小程序端同步推送模板消息
   - 审批通过/驳回后，原提交销售人员收到站内通知，说明结果（驳回时附带驳回原因）

5. **排产影响列表展示规则**：
   - 按影响天数降序排列
   - 最多展示 5 条，超出显示"还有 X 个订单受影响，[展开查看]"
   - 影响天数 = 0 的订单不展示（无影响）

### 2.6 订单状态标签色彩规范

| 状态 | 背景色 | 文字色 | 点色 | 语义 |
|---|---|---|---|---|
| 待确认 | `#FEF3C7` | `#B45309` | `#F59E0B` | 警告黄，需要操作 |
| 待排产确认 | `#FEE2E2` | `#B91C1C` | `#EF4444` | 错误红，紧急需处理 |
| 已确认 | `#E0F2FE` | `#0284C7` | `#0EA5E9` | 信息蓝，正常进行中 |
| 生产中 | `#DBEAFE` | `#1D4ED8` | `#3B82F6` | 主色蓝，积极进展 |
| 已完成 | `#DCFCE7` | `#15803D` | `#22C55E` | 成功绿，已达成 |
| 已关闭 | `#F1F5F9` | `#64748B` | `#94A3B8` | 中性灰，非激活 |

### 2.7 Loading / 错误 / 空态

- **加载中**：表格行 skeleton，统计卡片数字区 shimmer 动画
- **加载失败**：表格区域错误提示卡片（红色边框），含错误码、描述、重试按钮
- **空态（无订单）**：插画图标 + "暂无订单" + "新增第一条销售订单"操作按钮
- **空态（筛选无结果）**：插画图标 + "未找到符合条件的订单" + "清除筛选条件"按钮

## 三、通用动画与反馈规范

### 3.1 动画参数

| 场景 | 动画类型 | 时长 | 曲线 |
|---|---|---|---|
| Modal 进入 | scale 0.95→1 + opacity 0→1 | 200ms | `cubic-bezier(0.34,1.56,0.64,1)` spring |
| Modal 退出 | scale 1→0.95 + opacity 1→0 | 150ms | `ease` |
| Drawer 进入 | translateX 100%→0 | 300ms | `ease` |
| Drawer 退出 | translateX 0→100% | 250ms | `ease` |
| Toast 进入 | translateY 4px→0 + opacity 0→1 | 200ms | `ease` |
| 表格行 hover | background-color 渐变 | 150ms | `ease` |
| 开关切换 | thumb translateX | 200ms | spring |
| 紧急点闪烁 | opacity 1→0.3 | 周期 1.2s | `ease-in-out` |

### 3.2 Toast 反馈规范

| 类型 | 背景色 | 前缀 | 自动消失 |
|---|---|---|---|
| 成功 | `#16A34A` | ✓ | 3秒 |
| 错误 | `#DC2626` | ✕ | 5秒（含操作按钮） |
| 默认（信息） | `#1E293B` | 无 | 3秒 |

### 3.3 表单校验反馈

- 必填字段未填写：字段边框变红（`#EF4444`），字段下方出现红色 `0.75rem` 错误提示文字，动画 `fade-in` 200ms
- 格式校验失败：同上，文案描述具体格式要求
- 实时校验：在 `blur` 事件触发后进行（不在 `input` 时实时报错，避免干扰输入）
- 批量校验：点击提交时对所有必填字段同时校验，第一个错误字段自动 `scroll into view` 并聚焦

### 3.4 无障碍规范（WCAG 2.1 AA）

- 所有交互元素具备 `aria-label` 或可见文字标签
- 弹框开启时焦点自动移至弹框内第一个可聚焦元素，关闭后焦点返回触发按钮
- 颜色对比度：文字与背景 ≥ 4.5:1（小文字），≥ 3:1（大文字/图标）
- 状态变化不能仅依赖颜色区分，需配合文字或图标
- 紧急闪烁动画须遵循 `prefers-reduced-motion`：媒体查询检测到用户偏好减少动画时，改为静态红色标签
