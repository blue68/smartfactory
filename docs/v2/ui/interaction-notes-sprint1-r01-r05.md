## 2. R-01 类目管理 — 交互流程

### 2.1 进入页面

1. 用户点击侧边栏「主数据 > 类目管理」
2. 页面展示骨架屏（左侧列表 + 右侧表格同步加载）
3. 骨架屏消失，左侧一级类目列表渲染，默认不选中任何类目
4. 右侧面板显示空提示：「← 请从左侧选择一个一级类目以查看子类目」

**加载时长预期**：< 500ms（AC-001-06 非功能要求）
**若加载失败**：右侧面板显示 inline error alert，文案「列表加载失败，请刷新重试」，提供「刷新」按钮

### 2.2 选中一级类目

1. 用户点击左侧列表中的任意类目行
2. 当前行背景切换至 `--color-primary-50`，左边框变为 `3px solid --color-primary-500`，过渡时长 `150ms`
3. 右侧面板标题更新为所选类目名称，副标题显示子类目数量
4. 右侧表格加载该类目的二级子类目列表（若为空，展示空状态）

**触发区域**：整行可点击（含拖拽 Handle 之外的所有区域）
**Hover 状态**：行背景 `--color-gray-50`，同时显示操作按钮组（编辑/删除），隐藏元数据 Badge

### 2.3 新增一级类目

1. 点击左侧卡片头部「+ 新增」按钮
2. Modal 弹出（max-width 560px），焦点自动聚焦到「类目名称」输入框
3. 实时字数统计随输入更新（`0/20`）
4. 点击「确认新增」：
   - Loading 态：按钮禁用 + 显示 spinner + 文案「处理中...」
   - 成功：Modal 关闭，左侧列表追加新条目（动画 `fade-in` 200ms），Toast 成功提示
   - 名称重复：Modal 不关闭，输入框 border 变为 `--color-error-500`，输入框下方出现行内错误「该类目名称已存在」

### 2.4 编辑一级类目

1. Hover 类目行时，显示操作按钮组；点击「✏️」按钮
2. 若是系统预置类目（标有「预置」Badge）：允许修改名称和备注，不允许修改编码
3. 若是自定义类目：名称、备注、排序权重均可修改
4. 弹框回显当前值（类目编码为只读展示）
5. 提交成功后：Toast 提示「修改成功，关联 SKU 展示名称已自动同步更新」

### 2.5 拖拽排序

1. 鼠标 Hover 拖拽 Handle（`⠿`）时光标变为 `grab`
2. 按住拖拽时光标变为 `grabbing`，当前行透明度降至 50%
3. 拖拽目标位置显示 `2px solid --color-primary-500` 水平指示线
4. 松手后列表重排，后端异步保存排序（不阻塞 UI）
5. 若保存失败：列表回滚至原排序，Toast 警告提示

### 2.6 删除一级类目 — 系统预置

- 删除按钮对系统预置类目显示 `disabled` 状态（opacity 55%，cursor not-allowed）
- Tooltip 文案：「系统预置类目不可删除，仅允许修改名称」

### 2.7 删除一级类目 — 自定义（无子类目 + 无关联 SKU）

1. 点击删除按钮
2. 弹出警告 Modal（max-width 400px）
3. 文案：「确认删除「{类目名称}」？该类目下无子类目及关联 SKU，删除后不可恢复。」
4. 操作按钮：「取消」（ghost）+ 「确认删除」（danger）
5. 确认后：Modal 关闭，左侧列表移除该条目，Toast 成功

### 2.8 删除一级类目 — 有子类目 / 有关联 SKU（级联删除）

1. 点击删除按钮，**系统先查询关联数据**（显示 spinner，约 300ms）
2. 查询完成后弹出危险 Modal（max-width 560px）
3. Modal 内展示醒目的红色 Alert 区域，包含：
   - 操作摘要：「将同时删除 X 个子类目，Y 条 SKU 将变为未分类」
   - 数据统计卡片（子类目数、受影响 SKU 数）
4. 需用户在输入框中**重新键入类目名称**以确认（防误触）
5. 输入匹配前，「确认级联删除」按钮保持 disabled
6. 匹配后按钮激活，用户确认后执行删除
7. Toast：「删除成功，X 条 SKU 已变为未分类」

### 2.9 新增 / 编辑二级子类目

1. 右侧面板点击「+ 新增子类目」，弹出 Modal（同一级新增样式规范）
2. 「所属一级类目」字段只读展示当前选中的一级类目名称
3. 同一父类目下名称唯一验证（跨父类目可重名）
4. 支持**行内编辑**（inline edit）：
   - 点击行内「编辑」按钮，该行的名称单元格切换为 input，行背景变为 `--color-primary-50`
   - 出现「✓ 保存」「✕ 取消」两个微型按钮
   - Enter 键快捷保存，Esc 键取消
5. 提交成功后行返回普通展示态

### 2.10 操作日志抽屉

1. 点击页面右上角「📋 操作日志」按钮
2. 页面右侧滑出 Drawer（width 480px，`slide-in-right` 动画 300ms）
3. 主内容区不受影响（Drawer 浮层在上方，背景蒙层半透明）
4. Drawer 内展示时间线（垂直 Timeline 组件）：
   - 新增操作：绿色圆点
   - 修改操作：蓝色圆点 + diff 展示（删除线旧值 → 新值）
   - 删除操作：红色圆点
5. 支持按操作类型和日期范围筛选
6. 点击蒙层或「×」关闭

## 4. R-01 — 动画与过渡效果

| 动画 | 触发时机 | 实现方式 | 时长 |
|---|---|---|---|
| 页面加载骨架屏 | 进入页面时 | `shimmer` keyframe CSS，`.skeleton` class | 1.5s infinite |

| 新条目追加到列表 | 新增成功 | `fade-in` keyframe（opacity 0→1） | 200ms |

## 5. R-01 — 错误反馈机制

### 5.1 表单行内验证

| 错误场景 | 触发时机 | 展示位置 | 文案 |
|---|---|---|---|
| 类目名称为空 | 点击确认/失焦 | 输入框下方行内 | 「类目名称不可为空」 |
| 类目名称重复 | API 返回 4xx | 输入框下方行内 | 「该类目名称已存在」 |
| 名称超出长度 | 实时（oninput） | 输入框下方字数统计变红 | 「已超出 20 字限制」 |
| 同一父类目下子类目名称重复 | API 返回 4xx | 输入框下方行内 | 「该子类目名称在当前父类目下已存在」 |

**错误状态视觉规范**：
- 输入框 border 变为 `--color-error-500`
- 聚焦状态焦点环变为 `--color-error-100`（3px box-shadow）
- 错误文案颜色 `--color-error-600`，前缀 `⚠` 图标

### 5.2 操作失败 Toast

| 场景 | Toast 类型 | 标题 | 描述 |
|---|---|---|---|
| 新增/修改提交服务端 500 | error | 「操作失败」 | 「服务器异常，请稍后重试」 |
| 删除时关联数量查询失败 | error | 「无法执行删除」 | 「获取关联数据失败，请刷新后重试」 |
| 拖拽排序保存失败 | warning | 「排序保存失败」 | 「排序已恢复，请重新操作」 |
| 页面初始加载失败 | 内联 Alert（不用 Toast）| — | 「列表加载失败，请刷新重试」 |

### 5.3 删除防误触机制

- 普通删除（无关联）：单次弹框确认即可执行
- 级联删除（有子类目或有关联 SKU）：必须在输入框中重新键入类目名称后方可激活确认按钮
- 系统预置类目：删除按钮 disabled，无弹框

## 6. R-01 — 权限相关 UI 差异

| 角色 | 左侧新增按钮 | 编辑按钮 | 删除按钮 | 右侧新增子类目 | 行内编辑 | 操作日志 |
|---|---|---|---|---|---|---|
| admin（管理员） | 显示 | 显示 | 显示（预置类目 disabled） | 显示 | 可操作 | 可查看 |
| supervisor（车间主管） | 隐藏 | 隐藏 | 隐藏 | 隐藏 | 不可操作（只读） | 可查看 |
| 其他角色（工人/采购等） | 路由 403 拦截，页面不可访问 | — | — | — | — | — |

**非 admin 访问逻辑**：
1. 前端：路由守卫检查角色，非 admin 访问 `/system/categories` 路由时重定向至 403 页
2. 接口层：即便绕过前端，PATCH / DELETE 接口在后端校验角色，返回 403 + 错误 code
3. 页面内如有读权限（如 supervisor）：操作按钮组完全不渲染（不依赖禁用），页面顶部显示权限提示 banner

## 7. R-05 工序配置 — 交互流程

### 7.1 进入页面

1. 通过侧边栏「生产管理 > 工序配置」进入，也可从 SKU 主数据页顶部导航进入
2. 页面加载工序列表（骨架屏过渡）
3. **Admin 角色**：表格含「极限工时」「熟练工单价」「学徒工单价」三列，列头有 🔒 图标提示
4. **非 Admin 角色**：工价两列不渲染（前端主动控制，不依赖后端是否返回数据），显示权限提示 banner

### 7.2 极限工时内联编辑

1. 鼠标 Hover 「未配置」单元格时，出现「+ 配置」文字按钮（text 样式）
2. 点击「+ 配置」或点击已有数值旁的编辑小图标，当前单元格切换为 input
3. **input 样式**：宽 6rem，右对齐，聚焦状态焦点环蓝色
4. 输入后单位标签「h/件」紧跟在右侧（非 input 内部，位于外部 flex 布局）
5. 操作按钮：`✓`（保存，primary 小按钮）、`✕`（取消，ghost 小按钮）
6. 同时支持 `Enter` 键保存、`Escape` 键取消
7. 保存成功：单元格恢复为普通展示，Toast 成功提示
8. 若输入值 <= 0 或非数字：输入框边框变红，行内提示「请输入正数，最多 2 位小数」

### 7.3 配置工价（弹框）

1. 每行最右侧操作列提供「配置工价」按钮（text 样式，仅 admin 可见）
2. 点击弹出「工序工价配置」Modal（560px 宽）
3. 弹框内包含：
   - 工序基本信息只读展示区（名称、编码）
   - 极限工时输入（与列表内联编辑共享逻辑，弹框内统一操作更方便）
   - 极限工时超时预警阈值实时预览（极限工时 × 1.2）
   - 熟练工单价 / 学徒工单价输入（带 ¥ 前缀 addon）
   - 工价差值实时计算提示
4. 保存后 Toast 提示工价修改已写入审计日志

## 8. R-05 — 工价配置弹框交互流程

### 8.1 表单字段行为

| 字段 | 类型 | 验证规则 | 实时反馈 |
|---|---|---|---|
| 极限工时 | number input | 正数，最多 2 位小数 | 实时更新「超出阈值」提示（× 1.2） |
| 熟练工单价 | number input，带 ¥ 前缀 | 正数，最多 2 位小数 | 实时更新工价差计算 |
| 学徒工单价 | number input，带 ¥ 前缀 | 正数，最多 2 位小数 | 实时更新工价差计算 |

**工价差实时计算公式**：
`差值 = 熟练工单价 - 学徒工单价`
`差值百分比 = 差值 / 学徒工单价 × 100%`
展示在输入区下方 info 提示条中。

### 8.2 极限工时与超时预警关联

- 极限工时输入后，页面实时显示：「⚠ 极限工时 × 1.2 = {计算值} h/件（超出触发预警）」
- 该值为完工上报时超时判断的基准，需让用户清晰理解

### 8.3 权限校验

- 弹框内工价字段**仅对 admin 渲染**，车间主管打开同一弹框时工价区块不显示（需前端控制，不依赖 API）
- 保存时前端额外检查角色，非 admin 直接拦截不发起 API 请求

## 9. R-05 — 完工上报弹框交互流程

### 9.1 进入弹框

1. 工人在任务详情页点击「完工上报」按钮
2. 弹出 Modal（560px），顶部展示只读任务信息卡（任务编号、工序、工人等级、极限工时）
3. 焦点自动落在「完成件数」输入框

### 9.2 实时工资核算预览

- 用户**每次修改「完成件数」**时，底部预览区实时更新：
  - 已配置单价：`预计工资：¥ {件数 × 单价} 元`，格式化为 2 位小数
  - 未配置单价：显示「该工序暂未配置工价，本次完工不计工资」，背景变为灰色
- 预览区域底色：有单价时 `--color-primary-50`，无单价时 `--color-gray-50`

### 9.3 超时预警逻辑

**触发条件**：
```
人均工时 = 实际工时 ÷ 完成件数
超时 = 人均工时 > 极限工时 × 1.2
```

**触发时机**：`完成件数` 或 `实际工时` 任意一个变化时，实时检测
**视觉**：表单中间插入红色 Alert 区块（`fade-in` 动画），显示具体超时百分比
**不阻断提交**：超时预警为提示性，用户仍可点击「提交完工」
**未配置极限工时**：永远不触发超时预警，Alert 不出现

### 9.4 提交流程

1. 点击「提交完工」：
   - 校验完成件数（正整数 ≥ 1）、实际工时（正数）
   - 有错误：输入框红色边框 + 行内错误文案，不提交
   - 通过校验：按钮 loading 态
2. API 返回 200：Modal 关闭，Toast 成功（含工资金额 or 「未配置单价」提示）
3. API 返回 409（乐观锁冲突）：Modal 保持打开，错误 Toast「该任务已被更新，请刷新后重新查看」
4. 请求超时 > 10s：按钮恢复可点击，Toast 警告「网络请求超时，请检查网络连接后重试」

### 9.5 工人等级未配置（BD-002 核心场景）

- **在进入弹框前**：若当前工人的等级字段为 null，「完工上报」按钮本身 disabled，tooltip 文案「工人等级未配置，请联系管理员设置后再上报」
- **若绕过前端直接 API**：后端返回 400，弹框内展示红色 Alert 错误

## 10. R-05 — 工资核算报表交互流程

### 10.1 进入页面

1. 通过侧边栏「生产管理 > 工资核算」进入（仅 admin 和 supervisor 可见此菜单项）
2. 筛选区默认选中「当月 1 日至今」，自动加载报表数据
3. 默认展示**表格视图**

### 10.2 筛选操作

1. 时间范围为**必选**，清空时「查询」按钮 disabled，提示「请选择时间范围」
2. 工人/工序筛选为可选多选（使用原生 `<select multiple>` 或组件内 checkbox 下拉）
3. 等级筛选单选（全部 / 仅熟练工 / 仅学徒工）
4. 点击「查询」时按钮 loading 态，表格区域加载骨架屏
5. 返回结果后骨架屏替换为真实数据
6. 「重置」按钮：恢复所有筛选到默认值，**不自动触发查询**，需用户再次点击「查询」

### 10.3 表格视图

- 数据按工人分组，同一工人的多行记录 `rowspan` 合并工人姓名列
- 工资金额列右对齐，使用等宽数字（`font-family: --font-family-number`，`font-variant-numeric: tabular-nums`）
- 未配置单价的行：单价列显示「—」（警告色 `--color-warning-600`），小计列同样显示「—」
- 汇总行（tfoot）：
  - 背景 `--color-gray-50`，上边框 2px
  - 合计金额字体加大（`--text-body-l`），颜色 `--color-primary-600`
  - 若有未计入记录：注释文案红色 `*` + 灰色说明文字

### 10.4 图表视图

- 柱状图展示各工人工资总额，熟练工（蓝色 `--color-primary-500`）和学徒工（橙色 `--color-warning-500`）堆叠
- 图例在图表下方，清晰标注颜色对应关系
- 若某工人无学徒工记录，学徒工柱高度为 0（柱最小高度 `5px` 保持可见）
- 数据 tooltip（鼠标 hover 柱时显示）：`工人名：¥ xxx 元`

### 10.5 导出 Excel

1. 点击「导出 Excel」按钮：按钮 loading 态（spinner），不跳转页面
2. 导出范围：当前筛选条件下的全部数据（不受当前视图/分页限制）
3. 文件命名：`工资核算_{开始日期}_{结束日期}.xlsx`
4. 导出完成：浏览器自动下载，按钮恢复正常
5. 若筛选结果为空：Toast 提示「当前筛选条件下无数据，无法导出」

## 15. 无障碍规范（WCAG 2.1 AA）

### 15.1 键盘导航

| 操作 | 键盘支持 |
|---|---|
| 一级类目列表选中 | `Tab` 移动焦点，`Enter`/`Space` 选中 |
| 内联编辑 | `Enter` 保存，`Esc` 取消 |
| Modal 关闭 | `Esc` 键关闭 |
| Drawer 关闭 | `Esc` 键关闭 |
| 表单提交 | `Enter` 在最后一个输入框触发提交 |
| 弹框内焦点陷阱 | Modal 打开时焦点限制在 Modal 内，循环 Tab（标准 focus trap 实现） |

### 15.2 屏幕阅读器

所有原型 HTML 已包含以下 ARIA 属性，前端实现须保留：

- 列表：`role="listbox"` + `aria-label` + `aria-selected`
- Modal：`role="dialog"` + `aria-modal="true"` + `aria-labelledby="modal-title-id"`
- Drawer：`role="complementary"` + `aria-label`
- Toast 区域：`aria-live="polite"` + `aria-atomic="false"`（错误 Toast 用 `aria-live="assertive"`）
- 错误提示：`role="alert"` + `aria-live="assertive"`
- 加载状态：`aria-busy="true"` on 按钮
- 数字统计：`aria-label` 说明数字含义（如 `aria-label="完成件数：120 件"`）
- 只读字段：`aria-readonly="true"` 或使用 `<div>` 替代 `<input disabled>` 确保可被聚焦读取

### 15.3 色彩对比度

以下组合均满足 WCAG AA 4.5:1 最低要求：

| 前景色 | 背景色 | 用途 |
|---|---|---|
| `#FFFFFF` | `#2563EB` (primary-600) | 主按钮文字 |
| `#1E293B` | `#FFFFFF` | 普通文本 |
| `#64748B` | `#FFFFFF` | 次要文本（对比度约 4.6:1） |
| `#B91C1C` (error-700) | `#FEF2F2` (error-50) | 错误文本 |
| `#1D4ED8` (primary-700) | `#EFF6FF` (primary-50) | 选中状态文本 |

**注意**：`--text-secondary (#64748B)` 在白色背景下对比度约 4.6:1，接近临界值，不应用于小字（< 12px）场景；如需 12px 以下文字，改用 `--text-primary (#1E293B)`。

### 15.4 响应式与移动端

| 断点 | 布局变化 |
|---|---|
| `≥ 1100px` | 标准两栏（左 280px + 右 flex:1） |
| `900px~1100px` | 左栏收窄至 240px |
| `< 900px` | 侧边栏隐藏，类目管理切换为单栏（先显示一级列表，选中后切换显示二级） |
| `< 640px` | Toast 宽度占满屏幕宽度减 padding |
| `< 768px` | 报表表格隐藏部分非关键列（如「创建时间」） |

**Touch 设备**：Hover 类操作（如 Hover 显示操作按钮）改为点击展开/折叠操作区域（使用 toggle 态替代 hover 态）
