## 1. 页面结构总览

```
主导航
└── 我的任务（工人可见）/ 生产任务（主管、管理员可见）
    └── TaskPage
        ├── 筛选器区域（FilterBar）
        ├── 统计卡片区域（StatCards）
        ├── 任务列表（TaskList）
        │   └── 任务行（TaskRow）
        │       ├── 优先级标签
        │       ├── 状态标签
        │       ├── 超时预警角标（条件显示）
        │       └── 操作按钮组（权限差异）
        └── 侧边抽屉（TaskDetailDrawer）
            ├── 基本信息区
            ├── BOM 快照区（折叠）
            ├── 时间线区（操作日志）
            └── 操作按钮区（权限差异）
                ├── 开始生产（待开始状态）
                ├── 完工上报（进行中状态）
                ├── 异常上报（进行中状态）
                └── 主管处置按钮（主管/管理员，异常待处理状态）
```

## 3. 任务状态流转

### 3.1 任务主状态机

```
                    工人点击"开始生产"
  [待开始] ─────────────────────────────→ [进行中]
    |                                         |
    |                                   ┌─────┴──────┐
    |                           工人完工上报    工人异常上报
    |                                   |              |
    |                                   v              v
    |                              [已完成]     [异常待处理]
    |                                              |
    |                                    主管"标记已处理"  主管"挂起任务"
    |                                              |              |
    |                                              v              v
    |                                          [进行中]       [已挂起]
    |
    |（任务被主管重新分配）
    └────────────────────────────────────────→ [待开始]
```

### 3.2 状态标签颜色规范

| 状态 | 背景色 Token | 文字色 Token | 说明 |
|---|---|---|---|
| 待开始 | `--color-status-pending-bg` `#F5F5F5` | `--color-status-pending-text` `#8C8C8C` | 灰色 |
| 进行中 | `--color-status-active-bg` `#E6F4FF` | `--color-status-active-text` `#1677FF` | 蓝色 |
| 已完成 | `--color-status-done-bg` `#F6FFED` | `--color-status-done-text` `#52C41A` | 绿色 |
| 异常待处理 | `--color-status-error-bg` `#FFF1F0` | `--color-status-error-text` `#FF4D4F` | 红色 |
| 已挂起 | `--color-status-suspend-bg` `#FFF7E6` | `--color-status-suspend-text` `#FA8C16` | 橙色 |

状态切换时，标签组件执行颜色渐变过渡：`transition: background-color 0.3s ease, color 0.3s ease`

### 3.3 异常处置状态机

```
[异常上报提交]
        |
        v
[异常待处理] ─── 主管"标记已处理"（填写处理说明）──→ [进行中]
              └── 主管"挂起任务"（填写挂起原因） ──→ [已挂起]
```

## 4. 任务看板页交互

### 4.1 页面加载序列

1. 路由进入页面，立即渲染骨架屏（Skeleton）：
   - 统计卡片区：4 个卡片骨架（灰色矩形脉冲动画）
   - 任务列表区：6 行列表骨架（高度 56px，间距 8px，脉冲动画）
2. 接口响应完成后：
   - 骨架屏消失（`opacity: 0`，`100ms`）
   - 真实内容 `fadeIn`（`opacity: 0 → 1`，`200ms ease`）
   - 统计数字执行 `countUp` 动画（从 `0` 计至实际值，`600ms ease-out`，步长根据数值自适应）

### 4.2 筛选器交互

- 筛选器任意条件变化（状态多选、日期范围、工序类型）立即触发列表重新请求，无需点击"查询"按钮
- 条件变化到请求发出之间有 `300ms debounce` 延迟（防止快速操作触发大量请求）
- 请求发出期间，列表区域叠加半透明遮罩（`rgba(255,255,255,0.6)`）+ 顶部进度条 loading 指示（线性进度条，左→右滑动，颜色 `#1677FF`）

### 4.3 任务行交互

**默认状态**：白色背景，行高 `3.5rem`（56px），底部 1px 分割线 `#F0F0F0`

**Hover 状态**：背景色过渡至 `#FAFAFA`（`transition: background-color 0.15s ease`），鼠标指针 `cursor: pointer`

**Active（点击中）状态**：背景色 `#F0F7FF`，瞬间切换（`0ms`）

**超时预警标记行**：左侧红色 `3px` 实线边框 `#FF4D4F`，行背景 `#FFF1F0`

**紧急优先级标签**：`--color-priority-urgent-bg` `#FFF1F0`，文字 `#FF4D4F`，字体 `0.75rem` bold，圆角 `0.25rem`

### 4.4 统计卡片交互

- 四张卡片分别对应：待开始、进行中、已完成、异常待处理
- 点击卡片等同于点击对应状态筛选器（列表自动过滤为该状态）
- 被选中的卡片：底部 `3px` 蓝色实线 + 背景 `#E6F4FF`（`transition: 0.2s ease`）
- Hover 状态：`box-shadow: 0 2px 8px rgba(0,0,0,0.12)` 上浮效果（`transition: 0.2s ease`）

## 5. 任务详情侧边抽屉交互

### 5.1 展开与关闭

- **展开触发**：点击任务列表任意行
- **展开动画**：抽屉从右侧 `translateX(100%)` 滑入至 `translateX(0)`，同时左侧主内容区域压缩宽度（列表宽度由 `100%` 收缩至 `calc(100% - 480px)`）
  - 动画参数：`transition: transform 0.25s cubic-bezier(0.4, 0, 0.2, 1), width 0.25s cubic-bezier(0.4, 0, 0.2, 1)`
  - 抽屉宽度：PC 端 `480px`，平板 `375px`，移动端全屏
- **关闭触发**：点击抽屉右上角 "×" 按钮，或按 `Esc` 键，或点击抽屉外空白区域
- **关闭动画**：反向滑出，`translateX(0) → translateX(100%)`，`0.2s ease`

### 5.2 BOM 快照区

- 默认折叠，展示折叠标题："所需原材料（X 种）"，右侧展开图标（↓）
- 点击标题展开，展开动画：`max-height: 0 → max-height: 500px`，`transition: 0.3s ease`
- BOM 表格只读，禁止编辑，字段：原材料名称、规格型号、所需数量、单位
- 若 BOM 无数据，显示空态文案："该产品暂无 BOM 数据"

### 5.3 时间线（操作日志）

- 每条记录包含：操作类型图标、操作人姓名、操作时间（`YYYY-MM-DD HH:mm`）、操作内容描述
- 操作类型图标颜色：
  - 创建任务：`#1677FF` 蓝色圆点
  - 开始生产：`#52C41A` 绿色圆点
  - 完工上报：`#52C41A` 绿色勾图标
  - 异常上报：`#FF4D4F` 红色警告图标
  - 异常处置：`#FA8C16` 橙色扳手图标
  - 挂起任务：`#8C8C8C` 灰色暂停图标
- 初始展示最新 5 条，超出显示"查看全部记录"链接，点击展开全部（`max-height` 动画展开）

### 5.4 操作按钮区

按钮区固定在抽屉底部（`position: sticky; bottom: 0`），背景白色，顶部 `1px` 分割线

**待开始状态**：
- 主按钮："开始生产"（Primary，`#1677FF`，宽度 100%）

**进行中状态**：
- 主按钮："完工上报"（Primary，`#1677FF`，宽度 `calc(50% - 4px)`）
- 次要按钮："异常上报"（Default，描边样式，宽度 `calc(50% - 4px)`）
- 两按钮并排，间距 `8px`

**异常待处理状态（工人视角）**：
- 只读提示文案："已上报异常，等待主管处理"（灰色，居中，不显示操作按钮）

**异常待处理状态（主管/管理员视角）**：
- 主按钮："标记已处理"（成功色，`#52C41A`，宽度 `calc(50% - 4px)`）
- 次要按钮："挂起任务"（警告色描边，`#FA8C16`，宽度 `calc(50% - 4px)`）

## 6. 完工上报弹窗交互

### 6.1 弹窗展开

- **触发**：点击"完工上报"按钮
- **动画**：弹窗从屏幕中央以 `scaleY(0.8) + opacity(0)` 展开至 `scaleY(1) + opacity(1)`
  - 参数：`transition: transform 0.2s cubic-bezier(0.4, 0, 0.2, 1), opacity 0.2s ease`（即 fadeIn 0.2s）
- 背景遮罩：`rgba(0,0,0,0.45)`，`fadeIn 0.2s`
- 弹窗宽度：PC 端 `480px`，移动端 `100vw`（底部弹起式 bottom sheet）

### 6.2 表单字段交互

**完成件数**（必填，正整数）：
- 默认 placeholder："请输入完成件数"
- 输入后立即触发工资预览实时计算（`onChange` 事件，无 debounce）
- 输入非正整数时，边框变红 `#FF4D4F`，下方显示行内错误文案："请输入大于 0 的整数"（`fadeIn 0.15s`）

**实际工时（小时）**（必填，正数，最多 1 位小数）：
- 输入后触发超时预警检测（前端本地计算，公式：人均工时 = 实际工时 ÷ 完成件数）
- 超时预警检测（≥ 极限工时 × 1.2）时，在输入框下方显示黄色预警横幅：
  ```
  ⚠ 人均工时 {X} h/件，超出极限工时 {Y}%，提交将记录超时预警
  ```
  - 颜色：背景 `#FFFBE6`，边框 `#FFD666`，文字 `#AD6800`
  - 动画：`slideDown 0.2s ease`

**备注**（选填）：
- 字数计数器显示在右下角（`{已输入字数}/200`），超出 200 字时计数器变红

### 6.3 工资预览区

工资预览区位于表单下方，灰色背景卡片样式：

**有工价时**：
```
预计工资
工人等级：熟练工
计算公式：{完成件数} 件 × ¥{单价}/件
= ¥{工资金额}（加粗，字号 1.25rem，颜色 #1677FF）
```
- `完成件数` 字段值变化时，金额数字执行 `countUp` 滚动动画（`300ms ease-out`）

**无工价时**（工序未配置该等级单价）：
```
该工序暂未配置工价，本次完工不计工资
```
- 灰色文字 `#8C8C8C`，斜体

**工人等级未配置时**：
- 表单置灰（`disabled` 状态），顶部显示错误横幅（红色）：
  ```
  ✕ 您的工人等级尚未配置，无法提交完工上报。请联系管理员设置工人等级。
  ```
- "提交"按钮 `disabled`

### 6.4 超时预警确认流程

当检测到超时预警（人均工时 ≥ 极限工时 × 1.2）时，用户点击"提交"按钮后：

1. 不直接提交，先弹出二次确认对话框（嵌套弹窗，层级 `z-index` 高于当前弹窗）
2. 确认对话框文案：
   ```
   超时预警确认

   本次完工人均工时为 {X} 小时/件，超出极限工时 {Y}%。
   系统将在完工记录中标记"超时预警"。

   [取消]  [确认提交]
   ```
3. 点击"确认提交"：关闭确认对话框 → 继续提交流程（见 6.5）
4. 点击"取消"：关闭确认对话框，回到表单，用户可修改数据

### 6.5 提交流程

1. 用户点击"提交"按钮（或超时预警确认后）
2. 表单执行前端校验（必填项、数值范围）
   - 校验失败：对应字段边框变红，显示行内错误文案，焦点跳至第一个错误字段
3. 校验通过：
   - 按钮进入 loading 状态（文字替换为 spinner + "提交中..."，`disabled`）
   - 发送请求（携带 `version` 乐观锁字段）
4. 请求成功（HTTP 200）：
   - 弹窗执行关闭动画（`scaleY(1) + opacity(1) → scaleY(0.8) + opacity(0)`，`0.15s`）
   - 背景遮罩消失（`0.15s`）
   - 任务详情抽屉内状态标签更新为"已完成"（颜色渐变 `0.3s`）
   - 列表对应行状态同步更新
   - 显示成功 Toast（3 秒后自动消失）：
     ```
     完工上报已提交，本次工资：¥XX.XX 元
     ```
     （若未配置工价则 Toast 文案："完工上报已提交"，不显示金额）

### 6.6 错误处理

| 错误场景 | HTTP 状态码 | 前端响应 |
|---|---|---|
| 并发冲突（乐观锁） | 409 | 弹窗内顶部展示错误横幅（红色）："任务状态已变更，请刷新重试"。按钮恢复可点击，"提交"按钮文字变为"刷新并重试" |
| 请求超时（> 10s） | - | Toast 错误提示："网络请求超时，请检查网络连接后重试"。按钮恢复可点击状态（不自动重试） |
| 服务端未知错误 | 500 | Toast 错误提示："提交失败，请稍后重试"。按钮恢复可点击 |
| 工人等级未配置 | 400 | 弹窗内顶部错误横幅（红色）："工人等级未配置，无法核算工资，请联系管理员"。按钮 disabled |

## 7. 异常上报弹窗交互

### 7.1 弹窗展开

- **触发**：点击"异常上报"按钮
- **动画**：同完工上报弹窗（fadeIn 0.2s），弹窗宽度 PC 端 `520px`（因包含图片上传区域宽度略大）
- 移动端：底部弹起式 bottom sheet，占屏幕高度 85%，可上下滑动内容区

### 7.2 异常类型选择

- 单选按钮组，4 个选项竖排：
  - 设备故障
  - 物料缺失
  - 质量问题
  - 其他
- 每个选项右侧显示帮助图标（"?"，颜色 `#8C8C8C`）：
  - PC 端：鼠标 Hover 触发 Tooltip 显示（fadeIn 0.15s，白色气泡，阴影，圆角）
  - 移动端：点击触发底部弹出说明卡片（bottom sheet，100% 宽，内含类型说明文字）
- 选中的选项：背景色 `#E6F4FF`，左侧 `3px` 蓝色实线，单选圆圈填充 `#1677FF`
- 未选择提交时：选项区域显示红色虚线边框，下方行内错误文案："请选择异常类型"

**帮助文案内容**：

| 类型 | 说明 |
|---|---|
| 设备故障 | 生产设备无法正常运行，需要维修或更换 |
| 物料缺失 | 所需原材料或半成品不足，无法继续生产 |
| 质量问题 | 发现产品质量不符合要求，存在缺陷 |
| 其他 | 以上类型之外的异常情况，请在描述中详细说明 |

### 7.3 异常描述文本域

- placeholder："请详细描述异常情况（至少 10 个字）"
- 字数计数器：右下角 `{已输入字数}/500`
- 不足 10 字时：提交触发校验，边框变红，行内错误："异常描述至少需要 10 个字"
- 超出 500 字：禁止继续输入，计数器变红

### 7.4 是否影响生产进度

- 单选：是 / 否
- 默认无选中
- 提交时未选择：行内错误提示

### 7.5 附件图片上传

- 上传区域：虚线边框卡片，居中文案 "点击或拖拽上传图片（JPG/PNG，单张最大 5MB，最多 3 张）"
- **拖拽进入时**：边框颜色从 `#D9D9D9` 变为 `#1677FF`，背景 `#E6F4FF`（`transition: 0.15s ease`）
- **上传中状态**（每张图片独立）：
  - 图片缩略图区域显示半透明遮罩 + 线性进度条（底部）
  - 进度条颜色 `#1677FF`，高度 `3px`
  - 百分比数字显示在缩略图中央（白色，字号 `0.875rem`）
- **上传成功**：
  - 进度条消失，缩略图显示正常
  - 右上角出现 "×" 删除按钮（红色圆圈，白色叉，`hover` 时背景色加深）
- **上传失败**：
  - 缩略图区域红色遮罩 + "!" 图标
  - 图片下方显示独立错误文案（不影响其他图片）：
    - 超出 5MB："图片超出大小限制（最大 5MB）"
    - 非 JPG/PNG："仅支持 JPG/PNG 格式"
    - 接口失败："上传失败，请重试"（含"重试"链接）
- 已上传 3 张后：上传区域变为灰色 disabled 样式，文案变为"已达到最大上传数量（3 张）"

### 7.6 提交流程

1. 表单校验（异常类型、描述字数、影响生产进度），同完工上报弹窗
2. 图片上传（若有）需先全部完成（或失败）后方可提交
   - 若有图片仍在上传中，"提交"按钮 `disabled`，按钮下方提示："图片上传中，请稍候..."
3. 校验通过，发送请求（携带已上传图片的 URL 数组）
4. 请求成功：
   - 弹窗关闭（`fadeOut 0.15s`）
   - 任务状态标签更新为"异常待处理"（红色，颜色渐变 `0.3s`）
   - 成功 Toast（3 秒）："异常已上报，等待主管处理"
   - 导航栏铃铛图标上出现红点（通知提醒，针对主管端）

### 7.7 主管通知说明

- 异常提交后，主管端导航栏铃铛图标红点数量 +1
- 铃铛图标抖动动画（`shake` 动画，`0.5s`，仅触发一次）
- 铃铛下拉通知列表中新增一条通知：
  ```
  任务「{任务编号}」发生异常（{异常类型}），请及时处理
  {相对时间，如"刚刚"}  [查看详情 →]
  ```
- 点击"查看详情"：导航至对应任务详情页（TaskPage 并自动展开对应任务的侧边抽屉）

## 10. 动画与过渡效果规范

### 10.1 动画 Token 表

| Token 名称 | 值 | 用途 |
|---|---|---|
| `--motion-duration-fast` | `0.15s` | 微小反馈（按钮 Active，表单错误出现） |
| `--motion-duration-base` | `0.2s` | 弹窗出现，Toast 出现，筛选器展开 |
| `--motion-duration-slow` | `0.3s` | 状态标签颜色渐变，统计卡片选中 |
| `--motion-duration-drawer` | `0.25s` | 侧边抽屉滑入滑出 |
| `--motion-duration-count-up` | `0.6s` | 统计数字 countUp 动画 |
| `--motion-easing-standard` | `cubic-bezier(0.4, 0, 0.2, 1)` | 弹窗、抽屉入场 |
| `--motion-easing-decelerate` | `cubic-bezier(0, 0, 0.2, 1)` | 内容展开（BOM 折叠区） |
| `--motion-easing-ease-out` | `ease-out` | countUp 数字滚动 |

### 10.2 各组件动画明细

**弹窗（Modal）**：
- 入场：`transform: scaleY(0.8); opacity: 0` → `transform: scaleY(1); opacity: 1`
- 出场：反向，`0.15s`
- 背景遮罩：`opacity: 0 → 0.45`，`0.2s ease`

**侧边抽屉（Drawer）**：
- 入场：`transform: translateX(100%) → translateX(0)`，`0.25s cubic-bezier(0.4, 0, 0.2, 1)`
- 同步压缩列表宽度

**状态标签（StatusTag）**：
- `transition: background-color 0.3s ease, color 0.3s ease, border-color 0.3s ease`

**Toast 通知**：
- 入场：`transform: translateY(-20px); opacity: 0` → `transform: translateY(0); opacity: 1`，`0.2s ease`
- 自动消失：`opacity: 1 → 0`，`0.3s ease`，消失后 DOM 移除

**countUp 数字**：
- 使用 `requestAnimationFrame` 实现，`600ms ease-out`
- 从 `0` 到目标值，步长自适应目标值大小

**超时预警横幅 slideDown**：
- `max-height: 0 → max-height: 60px`，`0.2s ease`
- `overflow: hidden`

**铃铛抖动（shake）**：
```css
@keyframes bell-shake {
  0%, 100% { transform: rotate(0deg); }
  20%       { transform: rotate(-15deg); }
  40%       { transform: rotate(15deg); }
  60%       { transform: rotate(-10deg); }
  80%       { transform: rotate(10deg); }
}
/* duration: 0.5s, 仅播放一次 */
```

**骨架屏脉冲**：
```css
@keyframes skeleton-pulse {
  0%, 100% { opacity: 1; }
  50%       { opacity: 0.4; }
}
/* duration: 1.5s, infinite */
```

## 11. 错误与反馈机制

### 11.1 Toast 规范

| 类型 | 背景色 | 图标 | 显示时长 | 位置 |
|---|---|---|---|---|
| 成功 | `#F6FFED` 绿色 | `✓` | 3 秒 | 页面顶部居中 |
| 错误 | `#FFF1F0` 红色 | `✕` | 5 秒（错误给更长阅读时间） | 页面顶部居中 |
| 警告 | `#FFFBE6` 黄色 | `⚠` | 4 秒 | 页面顶部居中 |
| 信息 | `#E6F4FF` 蓝色 | `ℹ` | 3 秒 | 页面顶部居中 |

- 同时最多显示 3 条 Toast，超出时最早出现的 Toast 提前消失
- Toast 宽度：最小 `240px`，最大 `480px`，内容超长时自动换行

### 11.2 行内表单错误

- 错误文案位于输入框下方，字号 `0.75rem`，颜色 `#FF4D4F`
- 错误出现动画：`fadeIn 0.15s`
- 错误消失：用户输入修正后立即消失（`onChange` 实时校验）

### 11.3 弹窗顶部错误横幅

用于并发冲突（409）、权限错误等需要用户关注的系统级错误：
- 背景 `#FFF1F0`，左侧 `4px` 红色实线，图标 `✕`，文字 `#CF1322`
- 位于弹窗标题下方、表单内容上方
- 高度自适应内容，`padding: 12px 16px`

### 11.4 并发冲突（409）专项处理

1. Toast 错误提示（5 秒）："任务状态已变更，请刷新重试"
2. 弹窗内顶部同时显示错误横幅
3. "提交"按钮文字变更为"刷新并重试"，点击后：
   - 关闭弹窗
   - 刷新任务详情（重新获取最新 `version` 和状态）
   - 若状态已变更为非"进行中"，"完工上报"按钮自动隐藏

### 11.5 网络超时处理

- 超过 10 秒无响应时：
  - Toast 警告："网络请求超时，请检查网络连接后重试"
  - 弹窗按钮恢复可点击（不触发自动重试，由用户主动决定）
  - 超时后弹窗不自动关闭，用户数据不丢失（可直接重新点击"提交"）

## 12. 响应式行为说明

### 12.1 断点定义

| 断点名 | 范围 | 布局策略 |
|---|---|---|
| `xs` | 375px – 576px | 移动端单列，所有弹窗为底部 bottom sheet |
| `sm` | 576px – 768px | 移动端横屏，筛选器折叠为"筛选"按钮 |
| `md` | 768px – 1024px | 平板，抽屉宽度 375px，弹窗宽度 400px |
| `lg` | 1024px – 1440px | PC 标准，抽屉 480px，弹窗 480px–520px |
| `xl` | 1440px+ | PC 宽屏，任务列表最大宽度 1200px 居中 |

### 12.2 任务列表响应式

- `xs/sm`：列表列精简（仅显示：任务编号、工序、状态、优先级、操作），隐藏产品名称和计划时间列
- `md+`：展示全部列
- 操作按钮：`xs/sm` 下收折为三点（"..."）菜单；`md+` 直接显示文字按钮

### 12.3 侧边抽屉响应式

- `xs/sm`：抽屉全屏展示（宽度 100vw），顶部显示返回按钮（"← 返回任务列表"）
- `md`：抽屉宽度 375px，主内容区域不收缩（遮盖模式，背景遮罩）
- `lg+`：抽屉宽度 480px，主内容区域压缩（推拉模式，无遮罩）

### 12.4 弹窗响应式

- `xs/sm`：弹窗改为底部弹起 bottom sheet
  - 高度：85vh，可向上拖动至全屏（`max-height: 95vh`）
  - 顶部显示下拉手柄条（灰色圆角矩形，`40px × 4px`）
  - 入场动画：从底部 `translateY(100%) → translateY(0)`，`0.3s cubic-bezier(0.4, 0, 0.2, 1)`
- `md+`：居中弹窗，标准交互

### 12.5 最小宽度保证

- 所有页面最小可用宽度：`375px`（iPhone SE 基准）
- 关键操作按钮（完工上报、异常上报）最小触控区域：`44px × 44px`（WCAG 2.1 AA 标准）
- 文字最小字号：`0.875rem`（14px），移动端不缩放
