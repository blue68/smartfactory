## 一、已确认的 MOCK 数据问题

### GAP-R08-01 🔴 OrderPage.tsx 客户和产品使用硬编码 MOCK 数据

**位置**：`services/web/src/pages/sales/OrderPage.tsx` 第 22-34 行

`CUSTOMERS` 和 `PRODUCTS` 两个数组均为静态硬编码，包含 4 个虚构客户和 4 个虚构产品 SKU。

**具体问题**：
- 客户列表未调用 `useCustomerOptions()` 或任何客户查询接口，所有可选客户均为假数据
- 产品列表（`PRODUCTS`）未从 SKU 接口动态拉取，`skuId` 和 `bomId` 均为假值（1-4）
- 提交普通订单时 `customerId` 由客户数组下标 +1 计算（第 392 行），并非真实数据库 ID，极易错位
- 提交时 `skuId` 和 `bomId` 使用 MOCK 中的假值（如 `product?.skuId ?? 1`），必然导致后端关联错误

**影响**：新建订单会将错误的 customerId / skuId 写入数据库，功能无法正常使用。

**对比**：`SalesOrderListPage.tsx` 的 `CreateOrderModal` 已正确调用 `useCustomerOptions()`，两个页面存在双重实现且不一致。

## 二、表格列与设计稿对比

### GAP-R08-02 🟡 表格列顺序与设计稿不一致

**设计稿列顺序**：订单号 / 客户名称 / 下单日期 / 交付日期 / 金额（元）/ 状态 / 优先级 / 操作

**SalesOrderListPage.tsx 实际列顺序**：订单号 / 客户名称 / 订单日期 / 交期 / 金额 / **紧急**（单独列）/ 状态 / 操作

差异点：
- 设计稿"优先级"列合并了紧急标签（`<span class="urgent-tag">紧急</span>` + 非紧急显示 `—`），前端将其拆为独立的"紧急"列，命名也不一致（优先级 vs 紧急）
- 列标题"金额（元）"前端为"金额"，缺少单位说明

### GAP-R08-03 🟡 紧急行缺少浅红背景高亮

**交互说明 2.3**：紧急订单行需要浅红背景（`#FEF2F2`），紧急标签需要 opacity 闪烁动画（1→0.3，周期 1.2s）。

**实际**：`SalesOrderListPage.tsx` 表格行未对 `isUrgent === true` 的行应用特殊 className，无差异化背景色，无闪烁动画。`UrgentTag` 组件仅渲染纯文字标签，无闪烁效果。

## 三、统计卡片问题

### GAP-R08-04 🔴 统计卡片数据来源错误（当前页统计而非全量统计）

**位置**：`SalesOrderListPage.tsx` 第 824-829 行

```ts
pendingApproval: orders.filter((o) => o.status === 'pending_approval').length,
inProduction:    orders.filter((o) => o.status === 'in_production').length,
completed:       orders.filter((o) => o.status === 'completed').length,
```

`orders` 是当前分页的数据（最多 20 条），三个状态卡片的计数只反映当前页，不是全库数量。分页切换后数字会变化，严重影响数据可信度。

**正确做法**：后端应在分页响应中返回各状态汇总计数，或由前端发起独立的聚合统计请求。

### GAP-R08-05 🟡 统计卡片不可点击筛选

**交互说明 2.3**：点击统计卡片应切换当前选中状态（蓝色边框高亮），并自动筛选该状态的订单列表。

**实际**：`SummaryCards` 组件仅展示数字，无 onClick 事件，不可交互。

## 四、筛选栏问题

### GAP-R08-06 🟡 缺少日期范围筛选

**设计稿**：筛选栏包含开始日期和结束日期两个 date input，且结束日期不可早于开始日期（前端校验）。

**实际**：`SalesOrderListPage.tsx` 筛选栏仅有关键字搜索、状态筛选、紧急度筛选，完全缺少日期范围筛选。`SalesOrderListQuery` 类型中无日期范围字段，后端 `list()` 方法也无日期范围过滤参数。

### GAP-R08-07 🟡 搜索框缺少防抖

**交互说明 1.2（适用至 R-08）**：搜索框输入需防抖 300ms 后触发 API。

**实际**：`handleQueryChange` 在 `onChange` 时直接触发，无防抖处理，每次键盘输入都发起请求。

## 五、新建订单弹框（Modal）问题

### GAP-R08-08 🔴 新建订单存在两套实现，功能不一致

系统中存在两条并行的新建订单流程：
1. `SalesOrderListPage.tsx` 内嵌的 `CreateOrderModal`（使用 `salesOrder.ts` API，路径 `/api/sales-orders`）
2. 独立的 `OrderPage.tsx`（使用 `sales.ts` API，路径 `/api/sales/orders`）

两者调用不同的后端路径，传参结构不同，状态逻辑不同（OrderPage 有约束引擎卡片，ListPage 无），路由关系不明确，存在功能分裂风险。

### GAP-R08-09 🔴 新建弹框缺少"自动生成订单号"只读字段

**设计稿**：基本信息区包含"订单号"只读字段，显示系统自动生成的编号（`SO-YYYY-NNN（自动生成）`）。

**实际**：`CreateOrderModal` 和 `OrderPage.tsx` 均无此只读字段展示，用户无法在提交前知晓将被分配的订单号。

### GAP-R08-10 🔴 紧急插单开关开启后缺少产能影响横幅

**设计稿 + 交互说明 2.5.2**：勾选紧急开关后，需立即发起产能查询（`GET /orders/capacity-check?delivery_date=xxx`），展示包含最早可排产日期、预计完工日期、受影响在产工单列表的横幅（最多 5 条，超出折叠）。

**实际**：
- `CreateOrderModal`（ListPage 内）：勾选紧急后仅切换布尔值，无任何产能查询，无影响横幅
- `OrderPage.tsx`：紧急切换后显示静态文字提示（"紧急插单模式已启用"），无产能查询，无受影响工单列表
- 后端 `salesOrder.service.ts` 和 `sales.service.ts` 均无 `/capacity-check` 端点实现

### GAP-R08-11 🟡 明细行产品选择缺少 SKU 搜索下拉

**设计稿**：产品列使用 `<select>` 下拉且仅显示成品（FG 类）SKU，选中后自动带入默认单价。

**实际**：`CreateOrderModal` 使用两个独立文本输入框（产品编号 + 产品名称手工填写），没有从 SKU 接口获取选项，没有自动带入单价，没有仅显示成品类限制。

### GAP-R08-12 🟡 新建弹框缺少"保存草稿"按钮

**设计稿 footer**：包含"取消"、"保存草稿"、"提交订单"三个按钮。

**实际**：`CreateOrderModal` 仅有"取消"和"创建订单"两个按钮，缺少"保存草稿"功能（草稿状态后端已支持，`initialStatus = 'draft'` 对普通订单生效，但前端入口缺失）。

## 六、审批弹框问题

### GAP-R08-13 🔴 缺少独立的紧急插单审批弹框（Admin 专用）

**设计稿 + 交互说明 2.3**：列表页操作列中紧急待审批订单有"审批"按钮（仅 admin 可见），点击打开包含订单摘要 + 影响评估横幅 + 驳回原因区的审批弹框（`modal-approve`）。

**实际**：`SalesOrderListPage.tsx` 无独立审批弹框，审批操作被放在侧边抽屉（Drawer）的"状态操作"区域，通过"审批通过"和"拒绝"两个按钮触发，拒绝时弹出独立 `RejectModal`。

具体缺失：
- 列表操作列无"审批"快捷按钮（仅有"查看详情"）
- 无订单摘要展示（6 字段：客户、下单日期、要求交付、订单金额、产品明细、距今天数）
- 无产能影响评估横幅（"确认此紧急订单可能影响 X 个在产工单"）
- 驳回后订单状态后端实现为回退至 `draft`，但设计稿和交互说明要求驳回后状态为"已关闭"（`closed`），状态流转不一致

### GAP-R08-14 🔴 驳回后状态流转与设计稿不一致

**交互说明 2.2**：驳回后订单状态变为"已关闭"，并推送站内通知给原提交销售人员。

**后端实际**（`salesOrder.service.ts` 第 289-305 行）：驳回将订单状态改回 `draft`（草稿），而非 `closed`（已关闭）。

**影响**：被驳回的订单回到草稿状态，销售人员可以再次修改提交，但设计意图是驳回即终止此次插单申请（关闭）。

## 七、订单详情侧边栏（Drawer）问题

### GAP-R08-15 🔴 详情 Drawer 缺少状态时间线

**设计稿**：订单详情侧边栏包含完整的纵向状态时间线，展示：已完成节点（绿色实心圆+勾）、当前节点（蓝色实心圆+动态脉冲）、未来节点（灰色空心圆，透明度 40%），每个节点含时间戳 + 操作标签 + 描述文字。

**实际**：`OrderDetailDrawer` 仅展示基本信息和产品明细，无状态时间线，用户无法感知订单流转历史。

### GAP-R08-16 🟡 详情 Drawer 标题区缺少紧急标签和红色交付日期高亮

**交互说明 2.3**：详情侧边栏标题行固定显示紧急标签和红色交付日期（视觉强提示）。

**实际**：Drawer title 属性传入静态字符串 `"订单详情"`（第 638 行），无订单号、无紧急标签、无红色交付日期展示。订单号和状态 badge 仅在 body 区域以普通信息行展示。

### GAP-R08-17 🟡 "已确认"状态"标记发货"按钮权限过于宽松

**交互说明 2.3 操作按钮区**：已确认状态，"开始排产"仅 admin/supervisor 可见；设计意图是标记发货也应有角色限制。

**实际**：`confirmed` 状态下，"触发建工单"有 `isAdmin` 判断，但"标记发货"对所有角色均可见（第 558-566 行），权限控制不一致。

### GAP-R08-18 🔴 缺少 Admin 待审批横幅

**交互说明 2.3 + 2.4**：列表页顶部对 admin 角色显示"待审批数量"横幅，徽章数量实时更新（审批完一条减 1），降至 0 时横幅自动隐藏。

**实际**：`SalesOrderListPage.tsx` 无任何待审批横幅组件，`usePendingApprovals()` Hook 和对应 API（`GET /api/sales-orders/pending-approvals`）已在 `salesOrder.ts` 中实现但未在页面中使用。

## 九、状态流转与权限矩阵问题

### GAP-R08-19 🟡 状态标签命名与设计稿不一致

**设计稿状态**：待确认 / 待排产确认 / 已确认 / 生产中 / 已完成 / 已关闭

**实际前端状态标签**（`STATUS_LABELS`）：草稿 / 待审批 / 已确认 / 生产中 / 已发货 / 已完成 / 已关闭

差异点：
- 设计稿无"草稿"状态标签（设计中创建即提交，不暴露草稿状态给列表）
- 设计稿"待排产确认"对应后端 `pending_approval`，前端显示为"待审批"，语义有偏差
- 前端多出"已发货"（`shipped`）状态标签，设计稿无此中间状态展示

### GAP-R08-20 🟡 交互说明明确的"待确认"状态在后端不存在

**交互说明 2.2**：普通订单提交后状态为"待确认"（并非草稿）。

**后端实际**：普通订单创建后初始状态为 `draft`（第 175 行），不存在"待确认"独立状态，需 `confirm()` 操作才进入 `confirmed`。设计稿中"待确认"的含义与后端 `draft` + 业务逻辑不完全对应，需业务侧澄清。

### GAP-R08-21 🟡 前端权限矩阵实现不完整（BD-003）

**交互说明 2.4 权限矩阵**：销售人员查看订单列表仅能看自己提交的；新增订单按钮车间主管不可见；紧急标记车间主管不可勾选。

**实际**：
- `SalesOrderListPage.tsx` 无按角色过滤订单列表的逻辑（后端 `list()` 方法也无 `createdBy` 过滤参数）
- 新增订单按钮对所有角色均渲染（无 role 判断）
- 紧急插单开关对所有角色均可操作

## 十、API 路径双轨问题

### GAP-R08-22 🔴 两套 API 文件指向不同后端路径

| 文件 | 路径前缀 | 使用者 |
|---|---|---|
| `services/web/src/api/salesOrder.ts` | `/api/sales-orders` | `SalesOrderListPage.tsx` |
| `services/web/src/api/sales.ts` | `/api/sales/orders` | `OrderPage.tsx` |

两个文件中的 `useSalesOrderList`、`useCreateSalesOrder` 等 Hook 同名但接口路径不同，React Query key 也不同（`['sales-orders', ...]` vs `['sales', 'orders', ...]`），缓存无法共享，且若两个页面同时使用会导致数据不一致。

## 十一、缺失的后端接口

### GAP-R08-23 🔴 产能查询接口未实现

**交互说明 2.5.2** 要求：`GET /orders/capacity-check?delivery_date=xxx`

该接口在 `salesOrder.service.ts` 和 `sales.service.ts` 中均不存在，前端也无对应调用。

### GAP-R08-24 🟡 `estimatedDelivery` 字段后端始终返回 null

**OrderPage.tsx 第 610 行注释**：`createOrder` 返回的 `estimatedDelivery` 后端始终返回 null，导致"交期与产能预估"卡片的"预估最早交期"行永不展示，交期预警功能实际失效。

### GAP-R08-25 🟡 站内通知推送未实现

**交互说明 2.5.4**：插单提交后，所有 admin 收到站内通知（消息中心红点 +1），审批通过/驳回后原提交销售人员收到通知。

后端 `approve()` 和 `reject()` 方法中无任何通知推送逻辑。
