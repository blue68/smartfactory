## R-01: SKU类目管理

### 总体结论

设计稿采用**左右双面板布局**（左侧一级类目列表 + 右侧二级类目表格），前端实现采用的是**单表格扁平化树形列表**（一级/二级混合展示），属于布局结构层面的重大偏差。其余交互细节存在多处缺失。

### 差异清单

| 编号 | 严重度 | 设计稿要求 | 实际实现 | 修复方案(FE/BE) |
|------|--------|-----------|---------|----------------|
| R01-01 | 🔴重大缺失 | 左右双面板布局：左侧 280px 一级类目列表（可点击切换选中态），右侧展示当前选中一级类目下的二级子类目表格 | 实现为单一扁平表格，一级和二级行混合在同一张表中，无左侧面板、无选中态切换、无右侧独立二级表格 | FE：重构页面布局，拆分为左侧 `<ul>` 导航面板 + 右侧 `<table>` 详情面板，维护 `selectedCat1Id` 状态控制右侧渲染 |
| R01-02 | 🔴重大缺失 | 左侧一级类目列表：每项显示"子类目数量"徽章（如"8 子"），区分"预置"和"自定义"徽章，hover 时切换显示操作按钮（编辑/删除），非 hover 时显示 meta 信息 | 实现中一级行无子类目数量显示，无预置/自定义徽章区分，操作按钮始终可见而非 hover 触发 | FE：在一级行 meta 区域增加子类目计数徽章；通过 CSS hover 切换 meta/actions 的 display 状态 |
| R01-03 | 🔴重大缺失 | 右侧二级表格列结构：拖拽列（⠿）+ 类目名称 + 类目编码 + **关联 SKU 数** + **创建时间** + 操作 | 实现的表格列为：类目名称 + 编码 + 层级 + **排序值** + 状态 + 操作，缺少"关联 SKU 数"和"创建时间"两列，多出"层级"和"状态"两列 | FE：新增 skuCount、createdAt 列；BE：`getById`/列表接口需在二级类目字段中返回 skuCount 聚合值 |
| R01-04 | 🔴重大缺失 | 右侧面板顶部有当前选中一级类目的标题栏（含类目名称、预置徽章、"共 N 个子类目"统计）+ 独立的"+ 新增子类目"按钮 | 实现无此区域；新增入口为页头全局"+ 新增类目"按钮，不区分一级/二级上下文 | FE：在右侧面板顶部增加 `cat2-panel__header`，动态显示选中一级类目信息；新增按钮改为在此处触发，且 `level` 默认置为 2 并自动填入 `parentId` |
| R01-05 | 🔴重大缺失 | 页头操作区：设计稿有"📋 操作日志"和"← 返回 SKU 列表"两个按钮 | 实现只有"📋 操作日志"按钮，缺少"← 返回 SKU 列表"导航按钮 | FE：在 `pageHeaderActions` 区域增加"返回 SKU 列表"按钮，使用 `useNavigate` 跳转到 `/skus` |
| R01-06 | 🟡部分缺失 | 内联编辑仅设计为**二级类目**的名称字段，内联编辑中的操作区展示为：输入框 + "保存"按钮 + "取消"按钮（在操作列中显示） | 实现的内联编辑逻辑正确（仅 level-2 行），但内联编辑中保存/取消控件置于名称列内（`inlineEditCell`），而设计稿是置于操作列 | FE：将内联编辑的保存/取消控件从 name 列移至 action 列渲染，与设计稿保持一致 |
| R01-07 | 🟡部分缺失 | 左侧面板底部有固定的"+ 新增一级类目"按钮区域（`cat1-panel__add-btn-wrap`，带 border-top 分隔） | 实现中左侧面板不存在（见 R01-01），新增逻辑合并在全局页头按钮中 | FE：随左侧面板一起实现，底部固定"+ 新增一级类目"按钮 |
| R01-08 | 🟡部分缺失 | 骨架屏设计为与左右双面板完全对应的骨架结构：左侧列表骨架 + 右侧表格骨架（含表头行骨架） | 实现的骨架屏（`SkeletonLoading`）为左侧列表 + 右侧卡片两列，结构接近，但右侧骨架没有模拟表头行，且整体不对应实际双面板布局 | FE：随布局重构一并调整骨架屏，使其精确对应新的左右面板结构 |
| R01-09 | 🟢细节差异 | 设计稿页面标题含图标前缀"🗂 SKU 类目管理" | 实现的 `pageTitle` 文本为"类目管理"，无图标，标题不含"SKU"前缀 | FE：将 `setPageTitle` 调用改为"SKU 类目管理"，页面 `h1` 标题可选加图标 |
| R01-10 | 🟢细节差异 | 搜索框 placeholder 设计为"搜索类目名称..." | 实现 placeholder 为"搜索类目名称或编码..."，文本略有差异 | FE：按设计稿调整 placeholder 文案（低优先级） |

### 后端缺口

## R-02: 供应商导出+绩效对比

### 总体结论

供应商导出功能后端已实现，但前端**未接入导出 API**，整个导出流程（按钮状态机、loading 态、Toast 反馈、文件下载触发）均未实现。绩效对比 Modal 已实现核心图表，但与设计稿存在多处结构性差异，尤以**核心指标对比表格**和**供应商标签可移除**两项缺失为重。

### 差异清单

| 编号 | 严重度 | 设计稿要求 | 实际实现 | 修复方案(FE/BE) |
|------|--------|-----------|---------|----------------|
| R02-01 | 🔴重大缺失 | 筛选栏右侧有"📥 导出 Excel"按钮，具备 4 种状态：正常态（可点击）/ 导出中（loading + 禁用）/ 禁用（当前无数据）/ 成功后 Toast 反馈（文件名含日期） | `SupplierPage.tsx` 中无任何导出按钮、无导出 API 调用、无 loading 状态、无 Toast 反馈，`supplier.ts` API 文件中也无导出相关函数定义 | FE：在筛选栏增加导出按钮，实现 4 状态机（`idle/loading/disabled/success`）；调用 `GET /api/suppliers/export` 触发文件下载；成功/失败均通过 Toast 反馈。BE：接口已实现，无需改动 |
| R02-02 | 🔴重大缺失 | 对比弹框中有**核心指标对比表格**，9 行指标（准时交货率、质量合格率、价格竞争力、响应速度、订单完成率、服务满意度、合作年限、历史采购总额、平均交货周期），最优值绿色高亮、最差值红色高亮、无数据显示"—" | `PerfModal` 实现中仅有横向柱状图和雷达图、折线图，**完全没有指标对比表格** | FE：在 `PerfModal` body 区域增加 `compare-table` 组件，按各指标动态计算最优/最差值并应用高亮 class；BE：`comparePerformance` 接口需补充返回 qualityRate、priceScore、responseHours、completionRate、satisfactionScore、cooperationYears、totalPurchaseAmount、avgLeadDays 等字段 |
| R02-03 | 🔴重大缺失 | 选中工具栏（`selection-bar`）在勾选 ≥1 条时从表格底部浮出，含"已选 N 家供应商"计数 + "再选 M 家可达上限"提示 + "取消选中"+ "📊 绩效对比"按钮；设计最多支持 **5 家**对比 | 前端实现的是行内"加入对比"按钮（`CompareToggleButton`），没有底部浮动工具栏；且最大对比数为 **3**（`MAX_COMPARE = 3`），与设计稿"最多 5 家"不符 | FE：增加底部 `selection-bar` 浮动组件，替代行内按钮模式（两者可并存）；将 `MAX_COMPARE` 从 3 改为 5 |
| R02-04 | 🔴重大缺失 | 表格列结构设计为：checkbox + 供应商编码 + 供应商名称 + 等级 + 主营品类 + 联系人 + **联系电话** + 合作状态 + **综合评分（星级）** + 操作 | 实现列为：等级 + 供应商名称+联系人 + 主营品类 + 准时交货率（进度条）+ 质量异常率 + 账期 + 操作，缺少供应商编码列、联系电话列、综合评分/星级列，多出准时交货率、质量异常率、账期列 | FE：补充供应商编码（`code`）、联系电话（`contactPhone`）、综合评分（`rating` 对应星级展示）列；将准时率等指标列移至详情或对比弹框，与设计稿对齐 |
| R02-05 | 🔴重大缺失 | 表格首列为全局 checkbox（含全选 indeterminate 态），每行有行级 checkbox | 当前实现无任何 checkbox，使用行内"加入对比"按钮替代，导致批量选中语义与设计稿完全不同 | FE：在表格首列增加 checkbox 列，实现全选/全不选/indeterminate 三态；选中状态同步到对比 ID 集合 |
| R02-06 | 🟡部分缺失 | 对比弹框顶部有"已选供应商"标签行，每个供应商标签带"✕"可移除按钮，且标注"最少保留 2 家"；移除后重新计算弹框内容 | `PerfModal` 顶部无供应商标签行，无移除单个供应商功能，只能整体关闭弹框 | FE：在 `PerfModal` header 下方增加供应商标签行，维护可移除的供应商列表 state，移除后重新触发图表渲染；保留最少 2 家时"✕"自动 disabled |
| R02-07 | 🟡部分缺失 | 对比弹框顶部时间范围切换器为**分段按钮组**（近 3 月 / 近 6 月 / 近 12 月 / 全部），当前选中按钮有激活样式 | 实现为普通 `<select>` 下拉框（选项：近3个月/近6个月/近12个月），样式与设计稿不符，且缺少"全部"选项 | FE：将 `<select>` 改为分段按钮组（`time-range-selector`）；增加"全部"选项；联动 `months` 参数传给对比接口 |
| R02-08 | 🟡部分缺失 | 折线图设计为"月度采购金额趋势"（Y轴单位：万元），反映各供应商月度采购总金额 | 实现的折线图为"月度采购价格趋势"（Y轴为价格数值），数据来源为 mock 价格数据，非真实采购金额 | FE：修改折线图标题和 Y 轴说明；数据来源改为 `comparePerformance` 接口返回的 `recentAmounts`。BE：接口已返回 `recentAmounts`，需前端正确接入 |
| R02-09 | 🟡部分缺失 | 无数据告警条：当某供应商无历史绩效数据时，在弹框顶部显示全宽橙色告警条，说明该供应商暂无数据且图表以 0 值显示 | `PerfModal` 无告警条，数据不足时静默处理（用 mock 数据填充），用户无法感知 | FE：增加告警条逻辑：当 `compareSuppliers` 中存在全部维度为 0 的供应商时，渲染顶部 `no-data-banner` |

| R02-11 | 🟡部分缺失 | 筛选栏有"合作状态"和"主营品类"两个下拉筛选器 | `SupplierListQuery` 已有 `isActive` 和 `keyword` 参数，但前端页面的筛选区实现不完整，主营品类下拉使用的是静态 mock 选项，未接入实际品类数据 | FE：完善筛选栏 UI；主营品类选项改为从 `useSkuCategoryList` 动态加载 |
| R02-12 | 🟢细节差异 | 雷达图图例项下方显示"综合评分 88.7"数字，无数据的供应商显示"综合评分 —"并附"*数据不足，显示为 0"说明 | 实现的 `RadarChart` 图例仅显示供应商名称，无综合评分数字，无数据警告文本 | FE：在 `radarLegendItem` 中增加综合评分展示（基于 scores 均值计算）；无数据时显示"—"和说明文本 |
