## US-S1-001 SKU 一级类目自定义配置

**优先级**：P1

```
As a 工厂老板（admin 角色）
I want 在系统配置页中自由新增、修改、删除 SKU 一级类目
So that 工厂业务扩展产生新品类时，无需等待系统升级，能自主维护顶层分类体系
```

### 背景说明

V1 的 SKU 一级类目为系统预置固定枚举（如"原材料、半成品、成品、辅料"），工厂无法自行扩展。当客户需要新增"包材"等品类时，只能联系技术方修改代码，严重影响使用灵活性。

### 验收条件

**AC-001-01 入口可达**
- 具备 admin 角色的用户，登录后可通过"系统配置 > 类目管理"进入类目管理页面
- 非 admin 角色（销售、采购员、工人等）访问该路由，系统返回 403 页面，并提示"无权限访问"
- 类目管理页面默认展示当前租户下所有一级类目列表，按创建时间升序排列

**AC-001-02 新增一级类目**
- 点击"新增一级类目"按钮，弹出表单，字段：类目名称（必填，最大 20 字）、排序权重（选填，整数，默认按创建顺序）、备注（选填，最大 100 字）
- 同一租户下，一级类目名称不允许重复；提交重复名称时，在名称输入框下方展示行内错误提示："该类目名称已存在"，不关闭弹窗
- 新增成功后，列表刷新显示新类目；在 SKU 录入页的"一级类目"下拉选项中，新类目立即生效（无需重新登录）

**AC-001-03 修改一级类目名称**
- 点击类目记录的"编辑"按钮，进入内联编辑或弹窗编辑状态，可修改名称和备注
- 修改提交后，历史所有关联该类目的 SKU 记录，在列表展示时自动显示新名称（后端名称存储于类目表，SKU 表存储 category_id 外键，无需批量更新 SKU 数据）
- 修改操作写入操作日志（见 AC-001-06）

**AC-001-04 删除一级类目——有关联 SKU 时**
- 点击"删除"，系统后端查询该一级类目及其所有子二级类目下关联的 SKU 总数
- 若关联 SKU 总数 > 0，弹出二次确认弹窗，文案：「该类目及其子类目下共有 X 条 SKU，删除后这些 SKU 的类目将变为空（未分类），是否继续删除？」，需用户点击"确认删除"才执行
- 若关联 SKU 总数 = 0 且无子类目，弹出普通确认弹窗后直接删除

**AC-001-05 删除一级类目——有子类目时**
- 若该一级类目下存在子二级类目（无论子类目是否有关联 SKU），系统提示：「该类目下有 X 个子类目，删除将同时删除所有子类目。子类目下共有 Y 条 SKU 将变为未分类，是否继续？」
- 用户确认后，执行级联删除（一级类目 + 所有子二级类目）；相关 SKU 的 category_id 自动置空（NULL）

**AC-001-06 操作日志**
- 一级类目的新增、修改、删除操作均记录审计日志，包含字段：操作类型、操作人（user_id + 用户名）、操作时间、变更前值、变更后值、租户 ID
- 审计日志在类目管理页面底部或独立"操作日志"抽屉中可查，管理员可按时间范围筛选

### 非功能要求

- 类目列表加载时间 < 500ms（通常类目数量 < 100 条）
- 删除操作采用软删除（deleted_at 字段），保留历史数据可追溯，不做物理删除

### 任务分发

- @senior-backend-engineer：sku_categories 表开放 CRUD 接口，增加软删除字段，增加删除前关联数量查询接口，增加审计日志表写入逻辑
- @senior-frontend-engineer：CategoryConfigPage.tsx，实现一级类目列表、新增/编辑弹窗、删除二次确认、操作日志抽屉
- @senior-qa-engineer：覆盖 AC-001-04 和 AC-001-05 的边界场景；验证非 admin 角色权限拦截

## US-S1-002 SKU 二级类目自定义配置

**优先级**：P1

```
As a 工厂老板（admin 角色）
I want 在指定一级类目下自由新增、修改、删除二级子类目
So that 分类体系能精确映射工厂实际品类层级（如"原材料 > 木材"、"原材料 > 五金件"），提升 SKU 检索效率
```

### 背景说明

工厂 SKU 管理需要两级分类结构才能有效组织数百条物料数据。V1 的二级类目同样为固定枚举，无法适应不同工厂的品类差异。本 Story 与 US-S1-001 共同组成 R-01 的完整实现。

### 验收条件

**AC-002-01 二级类目展示**
- 在类目管理页面，一级类目列表每行提供"展开子类目"操作（或树形结构直接展示），点击后展示该一级类目下所有二级类目
- 二级类目列表字段：类目名称、创建时间、关联 SKU 数量、操作按钮（编辑/删除）

**AC-002-02 新增二级类目**
- 点击某一级类目行的"新增子类目"按钮，弹出表单：类目名称（必填，最大 20 字）、排序权重（选填）、备注（选填）
- 同一一级类目下，二级类目名称不允许重复；不同一级类目下的二级类目名称可以相同
- 新增成功后，SKU 录入页"二级类目"下拉选项（在选定对应一级类目后）立即显示新类目

**AC-002-03 修改二级类目**
- 可修改二级类目名称和备注；修改后关联 SKU 的展示名称同步更新（同 US-S1-001 AC-001-03 逻辑）
- 不允许将二级类目移动到其他一级类目下（父类目归属不可变更），如需调整需删除后重建

**AC-002-04 删除二级类目**
- 若该二级类目下有关联 SKU，弹出二次确认：「该类目下有 X 条 SKU，删除后这些 SKU 的二级类目将变为空，是否继续？」
- 若关联 SKU 数 = 0，弹出普通确认后直接删除
- 删除执行软删除，操作写入审计日志

**AC-002-05 类目选择联动**
- SKU 录入/编辑页面：选择一级类目后，二级类目下拉自动过滤为该一级类目的子类目；切换一级类目时，已选的二级类目自动清空
- 若某一级类目下无子类目，二级类目下拉显示"暂无子类目，请先在系统配置中添加"

**AC-002-06 空态处理**
- 当某一级类目下无子类目时，二级类目展开区域显示空态文案："暂无子类目，点击「新增子类目」开始配置"

### 任务分发

- @senior-backend-engineer：sku_categories 表层级字段（parent_id）已有，确保 CRUD 接口支持父子层级过滤；补充层级树查询接口（GET /categories/tree）
- @senior-frontend-engineer：CategoryConfigPage.tsx 中实现树形结构展示与二级类目管理功能；SKU 表单中实现一级/二级联动下拉
- @senior-qa-engineer：验证跨一级类目名称重复（应允许）、同一一级类目下名称重复（应拒绝）的边界场景

## US-S1-003 供应商列表导出

**优先级**：P2

```
As a 工厂老板 / 采购主管
I want 将供应商列表一键导出为 Excel 文件
So that 可快速生成供应商档案用于对外汇报、备档或与财务对账
```

### 背景说明

V1 后端供应商接口完整，但前端 SupplierPage.tsx 未独立实现（遗留 V1 ⚠️ 问题）。本 Story 要求同步补全供应商独立页面，并在此基础上添加导出功能。

### 验收条件

**AC-003-01 供应商独立页面（V1 遗留补全）**
- Web 端主导航存在"采购管理 > 供应商管理"独立菜单项，进入后展示供应商列表
- 列表字段：供应商编码、供应商名称、联系人、联系电话、主营品类、合作状态（合作中/已停用）、综合评分
- 支持按名称、编码、合作状态筛选；支持分页（默认每页 20 条）

**AC-003-02 导出触发**
- 列表页顶部提供"导出 Excel"按钮，点击后触发导出（无需二次确认）
- 导出范围：当前筛选条件下的全部记录（不受分页限制）；若无筛选条件，导出全部供应商
- 点击后按钮显示 loading 状态，导出完成后自动下载文件，按钮恢复正常

**AC-003-03 导出文件内容**
- 文件格式：.xlsx
- 文件命名：`供应商列表_{YYYY-MM-DD}.xlsx`
- 导出字段（列顺序）：供应商编码、供应商名称、主营品类、联系人姓名、联系人电话、联系人邮箱、合作状态、综合评分、创建时间
- 首行为表头，字体加粗；数据行从第二行开始
- 综合评分为数字格式（保留 1 位小数）；创建时间格式：YYYY-MM-DD

**AC-003-04 空数据处理**
- 若当前筛选条件下无供应商数据，点击导出后提示："当前筛选条件下无数据，无法导出"，不触发下载

**AC-003-05 权限控制**
- 导出功能对 admin、采购主管（supervisor）角色可见；销售人员和生产工人角色不展示导出按钮

### 任务分发

- @senior-backend-engineer：GET /suppliers/export 接口，接受与列表相同的筛选参数，返回 Excel 文件流；使用 ExcelJS 生成，样式满足 AC-003-03
- @senior-frontend-engineer：SupplierPage.tsx 独立实现（补全 V1 遗留）；导出按钮 + loading 状态处理
- @senior-qa-engineer：验证筛选条件传递是否正确（导出结果与当前列表一致）；验证空数据提示；验证权限拦截

## US-S1-004 供应商绩效对比

**优先级**：P2

```
As a 采购员 / 工厂老板
I want 选择 2-5 家供应商进行横向绩效对比，查看多维度评分雷达图和关键指标对比表
So that 在面临同一物料多家供应商时，能有数据依据地做出采购决策，减少凭经验盲选的风险
```

### 背景说明

V1 已有单一供应商的绩效评分数据（delivery_performance、quality_score 等字段），但缺少多供应商横向比较的视图。本 Story 在现有数据基础上新增对比功能，不涉及绩效数据采集逻辑变更。

### 验收条件

**AC-004-01 对比入口**
- 供应商列表页提供多选框；用户勾选 2-5 家供应商后，顶部出现"对比所选供应商"按钮
- 仅勾选 1 家或超过 5 家时，该按钮 disabled，并提示"请选择 2-5 家供应商进行对比"

**AC-004-02 绩效对比页面布局**
- 点击对比按钮后，进入绩效对比页（或弹出全屏弹窗）
- 页面顶部展示所选供应商名称列表，支持"移除"单家供应商（最少保留 2 家）
- 页面分两个区域：雷达图区域（上）、关键指标对比表（下）

**AC-004-03 雷达图展示**
- 雷达图维度（6 个）：准时交货率、质量合格率、价格竞争力、响应速度、服务满意度、综合评分
- 每家供应商用不同颜色的多边形展示，图例清晰标注供应商名称与颜色对应关系
- 若某供应商某维度数据为空（历史订单不足），该顶点显示 0，并在图例旁标注"*数据不足"

**AC-004-04 关键指标对比表**
- 表格结构：行为指标维度（含上述 6 个维度 + 合作年限 + 历史采购总额 + 平均交货周期），列为各供应商
- 每个指标的最优值（最高分或最短周期）用绿色高亮显示，最差值用红色高亮
- 若全部对比供应商某指标均无数据，该行显示"—"

**AC-004-05 空态提示**
- 若对比的供应商中有绩效数据为空（从未有过采购记录），在其对应列顶部显示提示条："该供应商暂无历史绩效数据，图表数据仅供参考"
- 对比图表仍正常渲染（空数据视为 0），不阻断用户操作

**AC-004-06 数据时间范围**
- 对比数据默认取近 12 个月的绩效记录；页面顶部提供时间范围选择器（近 3 个月 / 近 6 个月 / 近 12 个月 / 全部），切换后图表和表格实时刷新

### 任务分发

- @senior-backend-engineer：GET /suppliers/compare 接口，接受供应商 ID 数组和时间范围参数，返回多维度聚合绩效数据
- @senior-frontend-engineer：绩效对比页面，使用 ECharts 或 Recharts 实现雷达图，实现指标表格高亮逻辑
- @senior-qa-engineer：验证空数据场景（新供应商 0 条记录）；验证 1 家和 6 家供应商时的按钮 disabled 逻辑；验证时间范围切换后数据刷新

## US-S1-005 采购价格批量导入

**优先级**：P1

```
As a 采购员
I want 下载标准 Excel 模板，填写历史采购价格数据后批量上传至系统
So that 不需要逐条手动录入数百条历史价格记录，能在系统上线初期快速完成数据迁移
```

### 背景说明

工厂历史采购价格数据存储在各种 Excel 文件中，手工逐条录入体验极差，是系统上线推广的最大阻力。批量导入是降低迁移成本的关键功能。基于 BD-001 快照机制，价格数据的时效性由报价日期字段管理，历史记录以追加方式保存，不覆盖原有数据。

### 验收条件

**AC-005-01 导入入口与向导流程**
- 采购价格管理页面顶部提供"批量导入"按钮
- 点击后打开导入向导，分 4 步：第一步「下载模板」、第二步「上传文件」、第三步「预览校验」、第四步「确认导入」
- 向导顶部显示步骤进度条，当前步骤高亮

**AC-005-02 模板下载（第一步）**
- 提供可下载的 Excel 模板文件（.xlsx），命名：`采购价格导入模板_v1.xlsx`
- 模板第一行为表头（加粗），第二行为示例数据（灰色字体），共 7 列（列顺序固定）：

| 列名 | 说明 | 是否必填 | 格式要求 |
|---|---|---|---|
| SKU编码 | 与系统 SKU 编码精确匹配 | 必填 | 文本 |
| 供应商编码 | 与系统供应商编码精确匹配 | 必填 | 文本 |
| 单价 | 含税或未税价格数值 | 必填 | 数字，保留 2 位小数，不允许负数 |
| 货币单位 | 如 CNY、USD | 必填 | 文本，枚举值：CNY / USD / EUR |
| 含税标记 | 是否含税 | 必填 | 枚举值：是 / 否 |
| 报价日期 | 该价格的报价生效日期 | 必填 | 日期格式：YYYY-MM-DD |
| 备注 | 自由填写 | 选填 | 文本，最大 200 字 |

- 模板内嵌 Excel 数据有效性校验（货币单位、含税标记设置下拉选项）

**AC-005-03 文件上传（第二步）**
- 支持拖拽上传或点击选择文件；仅接受 .xlsx 和 .xls 格式
- 文件大小限制：单次上传最大 10MB
- 单次导入行数上限：5000 条（不含表头行）；超出时提示"超出单次上限，请分批上传（本次最多 5000 条）"，不允许继续上传

**AC-005-04 预览校验（第三步）**
- 文件解析完成后，展示预览表格（前 100 行可见，超出部分折叠并提示"共 X 行，当前显示前 100 行"）
- 校验规则（逐行校验）：
  - SKU编码不存在于系统：该行背景色标红，错误原因列显示"SKU编码不存在：{code}"
  - 供应商编码不存在于系统：该行背景色标红，错误原因列显示"供应商编码不存在：{code}"
  - 必填字段为空：该单元格标红，错误原因列显示"字段「{列名}」不可为空"
  - 单价为负数或非数字：该单元格标红，错误原因列显示"单价格式错误"
  - 货币单位不在枚举范围：该单元格标红，错误原因列显示"货币单位须为 CNY / USD / EUR"

```
So that 生产过程中发生设备故障、物料缺失等异常时，能立即在系统中留存记录，方便车间主管快速响应处理
```

### 背景说明

异常上报是生产过程中的重要应急功能，与完工上报并列为工人端的两个核心操作。异常上报后任务状态流转为"异常待处理"，车间主管需介入处置。本 Story 与 US-S1-009、US-S1-010 共同构成 R-06 Web 端任务管理的完整功能。

### 验收条件

**AC-011-01 异常上报入口**
- 任务状态为"进行中"时，任务详情页/操作区同时显示"完工上报"和"异常上报"两个按钮
- 点击"异常上报"后弹出异常上报弹窗，不跳转页面

**AC-011-02 异常上报表单字段**

| 字段 | 必填 | 说明 |
|---|---|---|
| 异常类型 | 必填 | 单选，枚举：设备故障 / 物料缺失 / 质量问题 / 其他 |
| 异常描述 | 必填 | 文本域，最少 10 字，最大 500 字 |
| 附件图片 | 选填 | 支持上传 1-3 张图片，格式 JPG/PNG，单张最大 5MB |
| 是否影响生产进度 | 必填 | 单选：是 / 否 |

**AC-011-03 异常类型说明提示**
- 每种异常类型后面提供"?"帮助图标，悬浮（PC）或点击（移动端）后展示该类型的说明：
  - 设备故障：生产设备无法正常运行
  - 物料缺失：所需原材料/半成品不足，无法继续生产
  - 质量问题：发现产品质量不符合要求
  - 其他：以上类型之外的异常情况

**AC-011-04 提交后状态变更**
- 提交成功后：任务状态变为"异常待处理"，弹窗关闭，列表刷新
- Toast 提示（3 秒消失）："异常已上报，等待主管处理"

**AC-011-05 车间主管通知**
- 异常上报提交后，系统向当前工单所属车间的车间主管推送站内通知：「任务「{任务编号}」发生异常（{异常类型}），请及时处理」
- 通知中包含跳转链接，点击直达该任务详情页

**AC-011-06 异常处置（主管操作）**
- 车间主管在任务详情页看到"异常待处理"状态的任务，可进行以下操作（工人无此操作）：
  - 「标记已处理」：填写处理说明（选填），任务状态恢复为"进行中"
  - 「挂起任务」：填写挂起原因（必填），任务状态变为"已挂起"，不计入生产进度

**AC-011-07 异常记录可查**
- 任务详情页历史记录时间线中，异常上报事件和处置记录均按时间顺序展示（操作人、时间、内容）

**AC-011-08 图片上传**
- 图片上传接口支持多文件上传；上传中显示各文件的进度条；上传失败时单文件独立提示，不影响其他文件提交
