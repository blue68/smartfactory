### 1.2 核心组件状态规范

#### 按钮（Button）

| 变体 | 正常态 | Hover | Active | Disabled | Loading |
|---|---|---|---|---|---|
| primary | bg `#2563EB`, text `#fff` | bg `#1D4ED8` | bg `#1E40AF` scale(0.98) | opacity 0.5 cursor-not-allowed | spinner + text 变 "处理中..." |
| secondary | border `#E5E7EB`, bg `#fff` | bg `#F9FAFB` | bg `#F3F4F6` | opacity 0.5 | spinner |
| danger | bg `#DC2626`, text `#fff` | bg `#B91C1C` | scale(0.98) | opacity 0.5 | spinner |
| ghost | bg transparent, text `#2563EB` | bg `#EFF6FF` | bg `#DBEAFE` | opacity 0.4 | spinner |

#### 输入框（Input）

| 状态 | 样式 |
|---|---|
| 默认 | border `#E5E7EB`, bg `#fff` |
| Focus | border `#2563EB`, box-shadow `0 0 0 3px rgba(37,99,235,0.1)` |
| Error | border `#DC2626`, box-shadow `0 0 0 3px rgba(220,38,38,0.1)`, 下方显示红色错误文字 |
| Disabled | bg `#F3F4F6`, text `#9CA3AF`, cursor `not-allowed` |
| Readonly | bg `#F9FAFB`, text `#6B7280` |

#### Badge / 状态标签

| 类型 | bg | text | border |
|---|---|---|---|
| 成功/启用 | `#F0FDF4` | `#15803D` | `#BBF7D0` |
| 警告/草稿 | `#FFFBEB` | `#92400E` | `#FCD34D` |
| 危险/停用 | `#FEF2F2` | `#B91C1C` | `#FCA5A5` |
| 信息/一级 | `#EFF6FF` | `#1D4ED8` | `#BFDBFE` |
| 中性/历史 | `#F3F4F6` | `#374151` | `#E5E7EB` |
| 紧急/橙色 | `#FFF7ED` | `#C2410C` | `#FED7AA` |

#### Toast 通知

- 位置：右上角，distance-top `80px`，distance-right `24px`
- 动画：从右侧 `translateX(110%)` 滑入，`translateX(0)`，3 秒后 fade + slide 退出
- 宽度：最小 `280px`，最大 `400px`
- 类型：success（绿色左边框）/ error（红色左边框）/ warning（黄色左边框）/ info（蓝色左边框）

#### Modal

- 遮罩：`rgba(0,0,0,0.4)` backdrop-blur `2px`
- 入场：scale `0.95 → 1` + opacity `0 → 1`，duration `200ms`
- 尺寸规格：`sm` 400px / `md` 560px / `lg` 720px / `xl` 960px
- 结构：header（标题 + 关闭按钮）+ body（scrollable）+ footer（取消 + 确认）

#### 加载态

- 页面级：全宽骨架屏（Skeleton），高度与实际内容行等高
- 按钮级：内联 spinner（16px 圆形动画），替换图标或与文字并排
- 表格级：overlay spinner（半透明遮罩居中 spinner）+ 禁止交互

### [artifact:交互说明] R-01

#### 搜索行为

- 触发方式：`onChange` 实时过滤，无防抖（数据量小，本地过滤）
- 匹配逻辑：OR 匹配类目名称（包含）和编码（包含），大小写不敏感
- 展示规则：
  - 若一级类目名称/编码匹配：该一级及其所有子类目全部显示
  - 若二级类目名称/编码匹配：该二级所属一级折叠显示（一级行保留作为父容器），仅匹配的二级行展示
  - 无匹配时：表格空态，文字"未找到匹配的类目"
- 清空搜索框：恢复完整树形列表，动画 `transition: opacity 0.15s`

#### 新增/编辑 Modal 交互

- 层级切换联动：
  - 选择"一级类目" → 隐藏"所属一级类目"下拉，`parentId` 置为 `null`
  - 选择"二级类目" → 显示"所属一级类目"下拉（fade-in `150ms`），该下拉列出当前所有一级类目（`name（code）`格式）
- 编码输入：`onChange` 时强制转大写（`value.toUpperCase()`），不允许中文输入
- 名称实时校验：blur 时检查是否为空、是否超过 20 字
- 提交时全量校验，首个错误字段 `focus`
- 创建成功：Modal 关闭，Toast `success` "已创建类目：{name}"，列表自动刷新，滚动到新增行（若可见）
- 保存失败：Toast `error` 显示服务端返回的错误信息（如"编码已存在"）

#### 删除 Modal 交互

- 打开：获取删除目标的子类目数量（本地计算，从 rawList 中统计）和关联 SKU 数量（API 返回或已有数据）
- 确认按钮使用 `danger` 变体，hover 加深红色
- 有子类目时：确认按钮文字变为"确认级联删除"，增加视觉警示
- 删除请求进行中：确认按钮显示 loading spinner，遮罩禁止关闭（`maskClosable: false`）
- 删除成功：Modal 关闭，Toast `success` "已删除类目：{name}"，列表刷新
- 删除失败（有 SKU 关联且后端拦截）：Modal 内显示内联错误提示"该类目下存在 SKU，无法删除"，不关闭 Modal

#### 状态反馈

| 操作 | 成功 Toast | 失败 Toast |
|---|---|---|
| 新增类目 | "已创建类目：{name}" | "创建失败：{message}" |
| 编辑类目 | "已更新类目：{name}" | "更新失败：{message}" |
| 删除类目 | "已删除类目：{name}" | "删除失败：{message}" |

### [artifact:交互说明] R-02

#### 导出 Excel 交互流程

1. 用户点击"导出 Excel ↓"按钮
2. 按钮立即变为 loading 状态（disabled + spinner + 文字"导出中..."）
3. 前端向 `GET /api/suppliers/export` 发起请求（响应为 Blob）

5. 请求失败：按钮恢复默认态，Toast `error` "导出失败，请稍后重试"
6. 超时（>15s）：自动取消请求，Toast `warning` "导出超时，建议刷新后重试"

#### 绩效对比交互流程

**图表数据为空时的处理**

- 整个供应商绩效数据不足（订单 < 3）：展示空态组件（见原型），不渲染图表

**数据刷新**
- 进入 Tab 时自动拉取选中供应商的绩效数据
- 提供刷新图标按钮（右上角），手动重新拉取

### [artifact:交互说明] R-03

#### Stepper 导航规则

- 回退：允许点击"上一步"按钮回退，但回退后当前步骤数据保留（不重置）
- 跳步：不允许。Step 指示器不可点击，只能通过"上一步/下一步"按钮顺序流转
- Step 1 → Step 2：无校验，直接进入（下载模板为可选操作）
- Step 2 → Step 3：需要已选中文件，未选中时"下一步"按钮 disabled；选中后自动向后端发送文件解析请求（loading 遮罩覆盖内容区），解析成功后自动进入 Step 3
- Step 3 → Step 4（执行导入）：
  - 选择"跳过错误，导入正确行"：弹出确认弹窗"将跳过 12 条错误行，导入 458 条，是否继续？"，确认后执行导入
  - 全部正确时：按钮文字"确认导入"，直接执行
  - 导入执行中：进度条展示（使用 SSE 或轮询）
- Step 4：不显示"上一步"，只显示"关闭"和"查看价格列表"

#### 文件上传交互

- 拖拽：`dragenter`/`dragover` 时上传区进入 `--dragover` 态；`dragleave` 或 `drop` 后退出
- 非法格式文件被拖入：不进入 `--dragover` 态，直接在 `drop` 后显示错误提示条
- 多文件拖入：仅取第一个文件，忽略其余
- 点击区域：触发 `<input type="file" accept=".xls,.xlsx">` 的 `click()`

#### 预览校验交互

- 错误行固定在表格顶部，正确行在下方，两个分区之间有分割线和标签
- 重复记录不视为错误（不标红），单独用黄色区分，展示在错误行之后、正确行之前
- 跳过错误行后，统计栏更新为"将导入 458 条（跳过 12 条错误）"
- "重新上传"按钮：清空文件，回退到 Step 2

#### 重复记录规则

- 相同的（SKU编码 + 供应商编码 + 报价日期）组合已存在于系统中
- 处理方式：默认追加为新版本（不覆盖），在预览阶段以黄色高亮提示
- 统计栏显示"⚠ 重复 X 条（将追加新版本）"

### [artifact:交互说明] R-06

#### 状态 Tab 切换

- Tab 上的数字徽标实时反映当前筛选下的任务数量
- 切换 Tab 时：内容区出现 loading skeleton（1~2 行卡片骨架屏），200ms 后渲染实际数据
- 数字徽标 > 99 时显示"99+"

#### 状态流转规则

```
待开始  ──[开始生产]──→  进行中  ──[完工上报]──→  已完成
  │                                                  │
  └──────────────────[异常上报]──→  异常──────────────┘
                    （任意非完成状态均可异常上报）
```

- 每个状态流转操作只能向前（不支持回退），符合 PRD 约束
- 操作按钮根据当前任务状态动态渲染，不可见/不可操作的操作不显示

#### 开始生产交互

1. 点击"开始生产"按钮 → 打开确认弹窗（小尺寸 Modal，`sm`）
2. 用户确认 → 发送 `PATCH /api/tasks/{id}/start` 请求（附带当前 `version` 乐观锁字段）
3. 成功响应：Modal 关闭，任务卡片/行状态标签更新为"进行中"，卡片操作按钮更新，开始时间显示
4. 乐观锁冲突（HTTP 409）：弹出"任务状态已变更"弹窗，用户点击"刷新任务"后重新拉取该任务最新数据

#### 完工上报交互

1. 点击"完工上报" → 打开 Modal（`md` 尺寸）
2. 件数输入：数字输入，范围 1 ~ 计划件数，`onChange` 实时更新工资预算
3. 工时输入：数字输入，`min="0.1"`，保留一位小数
4. 工资预算实时计算公式：`件数 × 当前用户等级单价`，展示在蓝色卡片中
5. 提交：发送 `POST /api/tasks/{id}/complete`（带 `version` 字段）
6. 成功：Modal 关闭，Toast `success` "完工上报成功，工资 ¥300.00 已记录"，任务状态更新
7. 乐观锁冲突（409）：Modal 内显示内联错误"任务已被更新，请关闭弹窗刷新后重试"，不关闭 Modal

#### 异常上报交互

1. 点击"异常上报" → 打开 Modal（`md` 尺寸）
2. 异常类型为单选，不可多选
3. 描述为必填，最少 10 字，超出 500 字时禁止输入（实时字数提示）
4. 图片上传：
   - 选择文件后立即在前端压缩（Canvas resize，长边不超过 1920px）
   - 压缩完成后自动上传至服务端，返回 url 后显示缩略图
   - 上传失败：缩略图位置显示"上传失败 ×"，可重试
5. 提交：所有图片上传完成后才允许点击"提交异常"（未完成时按钮 disabled + tooltip "图片上传中..."）
6. 成功：Toast `success` "异常已上报，车间主管将收到通知"，任务状态变为"异常"

#### 响应式布局规则

| 断点 | 布局 |
|---|---|
| `≥ 1280px`（桌面宽） | 默认表格布局，切换按钮可切为卡片 |
| `768px ~ 1279px`（平板） | 卡片布局，2 列 Grid |
| `< 768px`（移动端） | 卡片布局，1 列，操作按钮全宽 |

- 状态 Tab 在移动端水平滚动（`overflow-x: auto; white-space: nowrap`），不换行
- 筛选栏在移动端垂直堆叠（`flex-direction: column`）
- Modal 在移动端全屏展示（宽度 100vw，高度 100dvh），避免键盘遮挡时 Modal 被推出视口

#### 数据同步说明

- 页面进入时：拉取任务列表，同时记录各任务的 `version` 字段（乐观锁）
- 定期轮询：每 30 秒静默刷新任务列表（不显示 loading）
- 刷新时若检测到有任务状态变化：对应任务行/卡片以短暂高亮（背景闪烁 `#EFF6FF`）提示用户注意
