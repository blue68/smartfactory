## 五类分级结果

### 1. 可直接用

判定口径：
- 已设默认模板
- 至少 `2` 个工步
- 所有工步都有标准工时
- 所有工步都有工价

结果：
- 模板数：`0`

结论：
- 当前没有一套模板达到“无需补数据即可直接作为标准生产路线”的状态。

### 2. 需要补工时

判定口径：
- 模板下存在任一工步 `standard_hours` 为空或 `0.0000`

结果：
- 模板数：`746`
- 步骤数：`746`

代表样例：
- `T/F01床侧板-标准工序模板` / `Z310000050 / T/F01床侧板`

- `左外01护翼布-白01-标准工序模板` / `Z200000083 / 左外01护翼布-白01`

建议：
- 先补半成品板件、面布、围布、护翼等高频中间件工时。
- 优先补会参与排产和工价核算的工步，不建议先从低频试制模板开始。

- 不建议只补工价不补工时，否则页面可展示，但排产与核算口径仍然不完整。

### 4. 需要拆成多工步

判定口径：
- 模板仅有 `1` 个工步

结果：
- 模板数：`980`

高优先级样例：
- `床架-宽细竖条纹-绒布-白-F-标准工序模板` / `Z460000001` / 投料 `157`
- `床架-宽细竖条纹-绒布-白-Q-标准工序模板` / `Z460000002` / 投料 `157`
- `床架-宽细竖条纹-绒布-黑-Q-标准工序模板` / `Z460000005` / 投料 `155`
- `床架-护翼密竖条纹-绒布-白-Q-标准工序模板` / `Z070000004` / 投料 `155`
- `床架-宽细竖条纹-绒布-白-K-标准工序模板` / `Z460000003` / 投料 `154`
- `床架-护翼三条板-绒布-黑-F-标准工序模板` / `Z450000005` / 投料 `154`

投料密度分布：
- `1-5` 条投料：`509` 套模板
- `6-20` 条投料：`314` 套模板
- `21-50` 条投料：`104` 套模板
- `50+` 条投料：`54` 套模板

结论：
- 这批数据当前更像“SKU 对应一张投料模板”，不是完整的多工步工艺路线。
- 尤其投料超过 `50` 条的单工步模板，优先拆成多工步，否则后续排产、报工、工价、领料都不够清晰。

建议拆分优先顺序：
1. `50+` 投料的成品模板
2. `21-50` 投料的成品模板
3. 高频半成品模板
4. 低投料密度、低频模板

### 5. 需要设默认模板

判定口径：
- `is_default = 0`

结果：
- 模板数：`981`
- 覆盖 SKU 数：`981`

结论：
- 当前所有已导入模板都没有默认模板标记。
- 这会直接影响 SKU 工序配置页的“默认工艺”使用体验，也会影响后续从 SKU 自动带出模板的稳定性。

代表样例：
- `F01上梁-标准工序模板` / `Z310000027 / F01上梁`
- `Q01上梁-标准工序模板` / `Z310000031 / Q01上梁`
- `K01上梁-标准工序模板` / `Z310000034 / K01上梁`
- `F01面板-标准工序模板` / `Z310000026 / F01面板`
- `Q01面板-标准工序模板` / `Z310000030 / Q01面板`

建议：
- 如果一个 SKU 当前只有一套模板，建议直接批量标记为默认模板。
- 如果后续会存在标准版 / 客制版 / 试制版，需先定义默认模板选择规则，再批量回填。
