# 修订汇总 - 2026-03-24

## 概述

本次修订围绕三个目标推进：

1. 收敛销售主链的数据流、状态机与页面交互，打通 `客户 -> 订单 -> 审批/确认 -> 生产 -> 发货 -> 收货 -> 结算`。
2. 收敛采购主链的数据流与页面追溯，打通 `采购建议 -> 采购单 -> 送货单 -> 质检 -> 入库 -> 三单匹配 -> 尾单处理`。
3. 补齐结算与应收账款视图能力，并通过接口层、页面层、浏览器层回归锁定行为。

---

## 一、销售模块修订

### 1.1 数据流与结构漂移修复

- 新增销售结构对齐迁移：
  - `services/api/src/migrations/V5_sales_schema_alignment.sql`
- 修复并统一以下字段/表的历史漂移：
  - `sales_orders.expected_delivery`
  - `sales_orders.order_type`
  - `sales_orders.sales_person_id`
  - `sales_order_items.qty_ordered`
  - `sales_order_items.qty_delivered`
  - `order_constraint_checks`
  - `sales_deliveries`
  - `sales_delivery_items`
  - `sales_settlements`

### 1.2 订单主链打通

- 修复普通订单创建页接错接口的问题。
- 修复紧急订单“只分析不落库”的问题，改为：
  - 先 AI 影响评估
  - 再提交插单审批申请
- 修复 `unitPrice=0` 的错误提交。
- 修复列表页新建订单时 `skuId` 为空的问题。
- 修复草稿保存“假保存”问题，改为真实落库。
- 订单详情补齐：
  - 审批记录
  - 关联生产工单
  - 发货记录
  - 操作日志
  - 关键节点时间线

### 1.3 状态机与服务链路修复

- 销售确认/审批通过后，统一委托正式生产服务建工单，不再走简化写库逻辑。
- 发货、完成动作改为委托真实交付服务：
  - 发货写入 `sales_deliveries / sales_delivery_items`
  - 完成改为确认收货单
- 支持按明细分批发货，不再固定整单一次发完。
- 统一销售详情中的状态、日志、审批人、操作人显示语义。

### 1.4 销售浏览器层覆盖

新增并跑通销售链 Playwright 冒烟，覆盖：

1. 订单详情展示关键节点、工单、发货记录
2. 在产订单分批发货
3. 紧急订单审批通过
4. 已发货订单确认完成
5. 列表页创建草稿
6. 列表页创建待审批紧急订单
7. 独立新建页保存草稿
8. 独立新建页常规订单自动确认
9. 独立新建页紧急插单评估并提交审批

## 二、生产与 MRP 修订

### 2.1 建工单链路修复

- 修复工单创建时使用错误字段 `sales_order_id` 的问题，统一为 `order_id`。
- 修复工单创建状态校验不一致问题。
- 销售确认后自动创建正式生产工单。

### 2.2 缺料与采购建议联动

- 生产工单创建后，如存在缺料，会自动生成采购建议。
- `MrpService` 支持复用外部事务，保证销售确认 -> 建工单 -> 物料需求 -> 缺料检测在同一事务视角内一致。
- 修复缺料检测错误覆盖 `qty_reserved` 的问题。

---

## 三、采购模块修订

### 3.1 采购主链收敛

- 批量采购建议转单增加硬校验，禁止部分成功。
- 采购建议、采购单、送货单、入库单、三单匹配之间的数据口径统一。
- 手工建采购单和建议转采购单默认直接进入 `confirmed`。
- 采购订单状态推进统一收口：
  - `confirmed`
  - `partial_received`
  - `received`
  - `cancelled`

### 3.2 到货、质检、入库闭环

- 送货单创建 SQL 错位问题已修复。
- 合格入库后自动：
  - 回冲 `qty_in_transit`
  - 重评估缺料工单
  - 推进采购订单头状态
- 退货完成后也复用统一采购状态重算逻辑。

### 3.3 采购追溯页面补齐

新增或补强以下页面与联动：

- `PurchaseOrderPage`
- `PurchaseDeliveryPage`
- `PurchaseReceiptPage`
- `MatchPage`
- `IncomingInspectionPage`

页面支持的主链互跳包括：

- 采购订单 -> 送货单
- 送货单 -> 质检
- 质检 -> 入库单
- 入库单 -> 三单匹配
- 三单匹配 -> 返回采购单 / 入库单

### 3.4 三单匹配补强

- 三单匹配支持详情页与深链跳转。
- 差异确认后支持只读确认记录展示。
- 送货单页可直接看到是否已匹配，并可发起执行匹配。

### 3.5 采购浏览器层覆盖

新增并跑通采购链 Playwright 冒烟，覆盖：

1. 送货单详情时间线展示
2. 未匹配送货单跳执行匹配并预填
3. 从送货单发起质检并回跳
4. 从质检详情跳入库单并进入三单匹配
5. 三单差异确认
6. 差异确认后再次进入详情显示只读确认记录
7. 从入库单跳采购订单并自动打开详情
8. 从采购订单回看送货单和入库单

## 四、结算与应收账款修订

### 4.1 结算页权限与行为收敛

- `boss / supervisor`：
  - 可查看应收汇总
  - 可管理结算单
- `sales`：
  - 仅可查看列表
  - 不显示应收汇总
  - 不显示管理按钮

### 4.2 结算页能力补齐

新增或补齐以下能力：

- 关键字筛选：
  - 结算单号
  - 客户
  - 订单号
- 逾期筛选：
  - `仅看逾期`
- 逾期高亮：
  - 列表显示到期日
  - 逾期项显示 `已逾期`
- CSV 导出：
  - 新增 `GET /api/settlements/export/csv`
- 应收汇总切换：
  - 按客户
  - 按月份
  - 账龄
- 汇总反向过滤：
  - 点击客户汇总，直接过滤列表
- 账龄视图：
  - 逾期金额
  - 逾期笔数
  - `未逾期 / 1-30 / 31-60 / 61-90 / 90+`

### 4.3 结算接口修订

在 `services/api/src/modules/settlement` 中新增或补强：

- `keyword`
- `overdueOnly`
- `customerId`
- `groupBy=aging`
- `export/csv`

### 4.4 结算浏览器层覆盖

新增并跑通结算页 Playwright 冒烟，覆盖：

1. `boss` 可确认草稿并标记已付
2. `supervisor` 只可取消
3. `sales` 仅可查看
4. 汇总维度切换刷新
5. 关键字筛选、逾期高亮、CSV 导出
6. 客户汇总反向过滤列表
7. 账龄分桶与逾期金额汇总
