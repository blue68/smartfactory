## 一、AI 采购建议生成→审批→执行 完整流程

### 1.1 流程概述

```
系统定时触发（每日07:00）
       ↓
AI 分析中（思考状态）
       ↓
生成建议清单
       ↓
推送通知（采购员 + 老板）
       ↓
老板审批
  ├── 批准 → 采购员执行 → 录入采购单 → 追踪到货
  └── 驳回 → 采购员收到通知 → 可申请重新分析
       ↓
货物到达 → 仓库入库 → 库存更新 → 采购单关闭
```

### 1.2 阶段一：AI 分析中状态

**触发时机**：
- 每日 07:00 系统自动触发
- 用户手动点击"发起采购需求分析"

**用户行为**：用户点击"手动发起采购需求分析"按钮。

**页面状态变化**：
1. 按钮立即变为 loading 状态（spinner + "分析中..."），disabled，防止重复点击
2. 页面中央出现 AI 思考中模态框（`--shadow-xl`，背景遮罩 `rgba(0,0,0,0.4)`），动画：从下方 slide-up + fade-in（300ms，cubic-bezier(0.34, 1.56, 0.64, 1)）
3. 模态框内显示步骤列表，步骤依次执行：

```
步骤列表动态变化过程：
T+0.0s → 步骤1"读取在产订单数据"变为 spinning 状态（⟳）
T+0.8s → 步骤1完成（✓，绿色），步骤2"计算 BOM 物料需求"变为 spinning
T+2.5s → 步骤2完成，步骤3"匹配供应商库存与交期"变为 spinning
T+6.0s → 步骤3完成，步骤4"生成采购建议"变为 spinning
T+8.0s → 全部完成，模态框消失（fade-out 200ms），页面刷新显示新建议
```

4. 步骤超过 10 秒显示："预计还需约 X 秒"（倒计时）
5. 始终显示"取消"按钮，点击后：确认对话框 → 确认取消 → 停止分析 → 恢复页面状态

**动画效果**：
- 步骤完成勾号：scale from 0 to 1（200ms，spring 弹簧曲线）
- 当前步骤 spinning 图标：rotate 360deg，1s linear infinite
- 步骤文字从灰色 `--color-gray-400` 渐变到主色（当前步骤）/ 成功色（完成）

**异常/错误反馈**：
- 数据不足（BOM 缺失）：步骤失败（❌），模态框切换为错误状态，显示具体缺失订单
  - 操作：[去补录 BOM] [跳过问题订单继续分析] [取消]
- 网络超时（> 30s）：
  - 25s 时出现"比平时慢一些，请稍候..."
  - 30s 自动重试1次（用户无感知，仅步骤文字变为"重新连接中..."）
  - 二次失败：错误状态，[重试] [手动操作]

### 1.3 阶段二：老板审批交互

**用户行为**：老板在 PC 端或手机收到通知，点击进入审批。

**单条建议的审批操作**：

点击"批准"按钮：
1. 若金额 ≤ 5000 元：
   - 按钮立即变为 loading（200ms）
   - 显示成功 Toast："采购建议已批准，已通知采购员"（3s 自动消失）
   - 该条建议状态标签切换为"已批准"（绿色），行背景 `--color-success-50`，动画 200ms
   - 批准/驳回按钮替换为"查看执行进度"
2. 若金额 > 5000 元：
   - 弹出二次确认对话框（modal）：
     - 标题："确认批准大额采购"
     - 内容：物料名称、数量、金额（大号字体强调金额）
     - 操作：[确认批准]（红色警示按钮） [取消]
   - 对话框 slide-up 动画（250ms），背景遮罩
   - 确认后流程同上

点击"驳回"按钮：
1. 弹出驳回原因输入对话框（必填 textarea）
2. 原因输入框：最大 200 字，字数实时计数
3. 点击"确认驳回"：建议状态变为"已驳回"（红色标签），采购员收到通知
4. 点击"取消"：关闭对话框，建议状态不变

**批量审批**（多选）：
- 列表支持复选框选择
- 选中后出现底部固定操作栏（position: sticky bottom）："已选 X 条" + [批量批准] [取消选择]
- 批量批准：检查是否有超 5000 元项目，有则逐一确认；或显示汇总确认框

## 二、面料缸号入库→领料→一致性校验 流程

### 2.1 缸号入库流程（小程序）

**用户行为**：仓库管理员选择面料类物料进行入库操作。

**页面状态变化**：
1. 物料选择后，系统检测物料分类
2. 若为"面料"或"皮料"：缸号字段从隐藏状态动画展开（max-height transition，300ms ease-in-out），必填星号出现
3. 缸号字段展开动画结束后，键盘自动聚焦（iOS 和 Android 均需处理软键盘弹起）
4. 输入缸号时：
   - 实时搜索已有缸号（debounce 300ms），下方出现匹配建议列表
   - 选择已有缸号：字段背景变为 `--color-info-50`，底部提示"已有库存：XX 平方米，将累计入库"
   - 输入全新缸号：字段背景变为 `--color-warning-50`，底部提示"新缸号，将创建新批次"
5. 提交时校验缸号不为空（面料类），否则字段边框变红 + shake 动画（水平抖动，200ms）

**动画效果**：
- 缸号字段展开：max-height: 0 → max-height: 200px，opacity 0→1，300ms ease
- 缸号建议列表：fade-in + slide-down（150ms）
- 新旧缸号背景色切换：transition background-color 200ms

**错误反馈**：
- 缸号为空提交：字段边框红色 + 文字"缸号为必填项"（slide-down 显示，200ms）+ 字段 shake 动画
- 缸号超过 20 字符：输入框边框变警告色 + 提示"最多20个字符"

### 2.2 缸号领料校验流程（小程序）

**用户行为**：车间主管发起领料申请，选择面料类物料。

**页面状态变化**：
1. 选择面料物料后，系统查询该订单已有领料缸号
2. **有已用缸号时**：
   - 页面顶部出现信息横幅（`.alert--warning`）："该订单已使用缸号：DY-2026-001，请优先选择相同缸号"
   - 横幅从顶部 slide-down（250ms）
   - 库存列表中，同缸号库存排在首位，标注绿色"推荐"标签
   - 不同缸号库存排在后面，标注橙色"不同缸号"标签
3. **选择不同缸号时**：
   - 立即弹出强警告模态框（红色边框，感叹号图标）
   - 标题："警告：缸号不一致，可能导致色差"
   - 内容：说明已用缸号 vs 当前选择缸号
   - 操作区：[确认继续（需填写原因）] [重新选择]
   - 选择确认后：原因 textarea 出现（必填），提交后领料单标注"跨缸号"红色标签
4. **订单首次领料时**：
   - 无约束提示，自由选择
   - 选择后系统记录为该订单基准缸号
   - 提交成功时 Toast："已记录该订单基准缸号：DY-2026-001"

**动画效果**：
- 警告横幅 slide-down，200ms ease
- 强警告模态框：scale from 0.9 + fade-in，250ms spring
- 推荐/不推荐标签切换：在 list 中 reorder 时 transition 位置（FLIP 动画）

## 三、多单位物料入库/出库/领料 单位切换交互

### 3.1 单位切换触发

**用户行为**：用户点击入库/出库录入页的单位下拉选择器。

**页面状态变化**：
1. 点击单位下拉：下拉列表展开（fade-in + slide-down，150ms）
2. 列表内容：所有配置的单位（采购单位、库存单位、生产领用单位）
3. 选择新单位：
   - 下拉收起
   - 数量输入框旁的单位文字切换（字体 transition，200ms）
   - 若当前数量不为空：换算提示区域（`.unit-switcher__hint`）从下方展开，显示实时换算结果
   - 换算提示背景 `--color-info-50`，边框 1px 虚线 `--color-info-300`

**换算提示文案示例**：
- "1 箱 = 50 个，当前 3 箱 = 150 个（将以库存单位入库）"
- "换算说明：采购单位→库存单位，系数 50"

**动画效果**：
- 换算提示展开：max-height 0→80px，opacity 0→1，300ms ease
- 数字变化（换算后数值）：fadeUp 微动效（translateY 4px→0，opacity 0→1，200ms）

**错误/边界情况**：
- 该 SKU 未配置多单位换算：单位下拉不显示切换选项，仅显示默认库存单位
- 输入数量为 0 或空时：换算提示不显示（避免 "0 = 0" 的无意义提示）
- 换算后库存不足：提示区域切换为警告色 `--color-warning-50`，文字"换算后出库 X 个，当前库存仅剩 Y 个，库存不足"

### 3.2 库存列表单位切换

**用户行为**：在库存总览表格中点击列表头的单位切换按钮。

**页面状态变化**：
1. 所有数量列统一切换为新单位
2. 数字变化时：所有数量单元格同步执行 number-flip 动画（旧数字淡出，新数字从下方淡入，150ms）
3. 列头单位标签更新
4. 当前单位持久化到 localStorage（刷新后保持）

## 四、每日排产计划查看→调整→下发 流程

### 4.1 排产计划查看

**用户行为**：车间主管打开排产计划页面。

**页面状态变化**：
1. 若今日计划已生成（07:30后）：正常显示甘特图
2. 若今日计划生成中：显示全页 AI 思考中状态（参见第一节）
3. 若计划生成失败：显示错误状态 + 原因说明 + [重新生成] 按钮
4. 若在07:30前（计划尚未生成）：显示"今日排产计划将于 07:30 自动生成"的空状态页

**AI 风险提示横幅**：
- 若 AI 检测到排产风险，页面顶部显示 `.alert--warning` 横幅
- 横幅可展开查看详情（accordion，300ms）
- 可关闭（关闭后记录到 sessionStorage，刷新后复现）

### 4.2 任务拖拽调整

**用户行为**：车间主管拖动甘特图中的任务块。

**触发条件**：鼠标按下（mousedown/touchstart）任务块超过 200ms（防止误触）。

**拖拽过程状态**：
1. 按下 200ms 后，任务块进入"可拖拽"状态：
   - 原位置出现虚线占位框（`border: 2px dashed --color-gray-300`，背景 `--color-gray-50`）
   - 被拖动的任务块 opacity 变为 0.8，轻微放大（scale 1.02），box-shadow 增强
   - cursor 变为 grabbing
2. 拖动过程中：
   - 目标落位区域高亮（背景 `--color-primary-50`，边框 `--color-primary-300` 虚线）
   - 实时检测时间冲突：冲突区域高亮为红色警告
3. 放下时：
   - 任务块 snap 到最近的有效时间格（CSS transition 200ms）
   - 若无冲突：占位框消失，任务块落位
   - 若有冲突：任务块弹回原位（bounce 动画），Toast 警告："与已有任务时间冲突，请重新安排"

**调整原因记录**：
- 移动任务块后，出现小型 popover："请填写调整原因（可选）"，textarea
- 3s 无操作自动关闭不填写，记录"主管手动调整，原因未填写"

### 4.3 确认下发

**用户行为**：点击"确认并下发给工人"按钮。

**页面状态变化**：
1. 点击后：确认弹窗，汇总本次计划（X 名工人，Y 条任务）
2. 确认后：
   - 按钮变为 loading + "下发中..."
   - 页面各工人任务块上依次出现发送成功勾号动画（逐个出现，间隔 100ms）
   - 全部完成后：页面状态标签变为"已下发"，绿色
   - Toast："计划已下发，X 名工人已收到任务通知"（5s）
3. 下发失败（某工人通知失败）：
   - 整体成功但部分失败：黄色 Toast，"X 名工人通知成功，Y 名工人通知失败，请检查账号状态"
   - 失败工人任务块标注红色感叹号
   - 提供 [重新通知失败工人] 按钮

## 五、小程序扫码入库/出库 流程

### 5.1 扫码触发

**用户行为**：点击"扫码"按钮或扫描图标。

**页面状态变化**：
1. 请求摄像头权限（若未授权）：系统弹出权限请求弹窗
2. 权限拒绝：Toast 提示"需要摄像头权限才能扫码，请在设置中开启" + [去设置] 链接
3. 摄像头开启：进入扫码界面（全屏摄像头视图 + 中央扫码框 + 底部提示文字）
4. 扫码框四角有动态扫描线（绿色线条从上到下滑动，1.5s loop）
5. 识别成功：短振动（haptic feedback）+ 扫码框变绿并显示勾号 + 自动关闭扫码界面（200ms）
6. 识别失败（无法识别的条码）：扫码框短暂变红 + Toast "未找到对应物料，请手动搜索"

### 5.2 物料识别后的状态填充

**扫码成功后**：
1. 返回入库表单页，物料名称、规格、单位字段自动填充（动画：字段从灰色占位符文字 → 真实内容，typewriter 效果，150ms）
2. 自动聚焦到"数量"输入框，软键盘弹起（数字键盘）
3. 若为面料类物料：缸号字段自动展开（参见 2.1）

**手动搜索**：
1. 搜索框聚焦 → 显示历史搜索记录（最近5条）+ 下方实时搜索结果
2. 搜索结果列表：物料名称 + 编码 + 分类，每项高度 56px（手套操作友好）
3. 点击选中：同扫码成功后的状态

### 5.3 入库提交与成功反馈

**提交过程**：
1. 点击"确认入库"大按钮
2. 按钮变为 loading（spinner 居中）
3. 成功：全屏成功反馈页（绿色对勾动画，scale 0→1.2→1，300ms spring）
   - 显示：物料名称、入库数量、更新后库存、是否达到安全库存线
   - 2s 后自动出现操作选项：[继续入库] [返回首页]
4. 失败（库存超限、缸号冲突等）：全屏错误反馈页，红色图标 + 错误原因 + [修改并重试]

## 六、成品验货→问题记录→溯源查询 流程

### 6.1 创建验货单

**用户行为**：QC 验货员点击"新建验货单"。

**页面状态变化**：
1. 弹出验货单创建表单（bottom sheet，从底部 slide-up，400ms）
2. 选择订单（必填）+ 录入待验数量
3. 提交后创建验货单，进入逐件验货视图

### 6.2 逐件验货操作

**用户行为**：验货员对每件成品操作。

**页面状态变化**：
1. 当前件序号显示（"第 1 件 / 共 20 件"）+ 进度条
2. 大按钮：[合格] （绿色）和 [不合格]（红色），高度 64px，满足手套操作
3. 点击[合格]：
   - 按钮短暂 scale 动画（0.95→1，100ms）
   - 件数 +1，进度条更新（transition 300ms）
   - 自动进入下一件（若未到最后）
4. 点击[不合格]：
   - 底部表单展开（问题类型多选、严重程度单选、照片上传、备注）
   - 照片上传：支持拍照/相册，上传中显示图片缩略图 + loading spinner
   - 表单提交后：记录成功，进入下一件

### 6.3 溯源查询

**用户行为**：点击验货记录中的"查看溯源"按钮。

**页面状态变化**：
1. 弹出溯源链 drawer（从右侧 slide-in，350ms）
2. 溯源链加载中：骨架屏（溯源节点形状）
3. 加载完成：节点依次从顶部渐入显示（stagger 效果，每个节点间隔 80ms）
4. 数据完整节点：完整颜色 + 详细信息
5. 数据缺失节点：灰色 + "工序数据缺失" 标注
6. 面料节点：包含缸号标签（`.tag--dye-lot`，橙色虚线边框）

## 七、AI 对话交互详细规范

### 7.1 对话触发

**触发方式**：
- Web 端：点击顶部搜索框（或快捷键 Cmd/Ctrl+K）
- Web 端：点击右下角浮动 AI 按钮（56×56px 圆形，橙色，`--shadow-xl`）
- 小程序端：点击底部 AI 入口或页面内"问 AI"按钮

**打开动画**：
- Web 端浮层：从右下角展开（transform-origin: right bottom），scale 0.8→1 + opacity 0→1，300ms spring
- 小程序：全屏 modal，从底部 slide-up，400ms

### 7.2 消息发送

**用户行为**：在输入框输入文字，按 Enter 或点击发送按钮。

**发送中状态**：
1. 用户消息气泡立即出现（右侧，蓝色）
2. 输入框清空，焦点保持
3. AI 消息区域立即出现 thinking 气泡（三点跳动动画）
4. thinking 气泡内同步更新思考步骤文字（逐步显示）

### 7.3 流式输出

**AI 开始回复时**：
1. thinking 气泡消失（fade-out 100ms）
2. AI 回复气泡出现（fade-in 150ms）
3. 文字逐字追加（每 30-80ms 一个字，模拟打字速度）
4. 气泡末尾显示打字光标（`|`，blink 动画 0.7s）
5. 右下角显示"AI 生成中..."标签（橙色小标签）
6. 结构化内容（表格、列表）：等该模块全部生成完毕后整体渐入显示（fade-in 200ms）

**输出完成时**：
1. 打字光标淡出（opacity 1→0，300ms）
2. "AI 生成中..."标签消失
3. 消息时间戳出现（淡入）
4. 消息气泡底部出现操作区（复制、点赞、点踩，小图标，渐入）

### 7.4 思考中状态（复杂任务）

复杂任务（排产生成、采购分析）在对话中触发时：

```
T+0.0s: thinking 气泡出现
T+0.3s: 第一个步骤文字出现
T+每步完成: 勾号出现 + 下一步激活（带 stagger 动画）
T+10s:  若尚未完成，显示倒计时
```

步骤文字渐进显示（slide-right + fade-in，150ms）

### 7.5 错误/超时状态

**简单查询超时（>15s）**：
1. T+10s：thinking 气泡文字变为"还在思考中，比平时慢一些..."（橙色文字）
2. T+15s：自动重试1次（气泡文字变为"正在重新连接..."）
3. T+30s：进入错误状态

**错误气泡出现**：
1. thinking 气泡替换为错误气泡（交叉淡化，300ms）
2. 错误气泡：红色背景，感叹号图标，错误说明文字
3. 操作按钮：[重试] [手动查看数据]
4. 重试时：错误气泡替换为新的 thinking 气泡（注意保留原始输入内容，不要求用户重填）

**重试规范**：
- 自动重试：用户无感知，仅气泡文字有变化
- 手动重试：点击[重试]按钮后，按钮 loading 状态 → 重新进入 thinking 流程
- 重试记录：后台日志记录（用户无感知）

### 7.6 快捷操作区

对话面板顶部或输入框上方：快捷问题按钮（`.chat__quick-actions`）

```
[现在缺什么料？] [今日排产状态？] [本月库存概况] [有哪些待审批？]
```

点击快捷问题：直接填入输入框并发送（不需要再点发送）

## 八、工人任务接收→完工上报→异常上报 流程

### 8.1 任务接收

**触发**：车间主管确认下发后，工人小程序收到推送通知。

**页面状态变化**：
1. 小程序任务列表刷新（若已打开，实时推送；若未打开，次日首次打开时显示）
2. 新任务以高亮背景（`--color-primary-50`）显示，带"新"标签（badge）
3. 新任务标签在 5s 后自动消失

**任务卡片内容展示**：
- 优先级标签（左上角，颜色区分）
- 工序名称 + 订单号
- 产品名称 + 数量（大字体，24px）
- 预计用时
- 展开区域：所需物料清单（含面料缸号标识）

**面料任务特殊展示**：
- 物料列表中面料条目附带缸号标签（橙色，`.tag--dye-lot`）
- 若有多个缸号要求：列表分行显示，每行一个缸号
- 点击缸号可查看该批次详情（drawer 展开）

### 8.2 开始任务

**用户行为**：工人点击"开始任务"大按钮（高度 54px）。

**页面状态变化**：
1. 按钮点击 → scale 动画（0.96→1，150ms）
2. 成功：任务卡片状态变为"进行中"（黄色边框，计时器开始）
3. 计时器：`HH:MM:SS` 格式，每秒更新，绿色数字（大字体，24px）
4. 按钮组变为："上报完工" + "上报异常"两个按钮

### 8.3 完工上报

**用户行为**：工人点击"上报完工"按钮。

**页面状态变化**：
1. 弹出完工表单（bottom sheet）：
   - 完成数量（默认 = 计划数量，数字输入框，软键盘数字）
   - 损耗数量（可选，下拉选择原因）
   - 扫码确认（可选，US-404）
2. 完工数量 = 0 时：表单无法提交，输入框边框红色 + 提示"完成数量不能为0"
3. 完工数量 > 计划数量：警告提示（可确认继续）

**扫码部件（可选步骤）**：
- 工人点击"扫描部件标识"
- 扫码成功：部件信息卡片出现（部件名称、订单、成品），工人确认
- 工人可点击"跳过"（跳过的部件在溯源链标注"工序数据缺失"）

**提交完工**：
1. 按钮 loading
2. 成功：全屏成功页（绿色勾 + "完工上报成功！" + 工序耗时）
3. 任务卡片状态变为"已完工"（绿色背景，勾号图标）
4. 列表自动滚动到下一个待处理任务

### 8.4 异常上报

**用户行为**：工人点击"上报异常"按钮。

**页面状态变化**：
1. 弹出异常上报表单（bottom sheet，高度约 60vh）：
   - 异常类型（大按钮单选：缺料 / 质量问题 / 设备故障 / 其他）
   - 异常描述（textarea，占位符引导文字）
   - 拍照（可上传 1-3 张，缩略图预览）
2. 选择异常类型后，按钮高亮（对应颜色）
3. 拍照上传中：缩略图位置显示 loading skeleton
4. 提交后：
   - Toast："异常已上报，车间主管已收到通知"（5s）
   - 任务卡片出现"异常"红色角标
   - 工人可继续操作其他任务（不强制等待主管响应）

## 九、离线状态交互规范

### 9.1 离线检测与提示

**触发**：设备网络断开（监听 offline 事件）。

**页面状态变化**：
1. 立即（< 1s）：顶部出现橙色离线条（`.offline-bar`），从顶部 slide-down（250ms）
2. 离线条内容："当前处于离线模式，数据将在联网后自动同步 | 了解可用功能"
3. 需要网络才能操作的功能（AI 对话、审批提交）：按钮变为 disabled 状态 + 置灰 + tooltip "需要网络连接"

### 9.2 数据本地缓存（出入库录入）

**离线状态下提交出入库**：
1. 数据保存到 localStorage（加密）
2. 成功反馈页显示"数据已保存，将在联网后同步"（橙色文字，非绿色）
3. 任务列表本地缓存，可查看不可提交完工

### 9.3 恢复在线

**网络恢复**：
1. 离线条变为绿色"已恢复网络连接，同步中..."（500ms 后消失）
2. 自动上传缓存数据，上传进度在离线条处显示（若有缓存数据）
3. 同步完成：Toast "同步完成，X 条数据已上传"
4. 冲突数据（服务器端同时有变更）：弹窗通知用户，提供手动解决选项
