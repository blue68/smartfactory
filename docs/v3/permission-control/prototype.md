# 权限控制模块 Prototype

## 1. 信息架构

PC 端新增一级菜单：`系统管理`

当登录账号为 `platform_super_admin` 时，页面顶部增加全局“租户上下文”区域：

- 当前模式：`平台态 / 租户态`
- 当前租户：未选择或已进入的租户名称
- 操作：`切换租户`、`退出租户上下文`

系统管理下包含 6 个页面：

1. 租户配置
2. 菜单与功能
3. 角色配置
4. 人员配置
5. 角色授权
6. 人员角色分配

建议访问关系：

- 平台超级管理员：可见全部 6 页，并可通过顶部上下文切换进入目标租户
- 租户管理员/老板：默认可见“角色配置 / 人员配置 / 角色授权 / 人员角色分配”
- 普通业务角色：默认不可见

平台态下的约束：

- 可查看租户列表、系统模板、平台审计
- 不直接展示租户业务数据页面
- 进入租户后，页面顶部显示“当前正在代管：<租户名称>”

## 2. 页面原型总览

### 2.0 平台登录后首页 / 租户切换

#### 页面布局

- 顶部：平台身份标识、当前上下文、`切换租户` 按钮
- 主体左侧：平台级导航
- 主体右侧：租户列表卡片/表格

#### 核心交互

- 平台登录后默认落在平台首页，不自动进入任何租户
- 点击 `进入租户` 后弹出二次确认，说明即将进入代管模式
- 进入租户成功后刷新顶部上下文与权限快照
- 点击 `退出租户上下文` 返回平台态

#### 异常流

- 目标租户已停用：禁止进入，提示“目标租户已停用”
- 当前账号不是 `platform_super_admin`：不显示租户切换入口

### 2.1 租户配置页

#### 页面布局

- 顶部：页面标题、租户状态统计卡、`新建租户` 按钮
- 中部：筛选区
- 主体：租户列表表格
- 右侧/弹框：租户详情与编辑抽屉

#### 筛选区字段

- 关键字：租户名称/编码
- 状态：全部/启用/停用/到期
- 套餐：标准版/专业版/旗舰版/自定义

#### 表格字段

- 租户编码
- 租户名称
- 状态
- 套餐
- 功能模块数
- 默认管理员
- 到期日
- 最后更新时间
- 操作

#### 编辑弹框字段

- 租户名称
- 租户编码
- 联系人
- 联系电话
- 状态
- 套餐类型
- 到期日
- 默认管理员账号
- 默认管理员姓名
- 默认模板
- 功能开关列表
- 备注

#### 关键交互

- 新建时必须校验租户编码唯一
- 停用租户前弹出二次确认
- 套餐切换时自动更新功能开关建议值
- 平台超级管理员可从租户行直接点击 `进入租户`

#### 异常流

- 编码重复：阻止提交，提示“租户编码已存在”
- 停用租户但仍有在线用户：允许提交，但提示“将影响当前在线会话”

## 3. 关键状态流

### 3.1 新租户开通流

1. `platform_super_admin` 进入租户配置页
2. 点击 `新建租户`
3. 填写租户基础信息与默认模板
4. 系统生成默认管理员账号、预置角色和默认菜单授权
5. 返回租户详情页，状态为“启用”

### 3.2 新角色创建与授权流

1. 租户管理员进入角色配置页
2. 点击 `复制系统角色`
3. 修改角色名称与说明
4. 保存后跳转至角色授权页
5. 勾选菜单与功能点并设置数据范围
6. 保存成功，角色进入可分配状态

### 3.3 人员建档与角色分配流

1. 租户管理员新建人员
2. 系统提示“是否立即分配角色”
3. 跳转人员角色分配页
4. 勾选多个角色并设置主角色
5. 保存后人员获得登录权限

## 4. 批量操作定义

支持以下批量操作：

- 批量启用人员
- 批量禁用人员
- 批量分配角色
- 批量取消角色
- 批量启停菜单
- 批量启停功能点

批量操作统一规则：

1. 勾选数量为 0 时按钮置灰
2. 提交前展示影响摘要
3. 提交后返回成功/失败明细

## 7. 角色-菜单-功能点-数据范围关系

| 角色 | 菜单 | 功能点 | 数据范围 |
| --- | --- | --- | --- |
| admin | 全部系统管理菜单 | 全部 | all |
| boss | 角色配置、人员配置、角色授权、人员角色分配 | 查看、编辑、分配、导出 | all |
| supervisor | 人员配置、人员角色分配 | 查看、分配指定班组成员 | production_line_assigned |
| warehouse | 无系统管理菜单 | 无 | warehouse_assigned |
| sales | 无系统管理菜单 | 无 | customer_assigned |

说明：

1. `platform_super_admin` 默认停留在平台态，不直接管理任意租户业务数据。
2. `platform_super_admin` 必须先显式切换到目标租户上下文，才可进入该租户的系统管理页。
3. 老板为本租户最高业务管理员，但默认不管理其他租户。
4. 主管可被授权为“人员分配协管”，仅可见被分配范围内人员。

## 8. 页面提示与说明文案

关键提示文案建议：

- 菜单与功能页：`菜单决定入口可见，功能点决定按钮可用。`
- 角色授权页：`一个用户可以拥有多个角色，权限按并集计算。`
- 数据范围区：`数据范围仅决定看哪些数据，不决定是否能执行功能。`
- 人员角色分配页：`主角色用于默认首页和组织标识，多个角色权限同时生效。`
- 顶部上下文栏：`当前为平台代管视角，所有操作将记录到权限审计。`

## 9. 空态、错态、禁用态

### 9.1 空态

- 无角色：提示“尚未创建角色，可先复制系统角色模板”
- 无人员：提示“尚未录入人员，可先新建账号”
- 无授权：提示“该角色尚未配置权限”

### 9.2 错态

- 租户无效：提示“当前租户状态不可操作”
- 授权冲突：提示“部分授权对象已失效，请刷新后重试”
- 跨租户引用：提示“不可引用其他租户的数据对象”

### 9.3 禁用态

- 系统预置菜单删除按钮禁用
- 禁用角色不可被分配
- 禁用人员不可分配新角色
