# 权限控制模块 User Story

## 1. 用户故事清单

### US-001 平台级登录与进入租户

作为 `platform_super_admin`，我希望无需填写 `tenantCode` 即可先登录平台控制台，并显式切换进入目标租户上下文，以便在不打穿多租户边界的前提下执行平台代管。

验收条件：

- 平台登录不要求输入 `tenantCode`
- 登录后默认停留在平台态，不能直接查看租户业务页面
- 可从顶部或租户配置页选择“进入租户”
- 每次切换和退出租户上下文都生成审计记录

### US-002 租户开通

作为 `platform_super_admin`，我希望创建新租户并选择默认权限模板，以便新工厂可以快速启用系统。

验收条件：

- 可录入租户名称、租户编码、联系人、状态、到期日
- 可选择“家具制造标准模板”之类的默认权限模板
- 创建完成后自动生成默认管理员账号和预置角色

### US-003 租户功能开关

作为 `platform_super_admin`，我希望按租户启停 AI、报表、结算等模块，以便匹配不同付费套餐。

验收条件：

- 可查看租户已启用模块清单
- 禁用模块后，租户管理员和普通用户看不到对应菜单
- 已禁用模块接口调用返回无权限

### US-004 菜单树配置

作为 `platform_super_admin`，我希望维护系统菜单树和页面功能模块，以便统一管理导航结构。

验收条件：

- 可新增一级菜单、二级菜单、页面节点
- 可配置菜单名称、图标、路由、排序、所属分组
- 系统预置菜单不可直接删除，只能停用

### US-005 功能点配置

作为 `platform_super_admin`，我希望在每个页面下维护功能点，以便角色能精确到按钮级控制。

验收条件：

- 可为“销售订单管理”页面配置 `查看/新建/编辑/审批/导出`
- 功能点有唯一编码和中文名称
- 功能点可设置默认启用状态

### US-006 自定义角色创建

作为租户管理员，我希望创建“销售主管”“仓库盘点员”等自定义角色，以适配组织分工。

验收条件：

- 可复制系统角色为新角色
- 可编辑角色名称、编码、说明、角色状态
- 禁用角色后不能再分配给人员

### US-007 角色授权

作为租户管理员，我希望给角色分配菜单、功能点和数据范围，以便角色获得可控权限。

验收条件：

- 可在树形界面勾选菜单
- 可在右侧勾选功能点
- 可设置数据范围模板
- 保存后可即时查看授权摘要

### US-008 人员建档

作为租户管理员，我希望维护员工账号，以便完成系统登录和权限分配。

验收条件：

- 可录入账号、姓名、手机号、部门、岗位、状态
- 可设置初始密码或发送重置链接
- 禁用人员后不可登录系统

### US-009 人员角色分配

作为租户管理员，我希望为一个员工分配多个角色，以便兼容兼职岗位。

验收条件：

- 支持选择多个角色
- 支持设置主角色
- 支持生效时间和失效时间
- 可查看当前生效角色列表

### US-010 权限生效验证

作为老板，我希望授权调整后能立刻影响系统菜单和按钮可见性，以便确认配置真实生效。

验收条件：

- 重新登录后菜单变化可见
- 没有被授权的按钮不显示
- 未授权接口执行返回无权限

### US-011 审计追踪

作为老板或 `platform_super_admin`，我希望查看谁修改了谁的角色和权限，以及谁何时切换进入了哪个租户上下文，以便责任可追溯。

验收条件：

- 每次授权变更记录操作人、时间、对象、差异内容
- 审计记录支持按人员、角色、时间筛选
- 平台态“进入租户 / 退出租户”有独立审计类型

### US-012 兼容旧角色

作为系统管理员，我希望现有固定角色账号在新权限中心上线后继续可用，以避免系统中断。

验收条件：

- 老账号不做人工迁移也能登录
- 旧角色能映射到预置角色模板
- 原有关键菜单不出现大面积缺失
- 普通租户账号登录方式保持不变，仍要求填写 `tenantCode`

### US-013 批量授权

作为租户管理员，我希望一次给多名员工分配同一角色，以提升管理效率。

验收条件：

- 支持列表勾选人员批量分配角色
- 支持覆盖或追加模式
- 批量结果可查看成功/失败明细

## 2. 用户故事优先级

P0：

- US-001 平台级登录与进入租户
- US-002 租户开通
- US-004 菜单树配置
- US-005 功能点配置
- US-006 自定义角色创建
- US-007 角色授权
- US-008 人员建档
- US-009 人员角色分配
- US-010 权限生效验证
- US-012 兼容旧角色

P1：

- US-003 租户功能开关
- US-011 审计追踪
- US-013 批量授权

## 3. 关键约束

1. 所有用户故事均以租户隔离为前提。
2. 系统预置角色必须可读但受限编辑。
3. 菜单可见与功能可执行必须分离设计。
4. 人员角色分配要支持一人多角色。
5. 平台级管理员必须通过“切换租户上下文”进入租户态，不能默认无边界跨租户操作。

## 4. 统一验收口径

1. UI 层：菜单/按钮显隐正确。
2. API 层：未授权请求返回权限错误。
3. 数据层：跨租户不可见。
4. 审计层：授权变更可追踪。
