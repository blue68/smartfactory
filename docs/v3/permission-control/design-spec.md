# 权限控制模块 Design Spec

## 1. 设计目标

权限控制模块要与现有后台系统保持统一的“企业后台”语言，而不是单独做一套“管理台中台风格”。设计原则如下：

- 以白底卡片、轻阴影、浅边框为主。
- 以表格、筛选栏、抽屉、弹框、树形授权为核心信息载体。
- 颜色克制，以蓝色为主强调，辅以绿色、橙色、红色表达状态。
- 单页信息密度高，但通过分区卡片和右侧抽屉降低认知负担。

## 2. 视觉基线

### 2.1 设计语言

- 页面整体采用浅灰渐变或微弱径向渐变背景。
- 内容区采用圆角白色卡片，边框浅灰，阴影轻。
- 主按钮使用蓝色，危险按钮使用红色，次按钮为白底描边。
- 状态展示以标签、徽标、树节点半选、开关、禁用灰态表达。

### 2.2 基础 token

建议沿用现有后台风格，并固定如下 token：

- `color-bg-page`: `#f4f6fb`
- `color-surface`: `#ffffff`
- `color-surface-2`: `#f8fbff`
- `color-border`: `#dbe3ef`
- `color-border-soft`: `#e5ebf5`
- `color-text-primary`: `#1a202c`
- `color-text-secondary`: `#64748b`
- `color-primary`: `#3b82f6`
- `color-primary-hover`: `#2563eb`
- `color-success`: `#10b981`
- `color-warning`: `#f59e0b`
- `color-danger`: `#ef4444`
- `color-info`: `#0ea5e9`

### 2.3 圆角与阴影

- 页面级卡片：`12px - 16px`
- 列表卡片：`12px`
- 表单输入：`8px`
- 标签：`6px` 或 `999px`
- 阴影只用于悬浮和分组，不做强烈投影。

## 3. 页面结构规范

### 3.1 通用结构

每个页面建议统一为：

1. 页面标题区
2. 汇总统计区
3. 筛选区
4. 主内容区
5. 抽屉/弹框区

### 3.2 通用组件规则

- `统计卡`：用于租户数、角色数、人员数、启用菜单数等汇总。
- `筛选栏`：统一使用一行或两行布局，左侧关键词，中间条件筛选，右侧主操作。
- `表格`：列头固定层级清晰，操作列靠右，状态列使用标签。
- `抽屉`：用于详情、编辑、授权结果展示，宽度建议 `min(1100px, 92vw)`。
- `弹框`：用于新增、危险确认、复制角色、批量授权，宽度建议 `560-920px`。
- `树授权`：使用可展开树 + 右侧功能点面板，支持半选、全选、搜索、展开收起。

## 4. 页面级设计规范

### 4.1 租户配置

- 目标：治理租户生命周期、套餐与功能开关。
- 布局：顶部统计卡 + 筛选栏 + 租户表格 + 右侧详情抽屉。
- 重点信息：租户状态、套餐、功能模块数、默认管理员、到期日。
- 强调：租户启停、功能开关、默认模板。

### 4.2 菜单与功能

- 目标：维护系统菜单树与功能点树。
- 布局：左侧菜单树 + 右侧菜单详情卡 + 下方功能点表。
- 重点信息：菜单层级、路由、分组、系统预置、状态。
- 强调：页面节点与功能点分离，支持拖拽排序或上下移。

### 4.3 角色配置

- 目标：管理角色定义与基础属性。
- 布局：顶部统计卡 + 左侧角色列表 + 右侧角色详情。
- 重点信息：角色编码、类型、状态、数据范围、绑定人数。
- 强调：系统预置角色不可删，仅可复制和禁用。

### 4.4 人员配置

- 目标：管理系统登录人员与组织信息。
- 布局：统计卡 + 筛选栏 + 人员表格 + 详情抽屉。
- 重点信息：账号、姓名、手机号、部门、岗位、角色数、状态、最后登录。
- 强调：支持新建、启停、重置密码、批量导入。

### 4.5 角色授权

- 目标：给角色配置菜单、功能点和数据范围。
- 布局：左侧角色列表 + 中央菜单树 + 右侧功能点/数据范围。
- 重点信息：已授权菜单、已授权功能点、数据范围摘要。
- 强调：授权变更需要即时反馈并有保存前提示。

### 4.6 人员角色分配

- 目标：给人员分配一个或多个角色。
- 布局：人员列表 + 角色分配面板 + 批量分配入口。
- 重点信息：主角色、附加角色、生效/失效时间、是否覆盖。
- 强调：一人多角色与批量分配为核心效率场景。

## 5. 组件规则

### 5.1 表格

- 表头采用浅灰底和细分隔线。
- 数值列使用等宽数字风格。
- 状态列统一使用标签，不直接纯文本展示。
- 操作列最多保留 2 到 4 个按钮，其余放入更多菜单。

### 5.2 标签

- 启用：绿色
- 禁用：灰色
- 待处理：橙色
- 危险/删除：红色
- 预置：蓝色或中性标签

### 5.3 树

- 父节点半选态必须明显。
- 系统预置节点使用“锁”或弱提示。
- 未授权节点默认不高亮。

### 5.4 抽屉与弹框

- 抽屉用于“看详情/看授权结果/看影响范围”。
- 弹框用于“新建/编辑/危险确认/复制/批量操作”。
- 重要表单字段支持分组分栏，避免单列过长。

## 6. 响应式规则

- `>= 1440px`：双栏与三栏分区可并行展示。
- `1024px - 1439px`：左右双栏保留，部分区域折叠成上下结构。
- `< 1024px`：左侧树转顶部切换或抽屉式面板，表格保留横向滚动。
- `< 768px`：筛选栏变为堆叠模式，抽屉与弹框改为全宽。

## 7. 状态风格

- 加载态：骨架屏或轻量 loading 文案，不要空白。
- 空态：显示原因和主动作入口，例如“暂无角色，先新建角色”。
- 错态：红色提示条，保留重试按钮。
- 禁用态：按钮、开关、树节点均需要灰化且不可点击。
- 危险态：删除、停用、重置权限等动作统一二次确认。

## 8. 预置菜单分组

建议系统管理一级菜单下预置如下分组：

- `平台治理`
- `组织权限`
- `人员管理`
- `授权中心`

页面映射关系：

- 租户配置 → 平台治理
- 菜单与功能 → 平台治理
- 角色配置 → 组织权限
- 人员配置 → 人员管理
- 角色授权 → 授权中心
- 人员角色分配 → 授权中心
