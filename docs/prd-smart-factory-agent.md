## 一、WHY — 背景与问题定义

### 1.1 业务背景

非标家具制造业属于典型的小批量多品种定制型生产，单品订单金额高、交期要求严格、物料种类繁多。该类工厂普遍面临以下行业困境：

- 生产计划依赖老板经验，难以量化
- 物料管理停留在Excel阶段，缺料风险高
- 库存占用资金比例过大，资金周转慢
- 工序协同靠口头沟通，效率低下

本项目客户为一家年产值约1200万的非标家具工厂，规模15-30人，无ERP/MES系统，全程依赖Excel和人工协调运转。当前核心痛点导致订单交期延误率偏高，库存占用资金保守估计超过全年营收的20%。

### 1.2 市场与竞品分析

| 方案类型 | 典型产品 | 适配性评估 |
|---|---|---|
| 传统ERP | 用友、金蝶 | 实施周期长、成本高、不适合小规模工厂 |
| 轻量MES | 简道云、钉钉定制 | 可配置但AI能力弱，无智能决策 |
| AI运营助手 | 通用ChatBot | 无行业数据模型，缺乏落地能力 |
| 本产品 | 智造管家 | 轻量SaaS + AI Agent决策，专为小型非标工厂设计 |

### 1.3 核心问题陈述

> 一个15-30人的非标家具工厂，在没有IT基础设施的情况下，如何用最低成本实现物料精准管控、生产有序排产、库存合理压降，并通过AI Agent将老板从日常决策中解放出来？

## 二、WHAT — 产品定义

### 2.1 产品愿景

让每一个中小制造工厂都能用上"AI工厂运营官"，通过数据驱动替代经验驱动，实现降本增效。

### 2.2 用户角色定义

| 角色 | 描述 | 核心诉求 | 使用频率 |
|---|---|---|---|
| 工厂老板 | 最终决策者，资金管控核心；以手机为主要使用端，电脑为辅 | 掌控全局、减少日常干预、降低库存占用、管控销售过度承诺风险 | 每日1-3次 |
| 采购员 | 负责原材料采购，执行老板指令 | 获得精准采购建议、减少缺料、避免过量备货 | 每日多次 |
| 仓库管理员 | 管理原材料/半成品/成品出入库 | 实时准确的库存状态、简单易用的录入方式 | 每日多次 |
| 车间主管 | 负责生产排程、工序协调 | 清晰的每日排产计划、工序优先级可视化 | 每日多次 |
| 生产工人 | 执行具体工序，领用物料 | 知道今天做什么、领什么料、交什么货 | 每班次 |
| 销售人员 | 对接客户，录入订单 | 快速下单、了解产能和交期、跟踪交付状态 | 每日多次 |

### 2.3 业务目标与量化指标

| 目标维度 | 当前状态（估算） | 6个月目标 | 12个月目标 |
|---|---|---|---|
| 缺料导致停工次数 | 每月约8-12次 | 降低至每月3次以内 | 降低至每月1次以内 |
| 库存周转天数 | 估算60-90天 | 降低至50天以内 | 降低至35天以内 |
| 库存资金占比 | 原材料45%/半成品40%/成品15% | 原材料35%/半成品35%/成品30% | 原材料30%/半成品30%/成品40% |
| 采购决策等待时长 | 依赖老板，平均等待2-4小时 | 80%采购建议自动生成，等待时长降至30分钟 | 90%采购建议自动审批，老板仅处理异常 |
| 订单准时交付率 | 估算70-75% | 提升至85% | 提升至92% |
| 数据录入准确率 | 采购90%，生产环节缺失 | 整体达到95% | 整体达到98% |

### 2.4 MVP范围定义（第一期）

MVP聚焦以下4个核心闭环：

1. **库存数字化闭环**：从Excel迁移 + 实时出入库录入 + 库存预警
2. **采购建议闭环**：基于库存+订单的AI采购建议 + 老板确认执行
3. **生产排产闭环**：基于订单优先级的每日排产计划 + 工序任务推送
4. **销售订单闭环**：常规下单 + 智能约束引擎 + 基础交付确认

## 三、HOW — 功能方案

### 3.1 系统整体架构概述

产品形态：
- **Web端**（PC）：适合老板、采购员、车间主管使用，用于决策与全局管理
- **微信小程序**：适合仓库管理员、生产工人使用，用于现场数据录入和任务接收
- **AI Agent对话入口**：嵌入Web端和小程序，支持自然语言交互，降低使用门槛

### 3.2 功能模块清单

## 四、非功能需求

### 4.1 性能要求
- 页面加载时间 < 2秒（P95）
- AI建议生成时间 < 10秒（复杂排产计划）
- 小程序首屏加载 < 1.5秒
- 系统可用性 >= 99.5%（工作日）

### 4.2 易用性要求
- 新用户培训时间 <= 2小时（仓库/工人角色）
- 小程序核心操作（出入库录入）不超过3步完成
- 所有AI建议必须提供中文自然语言解释

### 4.3 安全要求
- 采购审批、资金相关操作必须双重确认
- 角色权限隔离（工人不可见经营数据）
- 数据备份每日一次，保留30天
- 敏感数据（供应商报价）加密存储

### 4.4 兼容性要求
- Web端支持Chrome 90+、Edge 90+
- 微信小程序支持微信8.0+
- 移动端适配：iOS 14+、Android 10+

### 4.5 数据要求
- 库存数据实时同步，延迟 < 5秒
- AI模型使用近12个月历史数据训练（冷启动期使用行业基准参数）
- 支持数据导出（Excel/CSV）

### 4.6 部署模式要求
- **同时支持 SaaS 多租户模式和私有化部署模式**
- SaaS模式：标准云端订阅，按月/年计费，数据隔离至租户级别，开箱即用
- 私有化部署模式：支持客户独立部署于自有服务器/内网环境，提供完整部署包与运维文档
- 两种模式共用同一套产品代码，通过配置文件区分部署形态
- 私有化部署须支持离线运行（AI能力可降级为本地轻量模型）
- 数据迁移：SaaS与私有化之间支持数据导出/导入迁移
