## 说明

本文档按用户角色组织用户故事，每个故事包含角色、功能、价值、验收条件和优先级。
P0 = MVP必做；P1 = 二期完成；P2 = 三期规划。

**用户角色一览**：工厂老板、采购员、仓库管理员、车间主管、生产工人、QC验货员、销售人员、AI Agent专项。

## 一、工厂老板（Boss）

---

### US-001 老板驾驶舱一屏掌控全局

**优先级**：P1

As a 工厂老板
I want 打开系统后第一屏就看到当前最重要的经营数据
So that 我不需要反复问人、翻Excel，就能掌握工厂当前状态

**验收条件**：
- [ ] 首页展示：今日在产订单数、本月完工产值、当前库存总金额、近7天缺料预警次数
- [ ] 所有数据刷新延迟 < 5秒
- [ ] 支持一键跳转到对应详情页
- [ ] 数字异常时（如库存金额超历史均值30%）以红色高亮显示
- [ ] PC端和手机端均可正常访问

---

### US-002 老板在手机端快速审批采购建议

**优先级**：P0

As a 工厂老板
I want 在手机上收到AI采购建议推送，并可以直接点击审批通过或驳回
So that 我不需要回到办公室或打电话，随时随地处理采购审批，减少等待时间

**验收条件**：
- [ ] AI生成采购建议后，微信小程序和PC端同时收到推送通知
- [ ] 推送消息包含：建议采购物料名称、数量、预估金额、原因摘要
- [ ] 老板点击后可查看AI建议详情（含推理依据）
- [ ] 支持一键"批准"或"驳回"操作，驳回时必须填写原因
- [ ] 批准后采购员立即收到执行通知
- [ ] 单笔采购金额 > 5000元时，显示二次确认弹窗
- [ ] 审批操作记录可查（时间、结果、备注）

---

### US-003 老板查看库存积压分析与AI降库建议

**优先级**：P1

As a 工厂老板
I want 系统告诉我哪些物料积压严重，并给出具体的消化建议
So that 我能有针对性地减少资金积压，而不是靠感觉去拍板

**验收条件**：
- [ ] 库存分析页面展示各SKU的库存天数（当前库存/日均用量）
- [ ] 自动识别库存天数 > 90天的物料为"呆滞风险"，标注红色
- [ ] AI给出消化建议，如："建议暂停该SKU采购X周"或"该物料可用于订单Y替代Z"
- [ ] 支持按物料分类、金额、库存天数排序查看
- [ ] 数据可导出为Excel

---

### US-004 老板与AI助手进行自然语言对话查询

**优先级**：P1

As a 工厂老板
I want 用说话的方式向AI询问工厂运营状况
So that 我不需要记住系统操作路径，直接问就能得到答案

**验收条件**：
- [ ] AI能正确回答："现在缺什么料？""订单XX什么时候能完工？""本月库存金额比上月怎么样？"
- [ ] 回答使用简洁中文，不使用技术术语
- [ ] 如AI无法确定答案，主动说明"数据不足，建议核实"，不编造数据
- [ ] 对话响应时间 < 5秒（简单查询）< 15秒（复杂分析）
- [ ] 对话历史保留7天

---

### US-005 老板查看质量溯源报告

**优先级**：P1

As a 工厂老板
I want 查看成品质量问题的溯源报告，了解哪些工序、工人、物料批次存在高频质量问题
So that 我能基于数据做出针对性改进决策，而不是靠口头反馈拍脑袋处理质量投诉

**验收条件**：
- [ ] 质量分析页面展示近30/90天的质量问题趋势图（按日/周统计不合格数量）
- [ ] 支持按问题类型（外观/尺寸/功能/材质）、严重程度（轻微/一般/严重）分类查看
- [ ] 可下钻到具体订单的验货单，查看完整溯源链（成品→部件→物料批次→工序→工人）
- [ ] 高频问题TOP5自动标注，并支持一键跳转到对应工人/工序的历史记录
- [ ] 涉及面料缸号的质量问题，溯源链中明确显示缸号批次信息
- [ ] 报告支持导出PDF，用于供应商沟通或内部培训

---

### US-006 老板审批超限订单

**优先级**：P0

As a 工厂老板
I want 当销售下的订单触发资金/产能/库存周转约束时需要我手机审批
So that 我能管控工厂不被过度承诺拖垮，同时不影响销售正常下单效率

**验收条件**：
- [ ] 约束引擎触发拦截后，5分钟内推送审批通知到老板微信小程序
- [ ] 推送消息摘要包含：订单摘要、触发约束类型（资金/产能/周转）、触发数值 vs 阈值
- [ ] 审批页面完整展示：订单详情、约束触发原因、AI影响分析（受影响订单、资金占用变化、库存周转天数变化）、AI建议操作
- [ ] 支持三种审批操作：批准（直接放行）/ 驳回（退回销售修改）/ 附条件批准（如调整交期后批准，需填写调整要求）
- [ ] 审批操作记录完整可查（时间、结果、附条件内容）
- [ ] 老板超过4小时未处理时系统二次推送催办提醒
- [ ] 审批通过后排产自动将该订单纳入计划；驳回后销售人员收到驳回原因通知

## 二、采购员（Purchaser）

---

### US-101 采购员接收AI采购建议并执行

**优先级**：P0

As a 采购员
I want 系统自动告诉我今天需要采购什么、采购多少、找哪个供应商
So that 我不需要自己翻库存和订单数据，减少遗漏和错买

**验收条件**：
- [ ] 系统每日7:00自动生成采购建议清单（基于库存预警+订单BOM需求）
- [ ] 每条建议包含：物料名称/规格、建议数量、推荐供应商、预计到货日期、建议原因
- [ ] 采购员可查看每条建议的计算依据（如："订单A需要10个，库存剩余3个，缺口7个"）
- [ ] 老板已审批的建议，采购员可标记"已下单"并填写实际下单数量和供应商
- [ ] 如采购员不认可建议，可发起人工调整并说明原因

---

### US-102 采购员录入采购到货信息

**优先级**：P0

As a 采购员
I want 货到后在系统里快速录入到货信息，并通知仓库收货
So that 库存数据实时更新，不再出现"货已到但系统没更新"的情况

**验收条件**：
- [ ] 采购员可在PC端或小程序填写到货记录：采购单号、物料名称、到货数量、供应商
- [ ] 录入后系统自动通知仓库管理员确认收货
- [ ] 仓库确认入库后，库存数量实时更新
- [ ] 支持部分到货（如下单100件，先到60件）
- [ ] 到货异常（数量不符、质量问题）可在系统标注，并推送给老板

---

### US-103 采购员用自然语言查询库存状态

**优先级**：P1

As a 采购员
I want 在采购AI助手中直接问"XX板材还有多少？够不够订单XX用？"
So that 我在跟供应商谈价格时能立刻知道底数，不需要去翻库存表

**验收条件**：
- [ ] AI助手正确解析物料名称（含模糊匹配，如"红橡木板"能匹配"红橡实木板材"）
- [ ] 回答包含：当前库存数量、已被订单占用数量、可用数量
- [ ] 如涉及多个规格，列出各规格分别的库存
- [ ] 响应时间 < 5秒

---

### US-104 采购员查看供应商交货绩效

**优先级**：P2

As a 采购员
I want 查看各供应商的历史准时交货率和质量异常记录
So that 我在选择供应商时有数据依据，而不是仅凭关系和习惯

**验收条件**：
- [ ] 供应商列表展示：近3个月准时交货率、质量异常率、平均交货天数
- [ ] 支持按准时率排序，快速识别优质和问题供应商
- [ ] 点击供应商可查看每笔交易明细

---

### US-105 采购员执行三单匹配对账

**优先级**：P0

As a 采购员
I want 系统自动对比PO、送货单、入库单的数量和价格差异，只把有异常的单子推给我处理
So that 我不再手动逐单核对，只需要关注和确认差异，大幅减少对账工作量

**验收条件**：
- [ ] 系统按PO号自动关联对应的送货单和入库单，完成三单匹配计算
- [ ] 数量、价格、物料规格完全一致的自动标记"已匹配"，无需人工干预
- [ ] 存在差异的标记"待处理"，差异项以红色高亮展示（含差异方向和差异数值）
- [ ] 采购员确认差异原因（从预设选项选择：供应商少发/入库漏录/价格调整/其他）后，方可完成结算
- [ ] 确认差异原因后，相关记录自动归档并可追溯
- [ ] 支持按供应商维度汇总月度对账单，可导出Excel用于线下对账
- [ ] 价格异常（本次采购价超出该SKU历史均值20%）时，即使数量匹配也标注价格预警

## 三、仓库管理员（Warehouse）

---

### US-201 仓库管理员用小程序扫码完成入库

**优先级**：P0

As a 仓库管理员
I want 货物到达时用微信小程序扫描或手动填写完成入库录入
So that 库存数量实时准确，不需要再手写纸质台账或回办公室录Excel

**验收条件**：
- [ ] 小程序支持扫描物料条码/二维码自动填充物料信息
- [ ] 无条码时支持手动搜索物料名称
- [ ] 入库录入字段：物料名称、数量、入库类型（采购入库/生产完工入库）、备注
- [ ] 全流程完成 <= 3步操作
- [ ] 录入成功后显示更新后的实时库存数量
- [ ] 录入操作记录不可删除（只能申请撤销，撤销需采购员或主管确认）

---

### US-202 仓库管理员处理领料出库申请

**优先级**：P0

As a 仓库管理员
I want 收到车间领料请求后在小程序确认出库，实时更新库存
So that 出库有记录可追溯，不再出现"料被拿走但不知道谁拿的"情况

**验收条件**：
- [ ] 车间主管通过系统发起领料申请（物料名称、数量、对应订单）
- [ ] 仓库管理员小程序收到待处理领料申请提醒
- [ ] 仓库管理员确认实际发料数量（可与申请数量不同，填写差异原因）
- [ ] 确认后库存实时扣减，车间主管收到发料通知
- [ ] 如当前库存不足，系统拒绝出库并触发缺料预警

---

### US-203 仓库管理员查看实时库存总览

**优先级**：P0

As a 仓库管理员
I want 随时查看各类物料的实时库存数量，并知道哪些已低于安全库存
So that 我能主动提前预警，不需要等到真正缺料才发现

**验收条件**：
- [ ] 小程序和PC端均可查看库存总览
- [ ] 低于安全库存的物料以红色/橙色标注
- [ ] 支持按物料分类筛选（原材料/半成品/成品）
- [ ] 支持关键字搜索物料名称
- [ ] 数据实时更新，刷新后延迟 < 5秒

---

### US-204 仓库管理员辅助完成月度盘点

**优先级**：P1

As a 仓库管理员
I want 系统生成盘点表，我录入实盘数量后系统自动分析差异
So that 月度盘点不再是噩梦，差异能快速定位原因

**验收条件**：
- [ ] 系统可按物料分类导出盘点底表（含物料编码、名称、系统账面数量）
- [ ] 支持录入实盘数量，自动计算盈亏差异
- [ ] 差异超过5%的物料自动标红提示
- [ ] 盘点结果确认后，系统库存以实盘数为准自动调整
- [ ] 盘点记录永久保存，可查历史

---

### US-205 仓库管理员按缸号入库面料

**优先级**：P0

As a 仓库管理员
I want 在面料/皮料入库时录入缸号信息，并在库存中按缸号维度查看各批次数量
So that 车间领料时能准确知道每个缸号有多少库存，避免同订单面料混用不同缸号造成色差

**验收条件**：
- [ ] 入库录入界面：当物料分类为"面料"或"皮料"时，缸号字段自动显示为必填项
- [ ] 支持手动输入缸号（文本，最多20字符），系统自动记录该缸号的入库日期
- [ ] 同一SKU多次入库不同缸号时，在库存中以独立行展示（SKU名称相同，缸号不同，数量独立）
- [ ] 库存总览页对面料类物料显示"缸号详情"展开按钮，点击后列出该SKU下所有缸号批次及各批次剩余数量
- [ ] 入库时如填写的缸号与该SKU已有缸号一致，系统自动合并计入对应批次数量，并更新入库日期记录
- [ ] 全流程完成 <= 4步操作（在原3步基础上增加缸号填写）

---

### US-206 仓库管理员按多单位管理物料出入库

**优先级**：P0

As a 仓库管理员
I want 入库时按采购单位录入数量（如按"箱"入库五金件），出库时按库存或生产单位录入（如按"个"发料），系统自动完成单位换算
So that 我不需要手动换算单位，录入更符合实际操作习惯，同时保证库存数据准确

**验收条件**：
- [ ] 入库录入界面：单位字段为下拉选择，默认显示该SKU的采购单位，可切换至库存单位；选择后数量输入框右侧显示对应单位名称
- [ ] 出库录入界面：单位字段默认显示库存单位，面料类物料可切换至生产领用单位（如平方米、片）
- [ ] 选择非库存单位录入时，界面实时显示换算提示（如"1箱 = 50个，本次入库 = 150个"）
- [ ] 库存看板中的"库存量"字段支持单位切换按钮，可在采购单位和库存单位之间切换显示（不影响底层数据）
- [ ] SKU主数据未配置多单位换算关系时，入库/出库界面仅显示库存单位，不显示切换选项
- [ ] 换算关系由SKU主数据中的配置决定，仓库管理员不可在录入时修改换算系数

## 四、车间主管（Workshop Supervisor）

---

### US-301 车间主管查看并确认每日AI排产计划

**优先级**：P0

As a 车间主管
I want 每天早上看到AI生成的当日排产计划，并可以根据实际情况调整后确认下发
So that 工人知道今天做什么，工序之间不再因为沟通不畅而闲置等待

**验收条件**：
- [ ] 排产计划每日7:30前自动生成并推送给车间主管
- [ ] 计划内容：各工人/工作站的任务清单、对应订单、优先级、所需物料
- [ ] 车间主管可拖拽调整任务顺序或重新分配工人
- [ ] 确认下发后工人小程序立即收到任务通知
- [ ] 计划调整记录可追溯（谁调整了什么，为什么）

---

### US-302 车间主管发起领料申请

**优先级**：P0

As a 车间主管
I want 在系统中发起领料申请，而不是口头叫仓库管理员
So that 领料有记录，生产物料消耗可追溯，不再出现物料莫名减少

**验收条件**：
- [ ] 车间主管可选择订单后，系统自动带出该订单所需物料（基于BOM）
- [ ] 支持批量申请（一个订单的所有物料一次性申请）
- [ ] 申请提交后仓库管理员实时收到通知
- [ ] 申请状态：待处理/已发料/部分发料/拒绝
- [ ] 如物料库存不足，申请提交时即刻显示预警

---

### US-303 车间主管监控生产进度与异常

**优先级**：P0

As a 车间主管
I want 实时看到各订单的生产进度，以及哪些工序出现异常或延误
So that 我能及时干预，避免订单在最后阶段才发现延误

**验收条件**：
- [ ] 生产进度看板展示：所有在制订单的完成百分比、当前所在工序、预计完工日期
- [ ] 预计完工日期晚于交货日期时，自动标红并推送预警给车间主管和老板
- [ ] 支持点击订单查看工序级别的进度明细
- [ ] 工人上报异常后，车间主管立即收到通知

---

### US-304 车间主管评估临时插单对排产的影响

**优先级**：P1

As a 车间主管
I want 客户临时要求插单时，系统告诉我这单插进来会对现有订单造成多大影响
So that 我在跟老板汇报时有数据支撑，而不是靠直觉猜测

**验收条件**：
- [ ] 车间主管可输入插单信息（产品、数量、期望交期）进行模拟分析

## 六、QC验货员（Quality Inspector）

### 角色说明

### US-701 QC验货员记录质量问题并溯源

**优先级**：P1

As a QC验货员
I want 在验货时用小程序逐件记录质量问题，并一键查看该成品的部件溯源链
So that 质量问题有据可查，能快速定位是哪道工序、哪个工人、哪批物料造成的，不再靠猜测互相推责

**验收条件**：
- [ ] QC验货员在小程序选择订单后，创建验货单，录入待验货数量
- [ ] 逐件验货：对每件成品标记"合格"或"不合格"；不合格时必须选择问题类型（外观/尺寸/功能/材质，支持多选）和严重程度（轻微/一般/严重）
- [ ] 支持拍照上传问题图片（每个问题最多3张），图片自动关联到对应问题记录

- [ ] 溯源链中标注数据完整度：有扫码记录的部件显示完整链路，无扫码记录的部件标注"工序数据缺失，仅可追溯至物料批次"
- [ ] 验货完成后，验货结果（合格率、问题清单）自动推送给车间主管和老板

## 七、销售人员（Sales）

---

### 角色说明

销售人员负责对接客户、录入销售订单，是销售承诺与工厂产能之间的桥梁。该角色主要使用 Web 端，以手机浏览器为辅。核心诉求是快速下单、当场给客户准确承诺交期，同时避免过度承诺导致工厂被动。

| 维度 | 说明 |
|---|---|
| 使用端 | Web 端（主）+ 手机浏览器（辅） |
| 使用频率 | 每日多次 |

### US-801 销售人员录入常规订单

**优先级**：P0

As a 销售人员
I want 在系统中快速录入客户订单并实时获知库存和交期反馈
So that 我能当场给客户准确承诺交货时间，不再靠猜测

**验收条件**：
- [ ] 下单流程：选择客户 → 选择产品 → 填写数量和期望交期 → 提交
- [ ] 提交时系统实时校验：库存可用性（含在制品）、产能负荷、基于当前排产给出预估最早交期
- [ ] 如预估交期晚于期望交期，显示黄色预警，并给出最早可承诺交期建议
- [ ] 如通过约束引擎校验，直接下单成功；如触发约束（资金/产能/周转阈值），系统拦截并提示"已提交老板审批，请等待"
- [ ] 下单成功后系统自动触发该订单的 BOM 物料需求计算，推送给采购员
- [ ] 销售人员收到下单确认通知，含订单编号和预估交期

---

### US-802 销售人员提交紧急插单

**优先级**：P0

As a 销售人员
I want 提交紧急插单时看到该插单对工厂现有排产和资金的影响分析
So that 我能在跟客户协商时有数据支撑，避免盲目承诺导致工厂生产失衡

**验收条件**：
- [ ] 下单时勾选"紧急插单"标记，触发 AI 影响分析（分析完成前禁止提交，展示"AI正在评估中..."）
- [ ] AI 影响分析结果展示：受影响订单列表及各订单延期天数、库存周转天数变化、新增资金占用金额、预估增加生产成本
- [ ] 销售人员可查看分析详情后选择：继续提交（进入审批流）或取消插单
- [ ] 超出约束阈值（如资金占用超预算80%）时，系统自动标注"高风险插单，需老板审批"
- [ ] 老板审批通过后系统通知销售人员，排产自动调整纳入插单
- [ ] AI 影响评估完成时间 < 30 秒

### US-803 销售人员修改已有订单

**优先级**：P0

As a 销售人员

- [ ] 修改操作留存完整记录：修改时间、修改人、修改前后对比、修改原因

### US-501 AI Agent主动推送缺料预警

- [ ] 高置信度建议的实际准确率（3个月后回归验证）应 >= 80%

- [ ] 系统自动重试1次，重试失败后才展示错误界面
- [ ] 错误界面提供"重试"和"跳过，手动操作"两个选项
